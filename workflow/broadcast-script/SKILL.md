# 五蕴法口播稿生成工作流

> 把认知产物编排成一条「色→受→想→行→识」的口播叙述链，并可按需求注入朴素之道的表达风格。

---

## 触发条件

当用户表达以下意图时激活本工作流：

- “写一条口播稿”
- “生成口播稿”
- “用朴素之道的风格写口播稿”
- “五蕴法口播稿”
- “把XX做成口播/短视频脚本”

---

## 输入识别

从用户输入中提取以下信息：

| 字段 | 必需 | 说明 |
|---|---|---|
| 主题 | 是 | 口播稿要讨论的核心话题 |
| 目标受众 | 否 | 默认普通大众，可指定为职场人、家长、创业者等 |
| 风格 | 否 | 默认「朴素之道」，可选通用口语/知识科普/故事型 |
| 时长 | 否 | 默认3分钟，可选1分钟/5分钟 |
| 升华方向 | 否 | 默认按个人→家庭→社会→国家递进，可指定重点层次 |
| 参考理论/书籍 | 否 | 用户希望引用的具体书籍或理论 |

**风格映射**：

| 用户表达 | 调用风格 |
|---|---|
| 朴素之道/朴素老师/像他那样讲 | `../distillation/output/pusuzhidao.skill/SKILL.md` |
| 通用口语/没有指定 | 仅使用五蕴法框架，保持口语化 |
| 知识科普 | 在「想」蕴增加理论密度，减少个人感受 |
| 故事型 | 在「色」蕴延长叙事，「想」蕴后置 |

## 依赖加载

### Step 1: 加载五蕴法框架

读取 `../../content-framework/broadcast-script/broadcast-script.md`，获取：

- 五蕴结构（色·受·想·行·识）
- 每一蕴的内容职责、组织要点、风格约束
- Few-shot 示例

### Step 2: 加载风格DNA（如需要）

如果用户要求「朴素之道」风格，读取 `../distillation/output/pusuzhidao.skill/SKILL.md`，重点提取：

- 心智模型：学习归因模型、接地气翻译模型、理性观察者模型、认知提升喜悦模型
- 决策启发式：开场必带场景、抽象概念先比喻、具体数字替代模糊、口语转折优先、结尾给认知升级或反问、禁用说教腔、允许自嘲、先承认复杂性再给出判断
- 表达DNA：短句、大白话、具体数字、轻微自嘲、断言但留余地
- 反模式黑名单

## 工作流程

### Step 3: 主题认知（调用 cognitive）

对复杂主题，先调用 `../../cognitive/general-logic/references/methodology.md` 进行简要分析：

1. **本质定义先行**：用一句话点破主题本质
2. **本质检验**：确保本质定义能解释该主题的核心现象
3. **输出**：一句话点破 + 3-5个关键认知角度

> 如果主题非常简单明确（如用户已给出现象），可跳过此步骤，直接进入五蕴填空。

### Step 4: 按五蕴生成初稿

按顺序生成每一蕴的内容，遵循以下规则：

#### 色 · 现象切入

- 用具体场景开场，场景必须包含：人物、动作、冲突/反差
- 场景结束后用一句话抛出核心问题
- 朴素之道风格：加具体数字，用反问收尾

#### 受 · 情感共鸣

- 表达真实感受（自我/他人/群体）
- 不要评价对错，先共情
- 朴素之道风格：用“说白了”“说到底”过渡，承认复杂性

#### 想 · 理论点亮

- 引用1-2个有名书籍/理论
- 把理论翻译成大白话，对应回前面场景
- 朴素之道风格：抽象概念先比喻，最多两本书，不堆砌术语

#### 行 · 行动落点

- 给出具体可执行的动作或思考方向
- 不用空喊口号
- 朴素之道风格：用提问式/场景式/认知式行动，留判断空间

#### 识 · 主题升华

- 按个人→家庭→社会→国家选择升华方向
- 升华必须从前面的逻辑自然推导
- 朴素之道风格：用反问或认知升级收尾，不用大词喊口号

### Step 5: 风格润色

如果启用朴素之道风格，对初稿执行以下润色：

1. **句式检查**：每句不超过15字，长句拆分
2. **词汇替换**：
   - “我们应该” → “你可以”
   - “很多人认为” → “我认识的五个人里，三个都这样”
   - “由此可见/综上所述” → “说白了”“说到底”
3. **节奏调整**：段落间插入口语转折词
4. **结尾强化**：把号召式结尾改为反问或认知升级
5. **自检说教腔**：删除所有“必须”“应该”“由此可见”

### Step 5.5: 口播化润色（关键优化）

这一步专门解决"录制时不自然"的问题：

1. **插入停顿标记**：
   - 每15-20字检查是否需要 `/` 换气
   - 段落之间加 `//` 重停顿
   - 关键结论前加 `，` 或 `——` 制造悬念

2. **加入语气词和口头禅**：
   - 段首加“你看”“你想”“说实话”“说白了”
   - 关键转折处加“其实”“说到底”“然后呢”
   - 共情处加“对吧”“是不是”“你也这样”

3. **允许口语化重复和修正**：
   - “这个、这个”“说白了、说白了”少量重复
   - “你可以理解为……”“怎么说呢……”等自我修正

4. **每段开头用"你"或"我"开场**：
   - 避免“这件事”“这个问题”等 distant 开场

5. **标记重音和情绪**：
   - 重要数字、核心概念用 `【重】`
   - 轻松处用 `【笑】`
   - 需要观众反应处用 `【停顿】`

6. **生成两个版本**：
   - **录制标记版**：保留 `/` `//` `【重】` `【停顿】` 等标记
   - **纯净朗读版**：删除所有标记，只留自然段落

### Step 5.6: 真人感润色（从“能读”到“像人说”）

这是最关键的一步，专门解决“机械感”问题：

1. **插入情绪废料**：
   - 在合适位置加“真的”“其实吧”“怎么说呢”“我当时就想”
   - 例："我当时就想，/这台风跟锅里的水，/是不是一回事？"

2. **加入自我修正**：
   - 允许先说一个近似说法，再修正
   - 例："也不是怪兽吧，/反正就是那种……特别大的气旋。"

3. **允许口语化重复**：
   - 重要概念多说一遍，制造强调和现场感
   - 例："它要散热，/它得散热，/不然海洋就热爆了。"

4. **增加即时互动**：
   - 中间不断抛给观众小问题
   - 例："你发现没？/每次台风来之前，/新闻都这么说。"

5. **加入个人锚点**：
   - 每篇至少一个“我”的具体经历或观察
   - 例："我去年看台风路径图，/盯了一晚上，/就看它怎么拐。"

6. **让比喻粗糙一点**：
   - 用“掰弯”“甩出去”“蔫了”等生活动词
   - 避免“科里奥利力使其旋转”这类精确术语

7. **打乱完美逻辑**：
   - 用“反正”“怎么说呢”“其实也不是”等词软化逻辑推进
   - 允许前后顺序微调，更像聊天

### Step 6: 质量自检

**必检项**（任一不通过则修正后重新生成）：

- [ ] 五蕴五段全部存在且内容完整
- [ ] 色蕴有具体场景（人物+动作+反差）
- [ ] 受蕴表达了真实感受，不是直接评价
- [ ] 想蕴引用了1-2个有名理论/书籍，并翻译成了大白话
- [ ] 行蕴给出了具体可执行动作，不是空喊口号
- [ ] 识蕴有升华，且升华由前文逻辑推导而来
- [ ] 全文无AI套话（“探索”“旅程”“赋能”“深入”“全面”等）
- [ ] 全文无说教腔（“我们应该”“必须认识到”“由此可见”等）

**朴素之道风格额外必检项**：

- [ ] 短句为主，每句尽量不超过15字
- [ ] 有具体数字或具体场景替代模糊表述
- [ ] 段落间有口语转折词
- [ ] 结尾是反问或认知升级，不是“点赞关注”
- [ ] 抽象概念有生活比喻

**录制友好必检项**（解决不自然问题）：

- [ ] 每15-20字有一个呼吸点（`,` `/` 或自然断句）
- [ ] 每段开头用“你”或“我”开场
- [ ] 关键数字/比喻前后有停顿
- [ ] 全文有至少3处直接跟观众对话（“你看”“你想”“你发现没”）
- [ ] 没有连续两个超过20字的长句
- [ ] 录制标记版保留 `/` `//` `【重】` `【停顿】` 等标记
- [ ] 纯净朗读版删除所有标记后仍然自然流畅

**真人感必检项**（解决机械感问题）：

- [ ] 每篇至少有1个“我”的个人经历或观察
- [ ] 全文至少有2处自我修正或口语化重复
- [ ] 全文至少有3处情绪废料（“真的”“其实吧”“怎么说呢”等）
- [ ] 全文至少有5处即时互动（“对吧”“你发现没”“是不是”等）
- [ ] 比喻用生活动词，避免精确术语
- [ ] 逻辑推进不过于严密，允许“反正”“怎么说呢”等软化词
- [ ] 听起来像真人边想边说，而不是AI在背诵

## 输出格式

输出一份可直接朗读的口播稿文本，不显示五蕴标记，只保留自然段落流。

### 默认输出结构

每次生成输出两个版本：

```
【口播稿标题】

=== 纯净朗读版（给观众/发布用）===

（色：自然段落开场，抛出场景和问题）

（受：表达感受，建立共鸣）

（想：理论点亮，大白话解释）

（行：行动落点，具体可执行）

（识：主题升华，反问或认知升级收尾）

=== 录制标记版（给录制用，保留停顿/重音标记）===

（色：带 / // 【重】 【停顿】 标记的版本）

（受：……）

（想：……）

（行：……）

（识：……）
```

### 可选：带分段提示版

如果用户需要用于剪辑或分镜，可在每段开头用 `# 色` `# 受` `# 想` `# 行` `# 识` 作为内部标记，但最终朗读稿中应移除。

## 输出示例

### 输入

> 用朴素之道的风格，写一个关于“学会拒绝”的3分钟口播稿。

### 输出

**标题：为什么你越不好意思拒绝，别人越敢麻烦你**

=== 纯净朗读版 ===

你有没有遇到过这种人？明明自己已经很忙了，别人一开口让你帮忙，你还是硬撑着说“没问题”。我朋友小王就是这样，上周他加班到晚上十点，同事还让他顺路带一份资料。他明明不顺路，还是绕了半个城市送过去。

说实话，我以前也这样。总觉得拒绝别人会显得自己很小气，怕破坏关系。可后来发现，那个总是麻烦你的人，根本不会因为你帮了他一次就收敛。他只会觉得：这个人好说话，下次还可以找他。

这让我想起《被讨厌的勇气》里说的“课题分离”。说白了就是，谁的事谁负责。他请求帮忙是他的课题，你答不答应是你的课题。我们把这两个课题混在一起，才会一边委屈自己，一边又不敢说出口。

那落到生活里，你可以怎么做？下次再遇到这种情况，先别急着说“好”。你可以先问一句：“这件事急吗？我手边也有个事在处理。”不是拒绝，是给双方一个重新评估的机会。如果对方真的急，你再帮；如果对方只是顺手推给你，这句话本身就把球踢回去了。

说到底，学会拒绝不是让你变得冷漠，是让你把精力留给真正重要的人和事。一个连自己都照顾不好的人，其实也很难真正帮到别人。你说是不是这个理？
