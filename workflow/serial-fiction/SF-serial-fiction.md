# 场景 E2：连载小说 · 多章节叙事系统

> 将知识、案例、话题或原创构想转化为多章节连载小说，通过世界观、人物、情节的渐进展开，持续吸引读者。

> **方法论**：三阶合一 —— 阶段1（是什么·点出本质·本质定义先行）→ 阶段2（为什么·建立因果·含反常识根因）→ 阶段3（怎么做·展开方法·视角升级）。
>
> **思考层 vs 输出层分离**：三阶合一、本质定义先行、顿悟触发是**思考层脚手架**（智能体写作前思考必走），但章节外形由**故事节奏决定**（可以是动作开场、对话开场、场景描写），不强制章节开头抛本质。详见 `../../cognitive/general-logic/references/methodology.md` 第九章。
>
> **连载小说场景特点**：本质定义用于"**核心概念内核**"的提炼——**每章思考层一个本质**。世界观、人物、情节在思考层由本质定义锚定；章节推进即本质的逐层展开（思考层规划）。产物章节按故事结构（开端/冲突/反转/寓意）组织，不强制按三阶段分节。
>
> **顿悟触发**（内容要求，非分节要求，每章必备）：一句话点破 + 反常识根因 + 视角升级 三件套必须存在于章节中，但**位置由故事节奏决定**——一句话点破→章节核心冲突/章节标题；反常识根因→章节反转；视角升级→章节结尾留白或下章钩子。不强制章节开头抛本质。
>
> **类-属性-方法-路由由本质定义锚定**：本质定义 → 故事类归属 → 属性展开（世界观/人物/冲突）→ 方法体现（叙事策略）→ 路由适配（章节载体）—— 作为思考工具，不强制露出产物。

---

## 触发条件

- 用户说「写连载小说」「小说连载」「连载XX」
- 用户希望把某个知识领域/历史事件/行业话题写成故事
- 用户要求管理一个已有的连载系列（续写、查看状态、调整大纲）

## 核心流程

```
步骤0：确定小说类型与定位
步骤0.5：每章本质定义先行（思考前置，写作前思考必走，章节按故事节奏组织）
步骤1：世界观构建（思考层走三阶合一，输出层按世界观文档结构）
步骤2：人物设定

步骤4：分季/分章大纲

步骤6：章节配图生成
步骤7：连载前检查（思考层/输出层 双层）
步骤8：推送到公众号草稿箱
步骤9：更新连载状态
```

## 步骤0.5：每章本质定义先行（思考前置步骤）—— 调用 cognitive

> **认知分析细节**走 [../../cognitive/general-logic/references/methodology.md](../../cognitive/general-logic/references/methodology.md) —— 三阶合一、本质定义先行、本质检验、顿悟触发机制、类-属性-方法-路由由本质定义锚定。本步骤只规定编排调用，本质定义/本质检验/三阶段主线/顿悟触发器规划等方法由 cognitive 执行。
>
> **⚠️ 思考层 vs 输出层分离的关键修正点**。本步骤是**智能体写作前思考的必经路径**（思考层），用于确定本章的本质定义、走通三阶合一、规划顿悟触发器；但**章节外形由故事节奏决定**，不强制章节开头抛本质，也不强制按"阶段1/阶段2/阶段3"分节。
>
> **类比**：盖房子时结构工程师用力学公式算承重（思考层），但房子外形由建筑师设计（输出层）。不能因为用了力学公式就让房子长得像公式。写小说同理：思考时走三阶算清本质，但章节按故事节奏写。

### 编排调用（写作前思考必走）

1. **调用 cognitive 提炼本章本质定义**：本章要揭底什么"核心概念内核"？（揭底，不是描述外延）
2. **调用 cognitive 执行本质检验**：本质定义能否解释本章所有关键属性（场景选择、人物动机、冲突设计、钩子方向）？
3. **确定故事类归属**：由本质定义确定本章属于什么类（"信任建立类""权力博弈类""成长顿悟类"）
4. **调用 cognitive 走通三阶段主线**（思考层规划内容，不强制章节分节）：阶段1（点出本质）→ 阶段2（建立因果）→ 阶段3（展开方法·视角升级）
5. **调用 cognitive 规划顿悟触发器**（思考层规划，输出层按故事节奏落实）：
   - 一句话点破 → 输出位置由故事节奏决定（章节核心冲突/章节标题——不一定开头抛）
   - 反常识根因 → 输出位置由故事节奏决定（章节反转/人物选择反差点）
   - 视角升级 → 输出位置由故事节奏决定（章节结尾留白/下章钩子/寓意）

### 输出层：按故事节奏组织的章节结构

| 章节开场方式（输出层） | 适用场景 | 思考层三阶落点 |
|------|------|------|
| 动作开场 | 冲突激烈、节奏快 | 一句话点破→动作中暗示；反常识根因→动作后反思；视角升级→动作结果留白 |
| 对话开场 | 人物关系紧张、信息密度高 | 一句话点破→对话核心句；反常识根因→对话反转；视角升级→对话结尾留白 |
| 场景描写开场 | 氛围重要、世界观展示 | 一句话点破→场景后的核心命题；反常识根因→场景中的反差；视角升级→场景收束的隐喻 |
| 心理独白开场 | 内心冲突、成长顿悟 | 一句话点破→独白金句；反常识根因→独白中的自我反诘；视角升级→独白收束的重新定义 |

> **判断准则**：当你不确定某个元素应该出现在章节哪里时，问三个问题：①这是思考工具还是章节内容？②这个位置由故事节奏决定还是由方法论决定？③删掉标签后章节是否更自然？

### 思考层元数据（写入章节 frontmatter，不露出正文）

```yaml
# 思考层元数据，仅作智能体校验用，不在章节正文中露出
chapter_essence: "用一句话点破本章核心本质"  # 思考锚点
chapter_class: "故事类归属"                    # 思考归属
insight_hooks:                                # 顿悟触发器规划，按故事节奏落实
  stage1_one_liner: "一句话点破（落点：章节核心冲突/章节标题）"
  stage2_counter_root: "反常识根因（落点：章节反转/人物选择反差）"
  stage3_perspective_upgrade: "视角升级（落点：章节结尾留白/下章钩子）"
```

### 类-属性-方法-路由映射（思考工具，不强制露出章节）

| 维度 | 本质约束 | 连载小说场景适配 |
|------|---------|----------------|
| **类** | 由本质定义确定归属 | 故事类归属（信任/博弈/成长/...） |
| **属性** | 必须是本质的展开 | 场景、人物、冲突、钩子 |
| **方法** | 必须是本质的能动方式 | 叙事策略 + 章节结构 |
| **路由** | 必须是本质的情境适配 | 章节载体（公众号连载/合集） |

## 步骤0：确定小说类型与定位

### 小说类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 知识小说 | 用故事讲清楚一个知识体系 | 用侦探小说讲逻辑学 |
| 行业小说 | 展现某个行业的规则与人性 | 投行、医院、律所故事 |
| 历史小说 | 在历史背景下虚构人物命运 | 宋代小吏的职场沉浮 |
| 科幻/奇幻 | 构建架空世界，探讨核心议题 | AI 觉醒后的社会 |
| 现实题材 | 贴近当下生活，反映普遍困境 | 中年程序员的转型 |

### 定位三问

1. **读者是谁？** 他们关心什么？
2. **核心冲突是什么？** 人物与世界、人物与人物、人物与自我的冲突。
3. **每期带走什么？** 每章结束时，读者除了悬念，还能获得什么认知或情绪？

## 步骤1：世界观构建

世界观文档是**规划层产物**（思考层），用于指导后续章节写作。按三阶合一主线展开思考（三阶 = 坡度三阶段，一一对应，不重复）：

| 思考阶段 | 问题 | 规划输出 | 顿悟触发器（规划，章节中落实） |
|------|------|------|----------|
| 是什么 | 这是一个什么样的世界？ | 时代、地点、社会规则、核心技术/制度 | 一句话点破世界本质（可作章节标题/开头金句） |
| 为什么 | 这个世界为什么值得写？ | 核心议题、普遍冲突、与读者的连接点 | 反常识根因（世界的反直觉点，章节反转中落实） |
| 怎么做 | 怎么让读者进入这个世界？ | 开场场景、代入感设计、信息释放节奏 | 视角升级（重新定义"这是个什么样的世界"，章节结尾落实） |

输出文档：`output/{series-name}/world-building.md`

详见 [references/world-building-guide.md](references/world-building-guide.md)

## 步骤3：故事主线设计

### 三幕式骨架

| 幕 | 占比 | 内容 |
|----|------|------|
| 第一幕：建置 | 25% | 介绍世界、人物、日常，触发核心冲突 |
| 第二幕：对抗 | 50% | 主角面对障碍，展开主要情节 |
| 第三幕：结局 | 25% | 高潮、解决、人物变化 |

### 核心冲突类型

- **人物 vs 人物**：利益、理念、情感对立
- **人物 vs 世界**：制度、时代、规则压迫
- **人物 vs 自我**：恐惧、欲望、价值观挣扎

## 步骤4：分季/分章大纲

### 分季原则

- 一季对应一个完整的故事弧线
- 每季 8-20 章
- 季末必须有一个阶段性结局，同时留下下一季钩子

### 分章原则

每章大纲必须包含：

| 要素 | 说明 |
|------|------|
| 章节编号 | 如 第1章、第2章 |
| 标题 | 吸引点击的短标题 |
| **章节本质定义** | 一句话点破本章核心本质（核心概念内核） |
| 场景 | 时间、地点 |
| 出场人物 | 本章出现的人物 |
| 本章冲突 | 人物遇到什么麻烦 |
| 情节推进 | 本章让整体故事前进多少 |
| 结尾钩子 | 悬念、反转、情绪爆点 |
| 顿悟触发器 | 一句话点破+反常识根因+视角升级 三件套 |
| 字数目标 | 2000-4000字 |
| 配图需求 | 人物卡/场景图/关系图 |

详见 [references/chapter-outline-template.md](references/chapter-outline-template.md)

## 步骤5：单章写作

> **写作格式规范**（字数/开头/中间/结尾/段落等）走 [../../format/markdown](../../format/markdown) 与 [references/fiction-style-guide.md](references/fiction-style-guide.md)，由 format 提供排版与文体规范。本步骤只规定编排调用顺序。

### 每章必备结构（思考层走三阶，输出层按故事节奏）

```
思考层：调用 cognitive 走阶段1(点出本质) → 阶段2(建立因果·反常识根因) → 阶段3(展开方法·视角升级)
输出层：场景切入 → 冲突升级 → 人物选择 → 意外/转折 → 视角升级收尾(钩子)
```

> **思考层走三阶，输出层按故事节奏**：思考时调用 cognitive 走阶段1/2/3规划内容；写作时章节按故事节奏（场景→冲突→转折→钩子）组织，不强制按"## 阶段1/2/3"分节。

每章必备三件套顿悟触发器（内容要求，位置由故事节奏决定）：
- 一句话点破：落点在章节核心冲突/章节标题/段落首句（不一定开头抛）
- 反常识根因：落点在章节反转/人物选择反差点
- 视角升级：落点在章节结尾留白/下章钩子/寓意

### 下章预告

每章末尾加「下一章：XXX」预告，增强追读动力。预告中可暗示下章的本质（思考层规划，预告中可含蓄表达）。

## 步骤6：章节配图生成

每章配图 3-5 张：

| 类型 | 数量 | 说明 |
|------|------|------|
| 章节封面 | 1张 | 标题+核心场景/人物，wide画幅 |
| 人物卡 | 0-2张 | 新出场人物或重点人物 |
| 场景图 | 1-2张 | 关键场景可视化 |
| 关系图/动态图 | 0-1张 | 人物关系或情节转折动画 |

小说配图风格：
- 强调情绪和氛围
- 可使用插画风格、剪影、场景氛围图
- 文字量较少，视觉焦点明确

## 步骤7：连载前检查（思考层 / 输出层 双层）

### 思考层检查（智能体内部，不露出章节）

| 检查项 | 标准 | 处理方式 |
|--------|------|---------|
| **章节本质定义**（思考层） | 每章在 frontmatter 有 chapter_essence 并通过本质检验 | 缺失则回到步骤0.5补提思考 |
| **三阶段主线无重复**（思考层） | 思考路径阶段2/3不重复阶段1 | 重复则按三阶合一裁剪思考路径 |

### 输出层检查（按故事节奏）

| 检查项 | 标准 | 处理方式 |
|--------|------|---------|
| **顿悟触发器三件套**（输出层） | 一句话点破+反常识根因+视角升级 齐备，位置由故事节奏决定 | 缺失则按故事节奏补植到合适位置 |
| 单章字数 | 2000-4000字 | 不达标则扩展或精简 |
| 开头是否直接 | 3段内进入场景/冲突（不强制抛本质，由故事节奏决定） | 否则删除铺垫 |
| 结尾是否有钩子 | 必须有，且含视角升级（可用寓意/留白引出） | 补充悬念或情绪爆点 |
| 人物是否一致 | 符合人物卡设定 | 不一致则调整 |
| 配图数量 | 3-5张 | 不足则补充 |
| 下章预告 | 必须有 | 补充 |
| 封面图 | 1张 | 生成章节封面 |
| **不露出思考工具** | 章节未列出"本质检验表"、"回溯到本质列"、"## 阶段1/2/3"分节标签 | 如有则删除——这些是思考工具，不露出章节 |

## 步骤9：更新连载状态

更新 `output/{series-name}/series-state.json`：

```json
{
  "series_name": "示例连载",
  "status": "ongoing",
  "current_season": 1,
  "total_chapters_planned": 20,
  "chapters": [
    {"number": 1, "title": "开篇", "status": "published", "word_count": 3200},
    {"number": 2, "title": "入局", "status": "published", "word_count": 2800},
    {"number": 3, "title": "暗流", "status": "draft", "word_count": 0}
  ]
}
```

详见 [references/series-state-spec.md](references/series-state-spec.md)

### 更新规则（迁自 series-state-spec.md）

1. 每次创建新章时，添加章节记录，状态为 `planned`。
2. 完成单章写作后，更新 `word_count`，状态改为 `draft`。
3. 推送发布后，更新 `published_at`，状态改为 `published`。
4. 任何时候修改连载信息，都要更新 `updated_at`。
