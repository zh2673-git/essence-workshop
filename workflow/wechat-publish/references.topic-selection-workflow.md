# 选题诊断与验证工作流（Topic Selection Workflow）

> 选题的"执行流程"：从触发条件到诊断步骤、再到验证闭环与管线衔接。
> 本文件只负责"怎么诊断、怎么验证、怎么衔接管线"，不定义基因与评分标准——后者（三层基因定义、基因评分表、评级标准、补强策略、标题公式库）原位于框架层 content-framework/essay/references/topic-selection-gene.md，已随框架模块精简删除，如需使用应在本工作流目录下重建。
> 由 `pipeline-wechat.md` 在选题阶段调用。

---

## 触发条件

- 用户说「写公众号」但没有给具体话题
- 用户说「帮我选题」「不知道写什么」
- 用户给了一个模糊方向（如"想写医保""想写教育"）
- 用户明确指定论文风格输出

---

## 步骤 -1b：前置三维定位

在基因评分之前，先完成以下三维定位，避免选题脱离读者与平台。

```
步骤 -1b：三维定位
├── 读者维度：目标读者是谁？核心焦虑与身份诉求是什么？
├── 平台维度：内容主要在微信哪个场景传播？（朋友圈 / 群聊 / 推荐流 / 搜一搜）
└── 情绪维度：选题希望触发哪种情绪？（焦虑 / 愤怒 / 好奇 / 认同 / 优越感 / 希望）
```

| 维度 | 关键问题 | 输出格式 |
|---|---|---|
| **读者维度** | 谁需要读这篇文章？他们的职业、年龄、痛点、身份标签是什么？ | 1-2 句话读者画像 |
| **平台维度** | 朋友圈传播需要社交货币，群聊传播需要实用信息，推荐流传播需要强钩子 | 选择主要传播场景 |
| **情绪维度** | 这篇文章想让读者感到什么？情绪是转发的重要燃料 | 选择 1-2 种主情绪 |

## 选题诊断流程

```
步骤-1a：判断用户是否有明确话题
├── 有明确话题 → 跳到步骤-1b
└── 没有明确话题 → 步骤-1b 中追加 1-2 轮追问

步骤-1b：前置三维定位
├── 读者维度：目标读者是谁？
├── 平台维度：主要传播场景是什么？
└── 情绪维度：希望触发什么情绪？

步骤-1c：论文风格判定
├── 用户是否选择论文风格？
│   ├── 是 → 启用 S1/S2/S3 检查
│   └── 否 → 跳过风格适配层

步骤-1d：三层基因评分
├── 传播基因 T1-T4 评分
├── 内容基因 C1-C6 评分
└── 论文风格基因 S1-S3 评分（若启用）

步骤-1e：评级与决策
├── S级 → 直接开写
├── A级 → 开写，补强弱项
├── B级 → 基因补强后再写
└── C级 → 建议换话题

步骤-1f：选题验证（可选，推荐执行）
├── 事前预测：预估打开率 / 转发率 / 完读率
├── 小范围测试：朋友圈 / 小群投放
├── 数据回测：与历史爆款对比基因组合
└── 沉淀：将结果写入选题知识库
```

> **基因定义、评分表与评级标准**：原位于框架层 `content-framework/essay/references/topic-selection-gene.md`，已随框架模块精简删除，如需使用应在本工作流目录下重建。

## 选题验证与迭代闭环

选题确定后，进入验证与迭代流程：

```
步骤-1f：选题验证闭环
├── 事前预测
│   ├── 预估打开率（基于 T1 + T2 + T4）
│   ├── 预估转发率（基于 T2 + T3）
│   └── 预估完读率（基于 C6 + S1）
├── 小范围测试
│   ├── 朋友圈/小群投放标题+摘要
│   ├── 收集 3-5 条真实反馈
│   └── 判断是否需要调整角度
├── 数据回测
│   ├── 与历史爆款对比基因组合
│   ├── 找出同基因组合的高/低表现案例
│   └── 更新对当前选题的预期
└── 沉淀
    ├── 将选题、基因组合、实际表现写入 topic-knowledge-base.md
    └── 为下一轮选题提供历史爆款延伸素材
```

**验证标准**：

| 指标 | 测试方法 | 通过标准 |
|------|---------|---------|
| 标题吸引力 | 小范围投票 | 3 人以上愿意点击 |
| 话题争议性 | 观察讨论 | 能引发不同观点 |
| 证据可获取性 | 预检索 | 30 分钟内找到 2-3 个核心证据 |
| 论文可展开性 | 写 200 字摘要 | 能自然引出 3-4 个分论点 |

## 与输出风格的衔接

选题确定后，风格选择决策如下（写作形式规范见对应的 references 文件）：

```
用户选择论文风格？
├── 是 → 启用 S1/S2/S3 检查，输出阶段执行论文风格改写
├── 否 → 跳过风格适配层
│   ├── 对话风格 → 输出阶段保留 Q&A 结构
│   └── 蒸馏 Skill 风格 → 输出阶段激活 workflow/distillation/output/ 中对应人物 Skill
```
