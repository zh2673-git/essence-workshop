# 微信排版规范（平台约束）

> 只列出微信公众号平台对 HTML/Markdown 的硬性约束。内容风格（加粗策略、emoji用法等）由内容框架层决定。

## Frontmatter 必填字段

文章 Markdown 文件头部必须有 frontmatter，否则推送后标题显示为"未命名文章"：

```yaml
---
title: 文章标题（必填，推送后显示为微信文章标题）
author: 作者名（可选）
digest: 一句话摘要（必填，转发文章时显示的摘要文案）
---
```

**推送前必须检查**：
- frontmatter 中 `title` 字段不能为空。
- frontmatter 中 `digest` 字段不能为空，应为一句话总结全文核心观点（用于转发时的摘要展示）。

## 平台禁止项

| 禁止 | 原因 |
|------|------|
| `:::block` 容器模块（hero/verdict/callout/cta/faq/toc） | 微信草稿箱会剥离容器样式，仅剩裸文字 |
| `# H1` 文章主标题 | H1由标题栏占用，正文中只用 `##` 和 `###` |
| 正文重复出现文章标题 | 标题已在微信标题栏展示 |
| 外部 CSS/JS | 微信不支持，样式必须100%内联 |
| `<style>` 标签或 `class` 属性 | 微信会剥离 |

## 安全替代方案

| 需求 | 不安全写法 | 安全替代 |
|------|-----------|---------|
| 文章开头 | `:::hero` | `>` + 开场引语，或直接正文 |
| 强调观点 | `:::verdict` | `**加粗**` + 文字 |
| 提示/引用 | `:::callout` | `> ` 引用（微信原生 blockquote） |
| 行动号召 | `:::cta` | `---` 分割线 + 文字 |
| 问答 | `:::faq` | `**Q：**` + `**A：**` |

## 结构建议

- **文章开头**可以使用 `>` 引言块营造氛围，但**文章结尾不要使用引言块**。结尾应使用自然段落收束，或总结句直接落笔，避免形成"悬浮的引用框"。
