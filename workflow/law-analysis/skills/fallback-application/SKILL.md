# 兜底适用 · 原子技能

## 法律声明

> 本技能仅对兜底条款与原则的适用进行学理论证，**不构成法律意见**。兜底适用具有高度开放性与风险，最终应由裁判机关或执业法律专业人员审慎判断。

---

## 一、核心概念

### 1.1 兜底条款类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 概括条款 | 授权裁判者进行价值补充 | 诚实信用原则、公序良俗原则 |
| 类推适用 | 类似案件适用类似规则 | 民法漏洞填补 |
| 兜底罪名/行为 | 刑法或行政法中的"其他"条款 | "其他严重情节" |
| 基本原则 | 法律体系的根本价值 | 比例原则、法律保留 |

### 1.2 穷尽具体规则原则

在适用兜底条款前，必须证明：
1. 已检索并排除相关具体规则；
2. 具体规则存在漏洞（明显不公平或无法覆盖新型事实）；
3. 兜底适用不会破坏法律安定性。

### 1.3 法律后果类型

| 领域 | 后果类型 |
|------|----------|
| 民事 | 赔偿损失、恢复原状、停止侵害、赔礼道歉 |
| 刑事 | 自由刑、财产刑、资格刑 |
| 行政 | 警告、罚款、没收、责令停产停业、吊销许可证 |

---

## 二、方法要点

- **穷尽具体规则审查**：列出已尝试的具体规则，说明为何无法适用
- **漏洞识别**：判断是开放漏洞还是隐藏漏洞；漏洞是否可通过扩张解释填补
- **选择兜底规范或原则**：选择最相近的具体规则进行类推，或选择合适的基本原则进行价值补充
- **论证适用**：说明为何兜底适用能够实现规范目的与个案正义
- **限制条件**：不得违反法律保留；不得类推加重刑罚（刑法禁止不利类推）；不得破坏法律安定性

---

## 三、输入/输出描述

### 输入

- `subsumption` 输出的"无法涵摄"结论；
- `norm-retrieval` 输出的候选规范清单。

### 输出

- 穷尽具体规则审查、漏洞识别（类型/说明）、兜底规范选择（规范/理由）、适用论证、限制条件、法律后果（基础后果/调整因素/最终方案）、风险提示。
- 输出格式由 format/markdown 处理，本技能不规定具体文档模板。

---

## 四、示例

**输入**：某新型网络服务合同无法被《民法典》合同编有名合同完全覆盖。

**输出**：

```markdown
### 一、穷尽具体规则审查

| 已尝试规则 | 无法适用的理由 |
|------------|----------------|
| 买卖合同 | 标的是非有形财产 |
| 委托合同 | 双方关系不符合委托特征 |
| 技术服务合同 | 部分特征匹配但不完整 |

### 三、兜底规范选择
- **选择的规范/原则**：《民法典》第467条（无名合同适用法律总则）+ 合同编通则
- **理由**：新型服务合同可参照最相类似的有名合同，并适用合同编通则

### 六、法律后果
- **基础后果**：违约责任
- **调整因素**：服务特性、履行方式
- **最终方案**：参照技术服务合同与委托合同的相关规定处理

### 七、风险提示
- 兜底适用应作为最后手段
- 结论置信度：低
```

---

## 六、失败模式与降级

| 失败场景 | 处理动作 |
|----------|----------|
| 未穷尽具体规则 | 拒绝兜底适用，返回规范检索 |
| 涉及刑法不利类推 | 明确禁止，建议不构成犯罪 |
| 无法确定合适的兜底规范 | 标注"法律漏洞"，建议立法或司法解释 |
| 可能破坏法律安定性 | 降低置信度，建议提交有权机关 |
