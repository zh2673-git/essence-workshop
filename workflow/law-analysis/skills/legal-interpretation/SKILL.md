# 法律解释 · 原子技能

## 法律声明

> 本技能提供的解释结论属于学理论证，**不构成法律意见**。法律解释具有开放性，最终解释结论应由有权机关或执业法律专业人员确定。

---

## 一、核心概念

### 1.1 解释方法顺位

| 顺位 | 方法 | 说明 |
|------|------|------|
| 1 | 文义解释 | 从法律条文的通常含义出发 |
| 2 | 体系解释 | 在上下条文中、在法律整体中理解 |
| 3 | 目的解释 | 探求立法目的与规范目的 |
| 4 | 历史解释 | 参考立法资料、修法过程 |

### 1.2 解释结果类型

| 类型 | 说明 |
|------|------|
| 扩张解释 | 将文义范围内的边缘情形纳入 |
| 限缩解释 | 将文义范围内但不合规范目的的情形排除 |
| 当然解释 | 举重以明轻、举轻以明重 |
| 类比解释 | 法律漏洞填补（民法领域谨慎使用） |

### 1.3 权威支撑顺位

1. 有权解释（立法解释、司法解释）
2. 立法说明与修法资料
3. 最高人民法院指导性案例、公报案例
4. 学者通说
5. 地方法院裁判规则

---

## 二、检索认知原则

> 检索执行（MCP 工具调用、联网搜索、具体数据库访问）由 workflow 负责。本节仅规定认知层面的检索原则。

### 2.1 权威支撑顺位原则

- 按权威性顺序检索：有权解释 → 立法说明/修法资料 → 指导性案例/公报案例 → 学者通说 → 地方法院裁判规则；
- 高顺位来源的支撑力优于低顺位来源。

### 2.2 来源标注原则

- 每条权威支撑必须标注来源类型与具体出处；
- 模型内部知识给出的支撑必须标注 `[待校验]`，不得作为确定结论的唯一依据。

---

## 三、方法要点

- **明确解释对象**：确定需要解释的条文及争议焦点（概念、范围、适用条件）
- **文义解释**：确定通常含义与可能的文义范围
- **体系解释**：分析上下文关系及与其他条文的协调
- **目的解释**：探求规范目的与立法目的
- **历史解释**：参考立法沿革与修法说明
- **权威支撑检索**：按权威支撑顺位检索并标注来源
- **形成解释结论**：明确采用何种解释方法、结论为何、置信度如何

---

## 四、输入/输出描述

### 输入

- `subsumption` 输出的涵摄分析报告（标注存疑或需解释的要件）；
- 或用户直接提出的解释问题。

### 输出

- 解释对象（条文/争议焦点）、四阶解释结果（文义/体系/目的/历史）、权威支撑（顺位/来源/支持结论/效力状态）、解释结论（采用方法/结论/置信度）、风险提示。
- 输出格式由 format/markdown 处理，本技能不规定具体文档模板。

---

## 五、示例

**输入**：需要解释《民法典》第1165条"过错"是否包括过失。

**输出**：

```markdown
### 一、解释对象
- **条文**：《民法典》第1165条
- **争议焦点**："行为人因过错侵害他人民事权益"中的"过错"是否包括过失

### 二、文义解释
- **通常含义**："过错"包括故意和过失
- **文义范围**：核心含义明确

### 三、体系解释
- 与《民法典》第1165条第2款（过错推定）并列，过错为一般侵权构成要件

### 七、解释结论
- **采用方法**：文义解释 + 体系解释
- **结论**："过错"包括故意和过失
- **置信度**：高
```

## 七、失败模式与降级

| 失败场景 | 处理动作 |
|----------|----------|
| 文义本身清晰但当事人仍争议 | 指出无解释必要，建议回到涵摄 |
| 权威支撑冲突 | 并列展示不同观点，标注主流意见 |
| 无法检索权威来源 | 结论标注 `[待校验]` |
| 涉及重大法律原则争议 | 建议咨询执业律师或提交有权机关 |
