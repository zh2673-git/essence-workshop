# 规范检索 · 原子技能

## 法律声明

> 本技能输出的是候选法律规范清单及检索路径，**不构成法律意见**。所有法条引用的最终效力与准确性，必须由执业法律专业人员通过官方法律数据库复核确认。

---

## 一、核心概念

### 1.1 正式法源层级

| 位阶 | 制定机关 | 示例 |
|------|----------|------|
| 宪法 | 全国人大 | 《中华人民共和国宪法》 |
| 法律 | 全国人大及其常委会 | 《民法典》《刑法》《行政处罚法》 |
| 行政法规 | 国务院 | 《劳动合同法实施条例》 |
| 地方性法规 | 省级/设区的市人大及其常委会 | 《广东省公路条例》 |
| 部门规章 | 国务院各部委 | 《工资支付暂行规定》 |
| 地方政府规章 | 省级/设区的市政府 | — |
| 司法解释 | 最高法/最高检 | 法释〔2020〕17号 |

### 1.2 两层检索结构

**第一层：立法法正式法源（所有案件必检）**

| 领域 | 检索顺位 |
|------|----------|
| 民事 | 合同/类合同 → 无因管理 → 物权 → 不当得利 → 侵权 |
| 刑事 | 刑法分则罪名 → 刑法总则 → 单行刑法 → 司法解释 |
| 行政 | 法律 → 行政法规 → 地方性法规 → 部门规章/地方政府规章 → 司法解释 |
| 宪法基本权利 | 宪法 → 法律 → 比例原则与法律保留 |

**第二层：行政规范性文件（按条件触发）**

| 情形 | 是否自动检索 |
|------|-------------|
| 行政机关行使职权的案件 | ✅ 自动 |
| 行政机关作为民事主体 | ❌ 不自动 |
| 民事/刑事/宪法案件 | ❌ 不自动（可按需提示） |
| 领域不明或行民交叉 | ⚠️ 提示用户 |

### 1.3 民事请求权九阶检视

```
第一阶：合同上的请求权
第二阶：单方法律行为的请求权
第三阶：类契约请求权
第四阶：身份法上的请求权
第五阶：无因管理请求权
第六阶：物权请求权
第七阶：占有保护请求权
第八阶：不当得利请求权
第九阶：侵权请求权
```

## 二、检索认知原则

> 检索执行（MCP 工具调用、联网搜索、具体数据库访问）由 workflow 负责。本节仅规定认知层面的检索原则。

### 2.1 检索顺位原则

- **正式法源优先**：先查立法法正式法源，再按条件触发行政规范性文件。
- **高位阶优先**：同一领域内从高位阶到低位阶检索（宪法→法律→行政法规→地方性法规→规章→司法解释）。
- **特别法优于一般法**：同一位阶存在特别与一般关系时，先查特别法。
- **民事请求权九阶检视**：民事案件按九阶顺序检视，避免遗漏请求权基础。

### 2.2 来源与效力校验原则

- 每条规范必须标注来源与检索时间；
- 必须标注效力状态（VALID / AMENDED / REPEALED / UNCERTAIN）；
- 无法确认效力的法条标注 `[效力存疑]`，不得作为确定性结论的唯一依据；
- 模型内部知识给出的法条必须标注 `[待校验]`，禁止将未校验法条作为确定性结论的唯一依据。

---

## 三、方法要点

- **检索准备**：确认法律关系领域（来自 `legal-relationship-typing`）、争议焦点与核心事实、地域信息（涉及地方性规定时）
- **正式法源检索**：按领域检索顺位，从高位阶到低位阶检索
- **行政规范性文件检索（条件触发）**：案件涉及行政行为且满足三要件时，自动进入第二层检索
- **请求权基础检视（民事案件）**：按九阶顺序检视请求权基础，避免遗漏
- **效力与来源校验**：交叉验证法条现行效力；标注来源、检索时间、效力状态

---

## 四、输入/输出描述

### 输入

- `legal-relationship-typing` 输出的法律关系定性报告；
- 或用户直接提供的领域与争点信息。

### 输出

- 检索前提（领域/争点/地区）、正式法源候选规范清单（含条文摘要与效力状态）、行政规范性文件清单（如触发）、民事请求权九阶检视结果、待校验条目与风险提示。
- 输出格式由 format/markdown 处理，本技能不规定具体文档模板。

## 七、失败模式与降级

| 失败场景 | 处理动作 |
|----------|----------|
| 检索执行不可用或受限 | 转由 workflow 决定降级策略；本技能仅要求所有结果标注来源，内部知识给出的法条必须标注 `[待校验]` |
| 无法确认法条效力 | 标注 `[效力存疑]`，降低置信度 |
| 法条来源无法核实 | 标注 `[待校验]`，不得作为确定性结论的唯一依据 |
| 地方性规定需要但无地域信息 | 提示用户补充地域信息 |
