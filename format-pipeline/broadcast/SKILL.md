# 口播稿管线

> 将认知产物按5镜头结构组装为口播稿 Markdown。管线只管格式转换——认知分析、风格引用、寓言化等由上层（场景层/认知层）处理后传入，本管线只负责按格式规范组装。产物为通用口播稿格式，下游平台由上层指定。

## 输入输出

- **输入**：认知产物（已由上层处理，可能已应用风格/寓言化），可携带 `--style`、`--narrative-mode` 等参数（仅供格式层感知，不在此处执行）
- **输出**：`broadcast-script.md`，5镜头结构，800-1500字（对应2-5分钟口播）

## 5镜头结构

| 镜头 | 时长 | 内容定位 | 画面/白板动作 |
|------|------|---------|--------------|
| 钩子 | 15-20s | 抛出问题/冲突/反直觉 | 标题 + 关键词 |
| 是什么 | 30-45s | 核心概念定义 | 核心概念框 |
| 为什么 | 60-90s | 因果联系建立 | 关系图/对比 |
| 怎么做 | 60-120s | 方法/步骤/应用 | 流程图/示例 |
| 收尾 | 15-30s | 总结 + 升华 | 回到标题 |

## 写作规则

- **口语优先**：每句≤20字，用"我们/你"而非"笔者/读者"
- **一段一动作**：每个段落对应一个画面/白板动作提示
- **结构齐全**：5镜头缺一不可，按顺序组织
- **总字数**：800-1500字（对应2-5分钟口播）

## 生成流程

```
1. 读取认知产物（已含三阶内容，可能已由上层应用风格/寓言化）
2. 将内容映射到5镜头结构（钩子/是什么/为什么/怎么做/收尾）
3. 写口语化短句（每句≤20字）
4. 为每个段落添加画面/白板动作提示
5. 自检：字数/句长/5镜头齐全/画面提示
6. 输出 broadcast-script.md
```

> 注：风格引用、寓言化、视角替换等认知处理不在本管线执行；若输入未做处理，应退回上层（场景层/认知引擎）。

## 失败模式

| 现象 | 处理 |
|------|------|
| 字数<800 | 补充案例或细化"怎么做"段落 |
| 字数>1500 | 删减或拆分为多集 |
| 句子>20字 | 强制拆分为短句 |
| 缺画面提示 | 为每个段落补全动作提示 |
| 5镜头缺失 | 补全缺失镜头段落 |
| 输入未做认知处理 | 退回上层（场景层/认知引擎）重新处理 |

## 质量自检

| 检查项 | 标准 |
|--------|------|
| 字数 | 800-1500字 |
| 句长 | 每句≤20字 |
| 5镜头 | 钩子/是什么/为什么/怎么做/收尾齐全 |
| 画面提示 | 每段落有动作提示 |
| 口语化 | 无书面语，可用"我们/你" |

## 与其他管线的关系

- **上游**：认知引擎层产出认知产物；风格引用、寓言化、视角选择等由上层（场景层/认知层）处理后传入，本管线不负责
- **下游**：口播稿是通用格式，下游平台由上层指定（可为白板录制、视频号、B站、抖音等），由平台适配层处理具体规格（分辨率、时长、字幕等）
- **并列**：视频管线（video）可用口播稿作为旁白来源；白板管线（whiteboard）可用画面提示生成场景
