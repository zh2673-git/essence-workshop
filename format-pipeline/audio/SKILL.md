# 音频管线

> 将文本元素转为音频产物。适用于播客、朗读版、对话音频等场景。

## 输入输出

- **输入**：认知产物中的文本元素（因果链、方法步骤、一句话点破等）
- **输出**：.mp3 音频文件（含可选背景音乐）

## 音频风格（由上层指定）

风格由上层通过 `--style` 参数指定，格式层不自行选择。支持的格式：

| 风格 | 说明 | 适用场景 |
|------|------|---------|
| 播客风格 | 双人对话式，Q&A结构 | 知识探索、概念讲解 |
| 朗读风格 | 单人叙述，平稳节奏 | 文章朗读、连载小说 |
| 对话风格 | 多角色交替，情感起伏 | 寓言故事、场景再现 |

## 生成流程

```
1. 文本元素 → 脚本生成（按音频风格组织）
2. 脚本 → TTS语音合成（分段生成）
3. 语音片段 → 拼接 + 静音调整
4. （可选）叠加背景音乐
5. 导出 .mp3
```

## 技术约束

- **采样率**：44100Hz
- **格式**：MP3（128kbps及以上）
- **单段时长**：建议<30分钟，超长内容分段
- **TTS服务**：复用 `format-pipeline/video/video-dsl/scripts/generate_audio.py`（Edge TTS）
- **静音间隔**：段落间插入300-500ms静音，章节间插入1-2s静音
- **BGM混音**：使用ducking（旁白时降低BGM音量至-20dB）

## 失败模式

| 现象 | 处理 |
|------|------|
| TTS合成失败 | 检查edge-tts安装和网络连接；失败段落重试3次 |
| 语音拼接错位 | 检查静音间隔参数；重新拼接 |
| BGM音量过大盖过旁白 | 调整ducking阈值至-20dB |
| 单段>30分钟 | 按章节拆分为多段音频 |
| 中文发音异常 | 检查文本中的多音字/专有名词，必要时用拼音标注 |

## 质量自检

| 检查项 | 标准 |
|--------|------|
| 语音清晰 | 无杂音、无截断、音量均匀 |
| 静音间隔 | 段落间300-500ms，章节间1-2s |
| 风格匹配 | 播客风格有对话感、朗读风格平稳、对话风格有角色区分 |
| BGM混音 | 旁白清晰可辨，BGM不喧宾夺主 |
| 时长 | 与脚本字数匹配（约150字/分钟） |

## 与其他管线的关系

- **视频管线**：音频管线生成的语音可直接作为视频管线的音轨
- **Markdown管线**：Markdown文本是音频脚本的来源
- **口播稿管线**（video-channel）：口播稿是音频脚本的来源之一
- **平台适配**：音频产物交由平台适配层适配到播客平台或视频号

## 脚本

```
audio/
└── scripts/          # 生成脚本（规划中）
    └── generator.py  # TTS生成+拼接+BGM
```

脚本将复用 `format-pipeline/video/video-dsl/scripts/generate_audio.py` 的TTS能力。
