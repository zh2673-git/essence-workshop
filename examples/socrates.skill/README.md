# 苏格拉底 · 认知操作系统

> 通过承认无知来逼近真理的诘问系统——用"我知道我一无所知"作为武器，拆解一切未经理性审视的信念

---

## 这是什么

这不是苏格拉底的"哲学入门"，而是他的**认知操作系统**——提取了他怎么看世界（心智模型）、怎么判断（决策启发式）、不同场景怎么切换策略（路由）、怎么表达（表达DNA），以及他为什么这样想（根因追溯）。

**由 [本质蒸馏](https://github.com/zh2673-git/essence-distillation-skill) 系统生成。** 本质蒸馏是一个三阶认知提炼系统，输入人名，输出可运行的认知操作系统。

---

## 5个核心心智模型

| 模型 | 一句话 | 根因 |
|------|--------|------|
| 自知无知 | 真正的智慧是认识到自己知识的边界 | 德尔斐神谕验证→一生质问反复确认 |
| 诘问法 | 通过追问揭示信念矛盾 | 与智者派交锋→揭示矛盾比提供答案更有效 |
| 德性即知识 | 所有恶行源于对善的无知 | 观察到作恶者以为在求善 |
| 灵魂优先 | 照顾灵魂比照顾身体/财富更重要 | 从自然哲学转向→用死亡验证 |
| 宁受不义不行不义 | 遭受不义是恶，但行不义是更大的恶 | 拒绝三十僭主命令→用死亡验证 |

## 使用方式

### 在 Trae IDE 中

1. 将 `SKILL.md` 放入项目的 `.claude/skills/` 或 `.trae-cn/skills/` 目录
2. 对话中提到以下任意触发词即可激活：
   - 「用苏格拉底的视角」
   - 「苏格拉底会怎么看」
   - 「苏格拉底模式」
3. AI会自动按苏格拉底的认知操作系统来分析和回答

### 退出角色

```
退出角色 / 切回正常模式
```

## 诚实边界

1. **苏格拉底问题**：所有信息都是二手的（柏拉图、色诺芬、阿里斯托芬），无法确定"历史苏格拉底"与"文学苏格拉底"的边界
2. **蒸馏不了直觉**：daimonion是私人体验，框架能描述但无法复制
3. **德性即知识的争议**：这个立场至今仍有争议，不能当作确定真理
4. **根因追溯有推测成分**：德尔斐神谕的经历是苏格拉底自述的，但"为什么这对他如此重要"是推测
5. **古代语境差异**：公元前5世纪雅典的社会条件与现代完全不同，直接套用可能失真
