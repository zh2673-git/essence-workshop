# 场景G：概念寓言化

> **方法论引用**：三阶合一框架定义、思考层 vs 输出层分离原则详见 [../general-logic/references/methodology.md](../general-logic/references/methodology.md)。本文件只描述寓言化视角的认知方法，不重复方法论定义。

> **寓言化的认知定位**：寓言是本质定义的叙事式表达——用一个完整情境让读者自行悟出概念本质。寓言的内核由本质定义锚定，反差/反转对应反常识根因，寓意收尾对应视角升级。"寓言横跨三阶"指思考层——开头呼应本质（阶段1）→中段建立反差（阶段2反常识根因）→结尾视角升级（阶段3），不是产物分节。

---

## 一、寓言化的认知方法

### 1. 寓言作为本质定义的叙事式表达

寓言不是比喻的堆砌，而是用一个完整情境让读者自行悟出概念本质。其内核必须直接来自阶段1的本质定义——不是描述性定义（描述外延），而是揭示本质属性的定义（能解释所有其他属性）。

寓言的叙事节奏与三阶对应（思考层对应关系，不是产物分节）：
- **寓言开头**：用故事场景隐性呈现阶段1的本质定义（不是直接说出本质定义，而是让本质在情境中显现）
- **寓言中段**：通过反差/反转触发阶段2的反常识根因，制造认知冲突
- **寓言结尾**：通过寓意收尾实现阶段3的视角升级，重新定义问题

### 2. 取象方法：选择寓言载体

| 载体类型 | 适用概念 | 示例 |
|---------|---------|------|
| 动物拟人 | 行为类、心理类概念 | 狐狸与葡萄 → 酸葡萄心理 |
| 器物拟人 | 系统/机制类概念 | 老钟表与新秒针 → 技术债务 |
| 自然现象 | 规律/法则类概念 | 两股洋流相遇 → 市场均衡 |
| 抽象拟人 | 元概念/思维类概念 | "犹豫"和"决断"打架 → 决策瘫痪 |

### 3. 寓言结构认知

```
角色A（代表概念主体）
   ↓ 想做X
遇到阻碍Y（代表概念要解决的核心矛盾）
   ↓ 尝试应对
产生后果Z（代表概念的运行结果）
   ↓ 点破
"这个故事讲的，其实是{概念}"
```

冲突必须映射阶段1本质定义展开的反常识根因，而非任意冲突。角色行为必须映射概念的核心要素（由本质定义推导）。**禁止**：没有冲突的纯描述、角色直接说出道理、旁白式说教。

## 二、寓言→概念映射回溯（认知校验）

寓言生成后，必须做映射回溯校验：每个寓言元素能否回溯到阶段1的本质定义。

```
| 寓言元素          | 概念要素           | 回溯到本质定义（内部校验列）             |
|------------------|-------------------|-------------------------------------|
| 守着空松果的松鼠   | 已投入不可回收的资源 | 体现"已投入的资源错误地影响未来决策"中的"已投入" |
| 冬天快来了         | 决策点（时间压力）   | 触发"未来决策"的时间约束              |
| 树洞里有新松果     | 未来收益           | "未来决策"应该考虑的对象              |
| 松鼠不肯搬         | 沉没成本谬误        | "已投入错误地影响未来决策"的行为体现    |
```

**映射铁律（认知校验）**：
1. 若某个寓言元素找不到概念对应物，则该元素是冗余装饰，必须删除
2. 若寓言元素能找到概念对应物但无法回溯到本质定义，则该对应物是表面描述，需重新映射
3. 反差/反转必须对应本质定义展开的反常识根因，而非任意冲突

## 三、迁移问题与视角升级（认知层面）

### 迁移问题（认知校验）

- **问题1·理解检验**：指向寓言中某个具体细节，问"为什么这个细节能体现概念"
  - 禁止问"这个概念是什么意思"（背诵不是理解）
- **问题2·跨域迁移**：给出一个新场景，问"概念在这个场景里怎么体现"
  - 禁止问"你还能在哪里用到这个概念"（太宽泛，无法验证）

### 视角升级（阶段3顿悟触发）

- **目标**：不是总结寓言，而是重新定义问题，给出比阶段1更高的视角
- **形式**：一句话点破"我们以为在解决X问题，其实在解决Y问题"
- **禁止**：用"总而言之""综上所述"做总结代替视角升级（无顿悟）

### 示例

| 概念 | 问题1（理解检验） | 问题2（跨域迁移） | 视角升级 |
|------|-----------------|------------------|---------|
| 沉没成本 | 松鼠守着空松果不肯搬走，这个"不肯"背后是什么心理？ | 如果把故事换成"一个公司继续给亏损项目投钱"，沉没成本体现在哪里？ | 我们以为在算"该不该继续"，其实是在问"你愿不愿意承认过去的选择错了" |
| 缓存 | 驿站为什么要把常用信件放在柜台而不是仓库？ | 如果把故事换成"浏览器把网页存在本地"，缓存的失效策略该怎么设计？ | 我们以为在解决"快慢问题"，其实在解决"用空间换时间的代价博弈" |

## 四、失败模式（认知层面）

| 失败场景 | 认知原因 | 处理动作 |
|---------|---------|---------|
| 概念过于宽泛，无法一句话说本质 | 本质定义未抓到 | 缩小到具体子概念，重新提炼本质定义 |
| 候选定义未通过本质检验 | 描述性定义代替本质定义 | 降级为描述性定义后再重写为本质定义 |
| 寓言没有冲突，变成说教 | 角色行为未映射核心要素 | 强制加入"角色A想做X，但遇到Y阻碍"结构，且冲突必须映射反常识根因 |
| 遮住概念名后读者无法猜到 | 角色行为未映射核心要素 | 检查角色行为是否映射核心要素，补强映射后重写 |
| 寓言元素能找到概念对应物但无法回溯到本质定义 | 元素无本质锚定 | 重新映射，确保每个元素都能回溯到本质定义 |
| 寓意收尾变成了"综上所述"式总结 | 错过顿悟触发点 | 重写为"重新定义问题"的视角升级句 |
| 用描述性定义代替本质定义 | 思考层阶段1错误 | 必须用本质定义（揭示本质属性），并通过本质检验 |

## 五、检查清单（认知层面）

- [ ] 本质定义先行：已提取揭示本质属性的定义（不是描述外延），并通过本质检验
- [ ] 一句话点破：已给出最简洁的本质表达（可独立传播，作为思考层顿悟触发点）
- [ ] 核心要素：已由本质定义展开2-3个关键齿轮
- [ ] 类归属：已由本质定义确定类
- [ ] 反常识根因：已挖掘至少1个反常识根因，作为构思冲突反转点的内部依据
- [ ] 视角升级：已生成视角升级句，作为寓意收尾的内部依据
- [ ] 映射回溯校验：寓言元素与概念要素已对应，且每一项都能回溯到本质定义
- [ ] 间接叙事：寓言让情节自己说话，不直接说教
