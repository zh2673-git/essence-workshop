# 案例检索 · 辅助原子技能

## 法律声明

> 本技能提供的案例检索结果仅供参考，**不构成法律意见**。案例受时间、地域、法院层级、具体事实差异影响，不能直接作为本案裁判依据。

---

## 一、核心概念

### 1.1 案例层级

| 层级 | 效力 | 示例 |
|------|------|------|
| 指导性案例 | 应当参照 | 最高人民法院指导性案例 |
| 公报案例 | 重要参考 | 《最高人民法院公报》案例 |
| 参考性案例 | 参考 | 高级人民法院参考性案例 |
| 一般裁判 | 参考 | 普通判决书 |

### 1.2 类案匹配维度

| 维度 | 说明 |
|------|------|
| 案由 | 相同或相近案由 |
| 法律关系 | 相同法律关系 |
| 争议焦点 | 相同法律问题 |
| 事实特征 | 关键事实相似 |
| 法院层级 | 上级法院案例参考价值更高 |

---

## 二、检索认知原则

> 检索执行（MCP 工具调用、联网搜索、具体数据库访问）由 platform-adapter 负责。本节仅规定认知层面的检索原则。

### 2.1 案例层级优先原则

- 优先检索指导性案例、公报案例，再检索参考性案例与一般裁判；
- 上级法院案例参考价值高于下级法院案例。

### 2.2 类案匹配原则

- 按案由、法律关系、争议焦点、事实特征、法院层级五个维度匹配；
- 相似度评估应明确标注高/中/低，并说明理由。

### 2.3 来源标注原则

- 每个案例必须标注案号、法院、来源与检索时间；
- 模型内部知识给出的案例必须标注 `[待检索]`，不得作为确定结论的依据。

---

## 三、工作流程

### 步骤 1：确定检索目标

- 需要证明什么法律问题？
- 需要哪个层级/地区的案例？

### 步骤 2：构造检索条件

提取案由、法律关系、争议焦点、关键词。

### 步骤 3：执行检索

按分层策略检索。

### 步骤 4：筛选与匹配

按类案匹配维度筛选最相关案例。

### 步骤 5：归纳裁判规则

提炼类案中的裁判要旨。

---

## 四、输入/输出描述

### 输入

- `legal-relationship-typing` 输出的法律关系定性报告；
- `norm-retrieval` 输出的候选规范；
- 或用户直接提供的检索需求。

### 输出

- 检索目标、检索条件（案由/法律关系/争议焦点/关键词）、检索结果（含层级、案号、法院、裁判要旨、相似度、来源）、裁判规则归纳、风险提示。
- 输出格式由 format-pipeline/markdown 处理，本技能不规定具体文档模板。

---

## 五、验证规则

| 检查项 | 通过标准 |
|--------|----------|
| 来源标注 | 每个案例均标注案号、法院、来源 URL |
| 层级区分 | 已区分指导性案例、公报案例、普通裁判 |
| 相似度评估 | 已评估与本案的相似度 |
| 规则提炼 | 已提炼可复用的裁判规则 |

---

## 六、失败模式与降级

| 失败场景 | 处理动作 |
|----------|----------|
| 检索执行不可用或受限 | 转由 platform-adapter 决定降级策略；本技能仅要求所有案例标注来源，内部知识给出的案例标注 `[待检索]` |
| 无高度相似案例 | 扩大检索条件，标注参考价值 |
| 案例层级较低 | 降低参考价值权重 |
| 案例来源无法核实 | 标注 `[待检索]`，不得作为确定结论的依据 |
