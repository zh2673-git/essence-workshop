# 第一阶：是什么 —— 法条适用的本质

## 一句话定义

> 法条适用是**在具体事实与抽象规范之间建立理性连接的推理技术**。
>
> 通用版：覆盖民事、刑事、行政及宪法基本权利场景。

## 坡度1：点出概念

法条适用不是背诵法条，而是在面对一个具体事件时，知道：
- 这件事属于哪个法律领域（民事、刑事、行政、宪法）
- 该按哪个领域的路径找法条
- 找到多个法条时该用哪个
- 法条含义不明时如何解释
- 没有直接法条时如何谨慎填补

## 坡度2：建立联系 —— 8个核心认知框架

| 框架名称 | 一句话定义 | 跨域证据 | 应用场景 | 局限 |
|---------|-----------|---------|---------|------|
| **事实裁剪框架** | 从生活事实中提炼具有规范意义的法律事实 | 民事侵权、刑事定罪、行政违法 | 用户描述具体事件时 | 无法替代证据认定 |
| **法律关系定性框架** | 判断案件属于民事/刑事/行政/宪法基本权利哪个领域 | 邻居打架（民/刑）、处罚纠纷（行政）、言论限制（宪法） | 开始找法条之前 | 竞合案件需分领域并行 |
| **规范检索框架** | 先检索立法法正式法源；行政行为满足三要件时，再检索行政规范性文件 | 民事请求权基础、刑法三阶层、行政合法性依据 | 确定法律关系领域后 | 定性错误会导致第二层检索错位；地方性规范容易遗漏 |
| **涵摄框架** | 将事实逐一对照法条构成要件 | 民事、刑事、行政各领域 | 判断某行为是否满足法条条件 | 需要充分证据 |
| **冲突裁决框架** | 用位阶与优先规则解决规范冲突；须先识别规范性质（法律/法规/规章/行政规范性文件） | 民法与公司法、新法与旧法、行政规范性文件附带审查 | 多个法条都可适用时 | 不能解决事实争议；不直接解决行政规范性文件合法性争议，需经附带审查 |
| **法律解释框架** | 从文义出发，经体系、历史到目的解释 | 知假买假、直系亲属认定 | 法条含义模糊时 | 不能超越文义边界 |
| **抗辩与效力检视框架** | 检查抗辩事由、违法性阻却、行政行为合法性要件、基本权利限制事由等 | 诉讼时效、正当防卫、行政处罚程序违法 | 评估规范适用最终能否得出结论 | 不同领域审查结构不同 |
| **兜底适用框架** | 穷尽具体规则后再启用兜底条款 | 反不正当竞争、一般侵权责任 | 没有具体规则可用时 | 容易被滥用 |

## 坡度3：详细解释

### 框架1：事实裁剪框架

**核心机制**：生活事实 → 法律事实 → 规范对象

**运行方式**：
1. 听取/阅读当事人描述
2. 剥离情绪、猜测、无关细节
3. 保留"主体、行为、客体、结果、时间、地点"等规范要素
4. 用法律语言重新表述

**局限**：事实裁剪建立在证据基础上，不能替代证据调查。

### 框架2：法律关系定性框架

**核心机制**：事实裁剪后，先判断法律关系领域，再决定找法路径。

**运行方式**：
1. 判断主体关系：平等主体 / 管理与被管理 / 国家追诉 / 基本权利限制
2. 判断行为性质：私法行为 / 犯罪行为 / 行政行为 / 国家行为
3. 行政行为识别（决定是否触发规范性文件检索）：
   - 主体：行政机关或法律授权组织
   - 权力：行使行政权力（处罚、许可、征收、强制、审批等）
   - 外部性：对相对人权利义务产生外部影响
   - 三要件同时满足 → 行政行为 → 规范检索进入第二层
4. 确定救济路径：民事诉讼 / 刑事控告 / 行政复议或诉讼 / 宪法救济
5. 确定核心争点：权利义务 / 犯罪成立 / 行政行为合法性 / 合宪性

**领域特征速查**：

| 领域 | 主体关系 | 核心问题 | 第一层检索（正式法源） | 第二层检索（规范性文件） |
|---|---|---|---|---|
| 民事 | 平等主体 | 权利/义务/责任 | 法律、法规、司法解释 | 不自动，按需提示 |
| 刑事 | 国家追诉嫌疑人 | 犯罪成立与刑罚 | 刑法、单行刑法、司法解释 | 不自动 |
| 行政 | 行政机关与相对人 | 行政行为合法性 | 法律、法规、规章、司法解释 | 自动（满足三要件时） |
| 宪法基本权利 | 国家权力与公民 | 基本权利限制与合宪性 | 宪法、法律 | 不自动 |

**局限**：刑民交叉、行民交叉案件需要分领域并行分析；定性错误会导致检索层级错误。

---

### 框架3：规范检索框架

**核心机制**：先检索立法法正式法源，再按条件检索行政规范性文件。

**运行方式**：

**第一层：立法法正式法源（所有案件必检）**

| 领域 | 检索顺位 |
|---|---|
| 民事 | 合同/类合同 → 无因管理 → 物权 → 不当得利 → 侵权 |
| 刑事 | 刑法分则罪名 → 刑法总则 → 单行刑法 → 司法解释 |
| 行政 | 法律 → 行政法规 → 地方性法规 → 部门规章/地方政府规章 → 司法解释 |
| 宪法基本权利 | 宪法 → 法律 → 比例原则与法律保留 |

**第二层：行政规范性文件（按条件触发）**

| 情形 | 是否自动检索 | 说明 |
|---|---|---|
| 行政机关行使职权的案件 | ✅ 自动 | 规范性文件常是行政行为直接依据 |
| 行政机关作为民事主体 | ❌ 不自动 | 按民事案件处理，除非用户要求 |
| 民事/刑事/宪法案件 | ❌ 不自动 | 通常不创设权利义务；如有需要可提示补充 |
| 领域不明或行民交叉 | ⚠️ 提示用户 | "是否需要进一步检索相关行政规范性文件？" |

**关键操作**：
1. 所有案件先走完第一层正式法源检索
2. 满足行政行为三要件时，自动进入第二层检索
3. 其他案件在结论中按需提示是否需要补充检索

**局限**：定性错误会导致第二层检索该启动没启动或不该启动却启动；地方性规范、新出台文件容易遗漏；需要持续更新规范库。

### 框架4：涵摄框架

**一句话**：将事实逐一对照法条构成要件。

**核心机制**：法条 = 构成要件（T）+ 法律效果（R）；事实满足T → R发生

**运行方式**：
1. 分解法条构成要件
2. 为每个要件寻找事实对应
3. 判断每个要件是否成立
4. 综合判断法律效果

**局限**：需要事实清楚、证据充分；不能处理法律漏洞。

### 框架5：冲突裁决框架

**核心机制**：上位法优于下位法；特别法优于一般法；新法优于旧法；行政规范性文件不得与上位法冲突。

**运行方式**：
1. **识别规范性质**：先判断该文件是法律、法规、规章，还是行政规范性文件（红头文件）
2. 判断规范层级是否相同
3. 相同层级时判断是否为一般/特别关系或新旧关系
4. 不同层级时直接适用上位法
5. 新旧一般/特别冲突时依法定程序裁决
6. 对行政规范性文件，必要时启动合法性附带审查

**行政规范性文件的位阶与审查**：
- 行政规范性文件不是《立法法》第二章至第四章规定的正式法律渊源
- 制定机关必须在其法定职权范围内制定，不得与上位法冲突
- 行政诉讼中，原告可依据《行政诉讼法》第53条一并请求审查规范性文件
- 法院认为规范性文件不合法的，不作为认定行政行为合法的依据

**局限**：只解决规范冲突，不解决事实认定和价值权衡；对行政规范性文件的合法性争议，需通过附带审查程序处理。

### 框架6：兜底适用框架

**核心机制**：具体类型化条款 → 例示性兜底 → 概括性兜底 → 一般条款

**运行方式**：
1. 穷尽具体规则
2. 例示性兜底用同类解释规则
3. 概括性兜底最后使用
4. 一般条款作为最后手段

**局限**：容易被滥用，导致裁判恣意。

### 框架7：法律解释框架

**核心机制**：文义是起点，体系是边界，目的是最高准则

**运行方式**：
1. 先确定词句通常含义
2. 再看条文在法律体系中的位置
3. 参考立法历史和目的
4. 做合宪性审查

**局限**：解释不能超越文义可能范围。

### 框架8：抗辩与效力检视框架

**核心机制**：规范适用成立 ≠ 结论最终成立，还需排除抗辩、阻却事由或合法性瑕疵。

**运行方式**：

**A. 民事领域**
1. 检视请求权是否成立
2. 检视是否存在抗辩事由（诉讼时效、同时履行抗辩、不可抗力等）
3. 判断抗辩是否成立
4. 得出最终法律效果

**B. 刑事领域**
1. 检视构成要件该当性
2. 检视违法性阻却事由（正当防卫、紧急避险等）
3. 检视有责性阻却事由（责任年龄、期待可能性等）
4. 得出最终法律效果

**C. 行政领域**
1. 检视主体是否适格
2. 检视权限是否合法
3. 检视程序是否合法
4. 检视内容是否合法
5. 检视依据是否合法（含规范性文件审查）
6. 得出最终法律效果

**D. 宪法基本权利领域**
1. 检视基本权利是否受侵害
2. 检视限制是否有法律依据
3. 检视是否符合比例原则（适当性、必要性、均衡性）
4. 得出最终法律效果

**局限**：不同法律领域的审查结构不同，不能混用。

## 框架关系与运行流程

```
[事实裁剪框架]
       │
       ↓
[法律关系定性框架]
       │
       ↓
[规范检索框架] ──→ [涵摄框架]
       │                   │
       ↓                   ↓
[冲突裁决框架]      [抗辩与效力检视框架]
       │                   │
       ↓                   ↓
[兜底适用框架] ←──→ [法律解释框架]
```

**关键顺序**：
1. 先裁剪事实，再定性法律关系
2. 先定性法律关系，再按领域检索规范
3. 先检索规范，再涵摄构成要件
4. 先涵摄成立，再检查抗辩/效力
5. 规范冲突时启动冲突裁决
6. 穷尽规则时启用兜底适用或法律解释
