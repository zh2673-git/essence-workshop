# 认知引擎 · 视角路由

> **v3.0 认知层**：本层是本质工坊的"大脑"——负责理解事物本质。通用逻辑是框架基座，各视角是实现形式。

## 架构

```
认知引擎（用什么视角理解）
├── general-logic/          # 通用逻辑推理（三阶合一框架基座）
│   └── 定义每阶段必须产出什么
├── yijing-perspective/     # 易经视角（原型式认知）
│   └── 用取象定象实现三阶合一
├── law-application/        # 法条适用（法律场景认知）
│   └── 用规则识别与适用实现三阶合一
└── fable-concept/          # 概念寓言化（叙事式认知）
    └── 用寓言叙事实现三阶合一
```

## 视角选择规则

### 单视角模式
用户指定一个视角 → 通用逻辑框架 + 该视角的实现方法

### 多视角组合模式
用户指定多个视角 → 通用逻辑框架 + 多视角并行/串行

```
组合示例：
  通用逻辑 + 易经 → 用取象定象做本质定义
  通用逻辑 + 法条 → 用规则识别做本质定义
  通用逻辑 + 寓言 → 用叙事做本质定义
  通用逻辑 + 易经 + 法条 → 双视角交叉验证
  通用逻辑 + 易经 + 寓言 → 原型定位+叙事表达
```

### 纯通用逻辑模式
用户不指定视角 → 通用逻辑独立运行（分析式本质定义）

## 蒸馏视角激活（动态视角）

除上述4个内置视角外，`distillation/output/` 中的蒸馏人物skill可作为**动态认知视角**被激活：

| 激活方式 | 示例 | 说明 |
|---------|------|------|
| 指定人物视角 | 「用老子的视角分析XX」「老子会怎么看XX」 | 激活 `distillation/output/laozi.skill/` |
| 指定人物视角 | 「用达尔文的视角看XX」「达尔文模式」 | 激活 `distillation/output/darwin.skill/` |
| 切换人物 | 「切换到孔子」「用王阳明的角度」 | 激活对应蒸馏skill |

**激活规则**：
1. 蒸馏人物skill的视角来源标记为 `distilled:{person}`（如 `distilled:laozi`）
2. 激活时，蒸馏skill的「核心心智模型」作为本质定义方法，「决策启发式」作为方法步骤来源
3. 蒸馏视角可与内置视角组合：如「用老子+易经的视角分析XX」
4. 蒸馏视角的认知产物格式与内置视角一致，交给格式管线层处理

**与内置视角的关系**：
- 内置视角（通用逻辑/易经/法条/寓言）= 可复用的认知方法
- 蒸馏视角（老子/达尔文/孔子...）= 具体的认知实例
- 蒸馏（distillation）元场景负责生产新的蒸馏视角，产出自动可被本层激活

## 认知产物（输出给格式管线层）

认知引擎的产物是**结构化认知数据**，不绑定任何格式（本质定义 + 因果链 + 方法步骤 + 视角升级等）。认知产物 YAML 格式详见 [general-logic/SKILL.md](general-logic/SKILL.md)。

此产物可被格式管线层（Markdown/HTML/视频等）和平台适配层（公众号/浏览器等）使用。

## 扩展新视角

新增认知视角只需在 `cognitive-engine/` 下创建新子目录，包含SKILL.md和references/，实现三阶合一的三个阶段即可。蒸馏（distillation）产出的新认知skill自动放入此层。
