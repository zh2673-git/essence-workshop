# 场景A：通用知识探索（正向推导）

> **v2 方法论锚点**：本场景采用 [methodology.md v2](references/methodology.md) —— 三阶合一、本质定义先行、顿悟触发机制、类-属性-方法-路由由本质定义锚定。

> **v2.1 场景特例说明**：场景A是**唯一允许"三阶段"作为产物分节**的场景——因为知识解构本身天然就是"是什么→为什么→怎么做"的认知路径。其他场景（C/D/E/E3/F等）的思考层走v2，但输出层按场景固有结构组织，**禁止把思考结构硬塞到产物分节**。详见 methodology.md 第九章"思考层 vs 输出层分离原则"。

## 执行流程（三阶合一）

```
第1步：阶段1（是什么·点出本质）—— 顿悟触发①
    ├── 抛本质定义（不是描述性定义，是揭底）
    │   ├── 描述性定义（禁用）：属加种差，描述外延
    │   └── 本质定义（强制）：揭示本质属性，能解释所有其他属性
    ├── 本质检验（必须通过）
    │   └── 候选定义能否解释这个东西的所有其他属性？能→保留；不能→降级重写
    ├── 类归属（由本质定义确定）
    ├── 关键属性表格（属性必须是本质的展开，标记✅需深入/❌已清晰）
    └── 一句话点破（顿悟触发①：最简洁的本质表达，可独立传播）
           ↓
第2步：阶段2（为什么·建立因果）—— 顿悟触发②
    ├── 因果链表格（因→果→逻辑→反差）
    │   └── 因果链必须由阶段1的本质推导，不是新引入的前提
    ├── 反常识根因（顿悟触发②：至少1个反直觉点制造认知冲突）
    ├── 结构性比喻（可选，比喻内部结构与本质同构）
    └── 依据支撑 + 质疑验证
           ↓
第3步：阶段3（怎么做·展开方法·视角升级）—— 顿悟触发③
    ├── 执行步骤表格（步骤号 + 资源需求 + 验证方法 + 回溯到本质）
    │   └── 每个步骤必须能回溯到本质，不是任意操作清单
    ├── 每个步骤详细说明
    └── 视角升级（顿悟触发③：结尾重新定义问题，给出比阶段1更高的视角）
           ↓
第4步：递归深入（v2 · 只重做阶段1）
    └── 对每个✅属性，递归执行阶段1（重新找子概念的本质定义）
        └── 不再重复阶段2/3，避免无限展开
```

## 输出文档结构

场景A的文档保存在**Obsidian Vault/通用笔记**下：

```
C:/Users/LX/Documents/Obsidian Vault/
└── 通用笔记/{分类}/
    ├── {概念名}-三阶解构.md              # 主文档（阶段1→阶段2→阶段3）
    ├── 01-{子概念A}-本质定义.md          # 第1层递归深入（只做阶段1）
    ├── 02-{子概念B}-本质定义.md          # 第1层递归深入
    │   ├── 02-01-{子概念B1}-本质定义.md  # 第2层递归深入
    │   └── 02-02-{子概念B2}-本质定义.md  # 第2层递归深入
    └── 99-知识图谱.md                    # 知识关系汇总
```

**编号规则**：
- 主文档：无编号（或根据分类前缀）
- 第1层子概念：`01-`, `02-`, `03-`...（两位数）
- 第2层子概念：`01-01-`, `01-02-`...（四位数，体现父子关系）
- 第3层子概念：`01-01-01-`...（六位数）
- 汇总文档：`99-`

## 文档命名规范

| 文档类型 | 命名格式 | 内容 |
|---------|---------|------|
| **主文档** | `{概念名}-三阶解构.md` | 完整三阶合一：本质定义+本质检验+类归属+一句话点破+因果链+反常识根因+视角升级 |
| **递归深入** | `{序号}-{子概念}-本质定义.md` | 仅阶段1：本质定义+本质检验+类归属+一句话点破（不重复阶段2/3） |
| **知识图谱** | `99-知识图谱.md` | 所有概念的关系图谱，标注本质间的从属关系 |

## 失败模式编码

> **设计依据**：借鉴 Darwin Skill dim3（失败模式编码）+ dim4（检查点显性标记）。只写正向流程的Skill是不完整的。

| 步骤 | 失败场景 | 处理动作 |
|------|---------|---------|
| 阶段1 是什么 | 本质定义无法通过本质检验（不能解释所有其他属性） | 降级为描述性定义，重新寻找本质属性 |
| 阶段1 是什么 | 概念过于宽泛，无法一句话点破 | 缩小范围到具体子概念，重新定义 |
| 阶段1 是什么 | 找不到✅属性（所有属性都是❌） | 检查是否拆解粒度不够，尝试从不同维度拆解 |
| 阶段1 是什么 | 一句话点破不可独立传播 | 重写，要求脱离上下文也能让人理解 |
| 阶段2 为什么 | 因果链断裂，无法建立逻辑联系 | 标注"逻辑缺口"，保留疑问而非强行连接 |
| 阶段2 为什么 | 找不到反常识根因 | 检查因果链是否过于顺向，尝试从反面切入 |
| 阶段2 为什么 | 多条因果链互相矛盾 | 保留矛盾，分领域记录，不强求统一 |
| 阶段3 怎么做 | 步骤无法回溯到本质 | 删除该步骤或重新设计，标注"游离步骤" |
| 阶段3 怎么做 | 视角升级退化为总结 | 重写，要求给出比阶段1更高的视角，禁用"综上所述"等套话 |
| 阶段3 怎么做 | 执行步骤无法验证 | 补充验证方法，若无法验证则标注"待验证" |
| 阶段4 递归深入 | 递归超过3层仍未终止 | 强制终止，在文档中标注"深入不足" |
| 阶段4 递归深入 | 子概念与父概念重复 | 跳过重复概念，标注"与父概念重叠" |
| 阶段4 递归深入 | 误重复阶段2/3 | 强制只做阶段1，标注"v2递归纪律" |

## 检查点

- 🔴 **CHECKPOINT · 阶段1完成后**：确认本质定义已通过本质检验、一句话点破可独立传播，若未通过则降级重做
- 🔴 **CHECKPOINT · 阶段2完成后**：确认因果链由本质推导、反常识根因已找出，若逻辑链断裂则标注缺口
- 🟡 **AUTO-CHECK · 阶段3完成后**：输出三阶分析结果摘要（含3个顿悟触发点），自动继续递归深入（用户可随时打断）
- 🔴 **CHECKPOINT · 每层递归完成后**：展示当前层级的本质定义，确认方向正确
