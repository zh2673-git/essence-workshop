# 易经视角与双引擎方法论对接规范

> **文档定位**：本文件是"本质工坊"中"易经视角"子Skill的核心参考文档，定义易经视角如何作为"双引擎"之一深度融合到方法论。
>
> **对接原则**：不破坏原有流程，增加可选的"易经路径"。所有术语（本质定义先行、三阶合一、顿悟触发、类-属性-方法-路由）保持原义，易经视角作为并行/串行/降级三种模式接入。
>
> **方法论锚点**：三阶合一（阶段1是什么·点出本质 → 阶段2为什么·建立因果 → 阶段3怎么做·展开方法·视角升级）× 本质定义先行 × 顿悟触发机制 × 类-属性-方法-路由（由本质定义锚定）。

## 一、双引擎概述

### 方法论原貌

> 方法论完整定义详见 [../../general-logic/references/methodology.md](../../general-logic/references/methodology.md)（三阶合一、本质定义先行、顿悟触发、类-属性-方法-路由）。本文件只描述易经视角与双引擎的映射方法，不重复定义方法论。

### 双引擎升级
- **新增"原型式本质定义"引擎**：与原有"分析式本质定义"形成双引擎
- **双引擎关系**：分析式（原有）走属性分析推导本质；原型式（易经）走取象匹配64种原型模式
- **升级目标**：让本质工坊具备"直觉判断+模式识别"能力，覆盖复杂模糊场景

### 升级原则
1. **不破坏原有流程**：所有产物（本质定义、本质检验、类归属、属性表格、因果链、执行步骤）保留
2. **增加可选的"易经路径"**：每个环节都可选择走分析式、原型式或双引擎协同
3. **术语严格对接**：易经映射项必须能填入原有产物结构，不另起炉灶
4. **回溯原则不变**：易经路径下的每一层（类/属性/方法/路由）仍必须能回溯到本质定义（此处为卦象）

## 二、双引擎机制

### 引擎A：分析式本质定义（原有）
- **方法**：通过属性分析推导本质
- **定义性质**：揭底式（揭示本质属性，能解释所有其他属性），禁用描述性定义（属加种差，描述外延）
- **适用**：可拆解分析的概念/系统/项目
- **产物**：本质定义 + 本质检验 + 类归属 + 关键属性表格 + 一句话点破
- **失败模式**：本质定义无法通过本质检验 → 降级为描述性定义，重新寻找本质属性

### 引擎B：原型式本质定义（易经）
- **方法**：通过取象匹配64种原型模式（卦象），以原型定本质
- **定义性质**：原型匹配式（以卦象作为事物的本质原型，卦德即本质属性）
- **适用**：复杂现象/模糊问题/需要直觉判断的场景
- **产物**：卦象 + 内外卦属性 + 卦辞/彖传/大象传 + 一句话点破（用卦辞或大象传）
- **失败模式**：取象不确定/生搬硬套 → 降级回引擎A

### 双引擎协同

| 模式 | 流程 | 适用场景 | 产物关系 |
|------|------|---------|---------|
| **并行模式** | 两个引擎同时运行，结果相互验证 | 需要交叉验证本质判断的准确度 | 分析式本质定义 ⇄ 卦象原型，校验是否指向同一本质 |
| **串行模式** | 先用引擎B快速定位原型，再用引擎A深入分析 | 复杂模糊问题，先定方向再细化 | 卦象 → 锚定类归属 → 引擎A在该原型下展开属性 |
| **降级模式** | 当引擎B取象不确定时，回退到引擎A | 取象生硬、卦象无法解释关键特征时 | 仅保留分析式产物，标注"易经路径降级" |

> **协同纪律**：双引擎不是简单叠加。若两引擎结果不一致，必须分析差异原因并明确选择哪个引擎的结果或如何融合，禁止"双产物并存却不校验"。

## 三、三阶合一的易经映射

### 阶段1：是什么·点出本质

| 方法论原版 | 易经映射 | 对接说明 |
|--------|---------|---------|
| 本质定义（揭底式） | 定象（匹配64卦原型） | 卦象即原型式本质定义，卦德即揭底的本质属性 |
| 本质检验 | 卦象能否解释事物的所有关键特征 | 候选卦象必须能解释该事物的所有其他属性，否则降级重取 |
| 类归属 | 内外卦归属（8种基本象） | 由卦象的内外卦确定事物的基本类属 |
| 关键属性表格 | 卦德 + 卦象 + 卦辞 + 彖传 + 大象传 | 每项属性标记✅需深入/❌已清晰，与方法论一致 |
| 一句话点破（顿悟触发①） | 用卦辞/大象传点破本质 | 最简洁的本质表达，可独立传播 |

### 阶段2：为什么·建立因果

| 方法论原版 | 易经映射 | 对接说明 |
|--------|---------|---------|
| 因果链表格（因→果→逻辑→反差） | 内外卦组合逻辑（根×境=属性） | 卦象的内外卦组合本身就是因果结构 |
| 反常识根因（顿悟触发②） | 卦象中蕴含的反直觉智慧 | 至少1个反直觉点制造认知冲突 |
| 结构性比喻（可选） | 卦象本身就是最高级的结构性比喻 | 比喻内部结构与本质同构 |
| 依据支撑 | 卦辞/彖传/象传原文 | 因果链必须由阶段1的卦象推导，不是新引入的前提 |

### 阶段3：怎么做·展开方法·视角升级

| 方法论原版 | 易经映射 | 对接说明 |
|--------|---------|---------|
| 执行步骤表格（步骤号+资源需求+验证方法+回溯到本质） | 爻变推演 + 变卦路径 | 每个步骤必须能回溯到本卦本质，不是任意操作清单 |
| 每个步骤详细说明 | 每个变卦回溯到本卦 | 变卦是方法的展开，本卦是本质的锚点 |
| 视角升级（顿悟触发③） | 变卦/错卦/综卦揭示的动态视角 | 结尾重新定义问题，给出比阶段1更高的视角，禁用"综上所述"等套话 |

## 四、顿悟触发机制的增强

### 顿悟触发①（阶段1·一句话点破）
- **原版**：一句话点破（最简洁的本质表达，可独立传播）
- **增强**：用卦辞/大象传点破
  - 示例：乾卦大象传"天行健，君子以自强不息"
  - 示例：坤卦大象传"地势坤，君子以厚德载物"
- **协同方式**：分析式一句话 + 易经一句话，双重点破。两者必须指向同一本质，否则触发差异分析

### 顿悟触发②（阶段2·反常识根因）
- **原版**：反常识根因（至少1个反直觉点制造认知冲突）
- **增强**：卦象组合的反直觉智慧
- **示例**：
  - **泰卦（地天泰）**——地在天上看似反常，却是通达（天地交而万物通）
  - **否卦（天地否）**——天在地上看似正常，却是不通（天地不交而万物不通）
  - **既济（水火既济）**——水在火上看似危险，却是成功；**未济（火水未济）**——火在水上看似无害，却是未成
- **协同方式**：分析式反常识根因 + 卦象反直觉智慧，双重制造认知冲突

### 顿悟触发③（阶段3·视角升级）
- **原版**：视角升级（结尾重新定义问题，给出比阶段1更高的视角）
- **增强**：变卦揭示的动态视角
- **示例**：
  - 乾卦上九"亢龙有悔" → 物极必反，用动态眼光看"成功"
  - 剥卦（山地剥）极变复卦（地雷复）→ 否极泰来，用周期眼光看"衰败"
  - 震卦（震为雷）"瀋雷震，君子以恐惧修省" → 用警醒眼光看"动荡"
- **协同方式**：分析式视角升级 + 变卦动态视角，双重重新定义问题

## 五、类-属性-方法-路由的易经锚定

> **蒸馏公式**：本质定义 → 类归属 → 属性展开 → 方法体现 → 路由适配 → 表达DNA接口（每一层必须能回溯到本质定义）

```
本质定义 = 分析式本质定义 OR 原型式本质定义（卦象）
    ↓
类归属 = 分析式分类 OR 内外卦归属（8种基本象：乾/坤/震/巽/坎/离/艮/兑）
    ↓
属性展开 = 分析式属性表格 OR 卦德+卦辞+彖传+大象传
    ↓
方法体现 = 分析式执行步骤 OR 爻变推演+变卦路径
    ↓
路由适配 = 场景→卦象映射表（新增）+ 原有场景路由表
    ↓
表达DNA = 原有风格 + 象辞式风格（可选）
```

### 回溯原则
- 卦象（本质定义）→ 内外卦（类归属）：类属必须由卦象确定
- 卦象 → 卦德/卦辞/彖传/大象传（属性展开）：属性必须由卦象展开
- 卦象 → 爻变/变卦（方法体现）：方法步骤必须能回溯到本卦
- 卦象 → 场景映射（路由适配）：路由必须能解释为何此场景匹配此卦

## 七、双引擎的质量检查

### 取象质量检查
- [ ] 取象是否基于事物特征（非生搬硬套）
- [ ] 内外卦判定是否有清晰理由（内卦=事物内核/根，外卦=事物表象/境）
- [ ] 卦象能否解释事物的所有关键特征（本质检验通过）
- [ ] 取象是否考虑了事物的动态变化（是否需要变卦视角）

### 双引擎一致性检查
- [ ] 分析式本质定义与原型式本质定义是否指向同一本质
- [ ] 若不一致，是否分析了差异原因
- [ ] 最终选择哪个引擎的结果，还是两者融合（必须明确，禁止双产物并存不校验）

### 检查清单（易经路径下仍必须满足）
- [ ] 阶段1：已给出本质定义（卦象作为原型式本质定义也算）
- [ ] 阶段1：本质检验已通过（卦象能解释所有其他属性）
- [ ] 阶段1：类归属已确定（内外卦归属）
- [ ] 阶段1：一句话点破已给出（卦辞/大象传，可独立传播）
- [ ] 阶段2：因果链已建立（内外卦组合逻辑）
- [ ] 阶段2：因果链由本质推导（由卦象推导，不是新引入的前提）
- [ ] 阶段2：反常识根因已找出（卦象反直觉智慧，至少1个反直觉点）
- [ ] 阶段3：执行步骤已设计（爻变推演，含资源需求+验证方法）
- [ ] 阶段3：每个步骤能回溯到本质（每个变卦回溯到本卦）
- [ ] 阶段3：视角升级已给出（变卦/错卦/综卦动态视角，非总结）

### 检查点
- 🔴 **CHECKPOINT · 阶段1完成后**：确认卦象已通过本质检验、一句话点破（卦辞/大象传）可独立传播，若未通过则降级重取
- 🔴 **CHECKPOINT · 阶段2完成后**：确认内外卦因果链由卦象推导、反常识根因已找出，若逻辑链断裂则标注缺口
- 🟡 **AUTO-CHECK · 阶段3完成后**：输出三阶分析结果摘要（含3个顿悟触发点：卦辞点破+卦象反直觉+变卦视角），自动继续递归深入（用户可随时打断）
- 🔴 **CHECKPOINT · 双引擎一致性**：若采用双引擎，必须校验两引擎结果是否指向同一本质

## 八、双引擎的适用边界

### 适合用引擎B（易经）的场景
- **复杂模糊问题**：难以直接分析、属性不清晰的概念
- **需要直觉判断的场景**：经验型决策、模式识别型问题
- **需要动态视角的趋势分析**：事物发展阶段、周期变化、转折点判断
- **需要换位思考的人际问题**：组织关系、上下级博弈、利益平衡
- **传统中国文化相关的主题**：中医、兵法、管理哲学、传统智慧现代化

### 不适合用引擎B的场景
- **纯技术问题**：如代码调试、系统配置、算法实现
- **精确计算问题**：数学证明、物理计算、财务核算
- **有明确标准答案的问题**：事实查询、规则判定、合规检查
- **需要数据支撑的定量分析**：统计分析、A/B测试、数据建模

### 降级触发条件
当出现以下情况时，引擎B应降级回引擎A：
1. 取象生搬硬套，卦象无法解释事物的关键特征
2. 内外卦判定理由不清晰，存在多种卦象均可匹配
3. 卦辞/大象传与事物本质明显不符
4. 用户明确要求纯分析式结论，不需要原型式解读

> **降级纪律**：降级不是失败，而是双引擎的自我保护机制。降级时在产物中标注"易经路径降级→分析式"，保留降级原因记录，便于后续优化取象能力。
