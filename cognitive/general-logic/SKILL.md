# 通用逻辑 · 三阶合一框架基座

> **本 Skill 是认知引擎的框架层**：定义三个阶段各自"必须产出什么"和"如何检验"。
> 各视角（易经/法条/寓言）是"实现层"，提供"每阶段具体怎么做"。框架+实现，无冲突。

---

## 三阶合一框架定义

```
阶段1（是什么·点出本质）
  ─────────────────────────────
  必须产出：
    1. 本质定义（揭底式：通过属性分析推导，"X的本质是____"）
    2. 本质检验（能解释所有其他属性吗？不能→重做）
    3. 类归属（由本质定义锚定，不靠隐喻硬接）
    4. 关键属性展开（由本质推导，标记✅❌）
    5. 一句话点破（顿悟触发①，可独立传播）
  检验标准：本质定义能否解释所有已观察属性

阶段2（为什么·建立因果）
  ─────────────────────────────
  必须产出：
    1. 因果链（起点必须回溯到阶段1本质定义）
    2. 反常识根因（顿悟触发②，至少1个反直觉点）
    3. 技术/方法选型与决策（由本质推导）
    4. 结构性比喻（可选，比喻内部结构与本质同构）
  检验标准：因果链起点能否回溯到本质定义

阶段3（怎么做·展开方法·视角升级）
  ─────────────────────────────
  必须产出：
    1. 方法步骤（每步必须回溯本质，标记依据）
    2. 路由/情境化适配（本质的情境化体现）
    3. 视角升级收尾（顿悟触发③，重新定义问题，非总结）
  检验标准：每个步骤能否回溯到本质定义
```

---

## 类-属性-方法-路由（由本质定义锚定）

```
本质定义（阶段1产出）
    ↓ 锚定
类归属（属于什么类别，由本质推导）
    ↓ 展开
属性（关键特征，由本质推导，标记✅❌）
    ↓ 体现
方法（动作/步骤，回溯本质）
    ↓ 适配
路由（情境化，本质的情境化体现）

回溯检验：每一层必须能回溯到本质定义。
不能回溯 = 本质定义不完整 或 该层与本质无关（应删除）。
```

## 顿悟触发机制（通用逻辑基线）

| 触发点 | 通用逻辑基线 | 视角增强（叠加时） |
|--------|-------------|------------------|
| ①阶段1 | 一句话点破 | +视角特有的点破形式 |
| ②阶段2 | 反常识根因 | +视角特有的反直觉点 |
| ③阶段3 | 视角升级 | +视角特有的升级维度 |

**纯通用逻辑模式**：仅使用基线三件套。
**组合模式**：基线 + 视角增强，二者协同而非替代。

## 本质检验规则

本质定义必须满足以下检验，否则重做：

1. **解释力检验**：本质定义能否解释所有已观察到的属性？不能→本质未抓到
2. **边界检验**：本质定义能否界定X的边界？什么属于X，什么不属于？
3. **预测力检验**：从本质定义能否预测X的未观察属性/未来行为？
4. **回溯检验**：每个属性/方法/路由能否回溯到本质定义？

**揭底式本质定义示例**：
- 描述性定义（❌不合格）："RAG是一种结合检索和生成的技术"
- 揭底式本质定义（✅合格）："RAG的本质是动态知识注入机制——通过检索外部知识，让模型的生成不再依赖静态参数，而是按需调用最新、可验证的知识源"

## 与视角的协同模式

### 模式1：纯通用逻辑（不指定视角）
```
阶段1 → 分析式本质定义（揭底式）
阶段2 → 分析式因果推导
阶段3 → 分析式步骤推导 + 视角升级
```

### 模式2：通用逻辑 + 单视角
```
阶段1 → 通用逻辑框架 + 视角实现方法
  例：通用逻辑（必须产出本质定义） + 易经（用取象定象产出本质定义）
阶段2 → 通用逻辑框架 + 视角实现方法
  例：通用逻辑（必须产出因果链） + 易经（用内外卦属性拆解产出因果链）
阶段3 → 通用逻辑框架 + 视角实现方法
  例：通用逻辑（必须产出方法步骤） + 易经（用变卦推演产出方法步骤）
```

### 模式3：通用逻辑 + 多视角
```
阶段1 → 通用逻辑框架 + 视角A实现 + 视角B实现（并行验证/串行定位）
  例：通用逻辑 + 易经（原型定位） + 法条（规则识别）→ 双视角交叉验证本质
阶段2/3 → 同理，多视角协同产出
```

## 输出：认知产物

通用逻辑独立运行时，输出以下结构化数据（与 cognitive/SKILL.md 定义的认知产物格式一致）：

```yaml
认知产物:
  本质定义: "（揭底式，通过本质检验）"
  一句话点破: "（顿悟触发①）"
  类归属: "（由本质定义锚定）"
  属性:
    - 属性1: "值1（✅已验证/❌待验证）"
    - 属性2: "值2"
  因果链:
    - 因: "内在因素（回溯到本质）"
      果: "外在表现"
      逻辑: "因果关系"
  反常识根因: "（顿悟触发②）"
  方法步骤:
    - 步骤1: "行动1"
      依据: "回溯到本质定义"
    - 步骤2: "行动2"
      依据: "回溯到本质定义"
  视角升级: "（顿悟触发③，重新定义问题）"
  视角来源: "general-logic"
```

此产物交给内容框架层（content-framework）组织为结构化内容，再经形式层（format）转换格式，最终由工作流层（workflow）编排交付。

## 与其他视角的关系

| 视角 | 实现方法 | 本质定义方式 | 与通用逻辑关系 |
|------|---------|-------------|--------------|
| 通用逻辑（本Skill） | 分析式推导 | 揭底式定义 | 框架基座 |
| 易经视角 | 取象定象 | 卦象式定义（原型式） | 实现层·引擎B |
| 法条适用 | 规则识别与适用 | 法理式定义 | 实现层·法律场景 |
| 概念寓言化 | 叙事 | 故事式定义 | 实现层·叙事式 |

**关键**：通用逻辑定义"必须产出什么"，各视角提供"具体怎么产出"。二者是框架+实现关系，不冲突。
