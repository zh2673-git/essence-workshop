# 法律分析 · 场景编排器

> **职责**：本场景只负责编排调用顺序（调用 cognitive-engine/law-application 的原子技能），不直接实现认知分析。认知方法由 cognitive-engine/law-application 定义，输出格式由 format-pipeline 处理。

## 法律声明

> 本场景输出的是结构化法律分析草稿，**不构成法律意见，不替代执业律师**。报告中的法条引用、案例引用、解释结论均需通过官方法律数据库复核确认。重大决策前请咨询执业法律专业人员。

---

## 一、编排定位

本场景通过调度以下认知引擎原子技能完成端到端任务（原子技能位于 [../../cognitive-engine/law-application/skills/](../../cognitive-engine/law-application/skills/)）：

```
fact-trimming（事实裁剪）
  → legal-relationship-typing（法律关系定性）
  → norm-retrieval（规范检索）
  → case-retrieval（案例检索）
  → subsumption（涵摄）
  → evidence-discovery（证据发现）
  → legal-interpretation（法律解释）
  → conflict-resolution（冲突裁决）
  → defense-and-validity-review（抗辩与效力检视）
       ↓ 若用户已有证据 → evidence-evaluation（证据评估）
  → legal-consequence-calculation（法律后果量化）
  → fallback-application（兜底适用）
```

## 二、质量检查点（QC）

| 检查点 | 位置 | 检查内容 | 失败动作 |
|--------|------|----------|----------|
| QC1 | 事实裁剪后 | 要素是否完整、是否遗漏关键事实 | 回溯至用户，要求补充 |
| QC2 | 法律关系定性后 | 领域判断是否明确、行政行为三要件是否检视 | 回溯至事实裁剪 |
| QC3 | 规范检索后 | 候选规范是否覆盖争议焦点、来源是否标注 | 回溯至法律关系定性 |
| QC4 | 涵摄后 | 要件是否满足、中项是否明确、逻辑是否有效 | 回溯至规范检索或法律解释 |
| QC5 | 证据发现后 | 取证方案是否覆盖全部待证要件 | 回溯至 subsumption 补全 |
| QC6 | 冲突裁决后 | 解决方案是否一致、是否排除合理方案 | 回溯至涵摄 |
| QC7 | 抗辩检视后 | 抗辩是否影响结论、证据是否审查 | 回溯至涵摄或证据评估 |
| QC8 | 兜底适用后 | 是否确已穷尽具体规则 | 回溯至规范检索 |
| QC9 | 终稿前 | 报告完整性、置信度标注、免责声明 | 补充缺失内容 |

## 三、场景路由

| 场景 | 特征 | 阶段特殊处理 |
|------|------|--------------|
| 民事纠纷 | 平等主体、权利义务争议 | 启用九阶请求权检视 |
| 刑事案件 | 涉嫌犯罪、国家追诉 | 启用罪名检索、量刑计算 |
| 行政案件 | 行政行为三要件满足 | 自动检索行政规范性文件 |
| 宪法基本权利 | 涉及基本权利限制 | 启用比例原则、法律保留审查 |
| 复合型案件 | 刑民/行民交叉 | 并行多领域分析 |

## 四、标准调用链

```
输入：用户事件描述
  │
  ▼
S1  fact-trimming（事实裁剪）
  ↓ QC1
  ├─ 要素完整 → 继续
  └─ 要素缺失 → 要求用户补充
  ▼
S2  legal-relationship-typing（法律关系定性）
  ↓ QC2
  ├─ 领域明确 → 继续
  └─ 领域不明/竞合 → 标注并行路径
  ▼
S3  norm-retrieval（规范检索）
  ↓ QC3
  ├─ 规范覆盖完整 → 继续
  └─ 规范缺失 → 回溯或提示
  ▼
S4  case-retrieval（案例检索）
  ▼
S5  subsumption（涵摄）
  ↓ QC4
  ├─ 逻辑有效 → 继续
  └─ 逻辑存疑 / 要件缺失 → 回溯至 S3 或要求补充事实
  ▼
S5b evidence-discovery（证据发现）
  ↓ QC5
  ├─ 取证方案覆盖完整 → 继续
  └─ 覆盖不完整 → 回溯至 S5 补全要件
  ▼
S6a legal-interpretation（法律解释）
  ↓ 若无非明确规范则输出"本阶段无需法律解释"
  ▼
S6b conflict-resolution（冲突裁决）
  ↓ QC6
  ├─ 存在规范冲突 → 输出裁决结论
  └─ 无冲突 → 输出"本阶段无需冲突裁决"
  ▼
S7  defense-and-validity-review（抗辩与效力检视）
  │   ├─ 有证据 → evidence-evaluation（对照 S5b 取证方案评估）
  │   └─ 无证据 / 证据不足 → 基于 S5b 取证方案给出取证建议
  ↓ QC7
  ├─ 抗辩成立 → 调整结论，必要时回溯
  └─ 抗辩不成立 → 继续
  ▼
S8  legal-consequence-calculation（法律后果量化）
  ↓ 若无法量化则输出"本阶段无法量化，仅作定性分析"
  ▼
S9  fallback-application（兜底适用）
  ↓ QC8
  ├─ 已找到具体规则 → 输出"本阶段无需兜底适用"
  └─ 穷尽具体规则 → 输出兜底方案
  ▼
输出：结构化法条适用分析报告（格式由 format-pipeline/markdown 处理）
```

## 五、条件分支与回溯机制

### 5.1 信息不足分支

```
if 用户描述信息不足以完成事实裁剪:
    输出已提取要素 + 关键待补充问题
    暂停流程，等待用户补充
    补充后从 S1 重新开始
```

### 5.2 行政行为分支

```
if legal-relationship-typing 判断为行政行为:
    norm-retrieval 自动进入第二层行政规范性文件检索
else:
    仅在用户明确要求时检索行政规范性文件
```

### 5.3 法律解释分支

```
默认调用 legal-interpretation
if 存在需要解释的非明确规范:
    输出解释对象、方法、结论
else:
    输出"本阶段无需法律解释"
```

### 5.4 冲突裁决分支

```
默认调用 conflict-resolution
if 存在多个可适用规范:
    输出冲突类型、解决规则、结论
else:
    输出"本阶段无需冲突裁决"
```

### 5.5 抗辩成立分支

```
if defense-and-validity-review 发现抗辩成立:
    调整或推翻先前结论
    若涉及新的事实，回溯至 fact-trimming
```

### 5.6 兜底适用分支

```
默认调用 fallback-application
if 无具体规则可涵摄:
    输出穷尽具体规则说明与兜底方案
else:
    输出"本阶段无需兜底适用"
```

## 六、输入/输出

### 输入

- 用户提交的事件描述、问题或案件材料。

### 输出

输出文档格式由 [format-pipeline/markdown](../../format-pipeline/markdown/SKILL.md) 处理。文档结构建议如下：

```markdown
# 法条适用分析报告

## 一、案件摘要
[1-3 句话概括]

## 二、事实裁剪（S1）
[结构化法律要素清单]

## 三、法律关系定性（S2）
- 主领域：[民事/刑事/行政/宪法]
- 子类型：[具体类型]
- 是否行政行为：[是/否]
- 救济路径：[路径]

## 四、规范检索（S3）
[候选规范清单、来源、效力状态]

## 五、案例检索（S4）
[类案清单与裁判规则；无相关类案时标注"未发现典型类案"]

## 六、涵摄分析（S5）
[P-F-C 三段论链条]

## 七、证据发现（S5b）
[待证要件 → 取证材料 → 获取主体/渠道 → 证明力等级 → 优先级建议]

## 八、法律解释（S6a）
[解释对象、方法、结论；无需要解释的要件时标注"本阶段无需法律解释"]

## 九、冲突裁决（S6b）
[冲突类型、解决规则、结论；无冲突时标注"本阶段无需冲突裁决"]

## 十、抗辩与效力检视（S7）
[抗辩清单、审查结论、对原结论的影响]

## 十一、法律后果量化（S8）
[计算过程与结果区间；无法量化时标注"本阶段无法量化，仅作定性分析"]

## 十二、兜底适用（S9）
[穷尽具体规则说明、兜底方案；已找到具体规则时标注"本阶段无需兜底适用"]

## 十三、综合结论
- **核心结论**：[结论]
- **置信度**：[高/中/低]
- **主要风险**：[风险]
- **建议行动**：[建议]

## 十四、免责声明
> 本报告仅供参考，不构成法律意见...
```

## 七、验证规则

| 检查项 | 通过标准 |
|--------|----------|
| 流程完整 | 已按标准调用链执行 |
| QC 通过 | 各检查点均通过或已标注回溯 |
| 来源标注 | 法条、案例引用均已标注来源 |
| 置信度标注 | 不确定结论已标注置信度 |
| 免责声明 | 终稿包含免责声明 |

## 八、失败模式与降级

| 失败场景 | 处理动作 |
|----------|----------|
| 用户输入完全无法理解 | 输出澄清问题，要求重新描述 |
| 信息严重不足 | 输出待补充清单，暂停流程 |
| MCP 与联网搜索均不可用 | 使用模型内部知识给出框架，全部法条标注 `[待校验]` |
| 全流程后仍无法得出结论 | 明确标注"无法形成确定性结论"，建议咨询执业律师 |
| 某一要件无对应取证材料 | 标注"证据缺位"，建议举证责任倒置、法律推定或申请法院调取 |
| 行政机关拒不提供材料 | 给出政府信息公开、行政复议、行政诉讼三条救济路径 |
