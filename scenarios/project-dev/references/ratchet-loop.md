# 棘轮循环（Ratchet Loop）原理

> 本文档解释 Ratchet Loop 的原理和来源。完整实现流程见 [iterative-loop.md](../iterative-loop.md)。

## 1. 什么是棘轮循环

棘轮（Ratchet）是一种机械装置，只允许单向旋转——可以前进，不能后退。

在迭代开发中，棘轮循环意味着：

```
改进 → 保留（棘轮前进）
退步 → 回退（棘轮不动）
```

每次迭代只保留让系统变好的改动，退步的改动立即丢弃。系统质量只会上升，不会下降。

## 2. 来源：autoresearch

Karpathy 的 autoresearch 项目用以下方式实现棘轮循环：

| 要素 | 实现 | 本质 |
|------|------|------|
| 评估指标 | `val_bpb`（验证集比特每字节） | 统一、可量化、公平可比 |
| 时间预算 | 固定5分钟训练 | 所有实验条件相同 |
| 保留 | val_bpb降低 → git commit | 改进才提交 |
| 回退 | val_bpb升高 → git revert | 退步就丢弃 |
| 记录 | `results.tsv` | 所有实验（成功+失败）都有记录 |

### 关键数字

- 首轮测试：~700次编辑实验
- 有效改进：~20个（2.9%成功率）
- 效果：训练时间从2.02h降至1.80h（-11%），后续降至1.65h（总-18%）
- 启示：低成功率是正常的，失败的经验同样有价值

## 3. 局限性

| 局限 | 说明 | 应对 |
|------|------|------|
| 局部最优 | 棘轮只进不退，可能卡在局部最优 | 允许"探索轮"：偶尔尝试激进改动 |
| 评估指标单一 | 单一指标可能忽略其他维度 | 多指标评估，但只有一个主指标 |
| 成功率低 | autoresearch只有2.9%成功率 | 正常现象，失败也有价值 |
| 需要量化 | 无法量化的项目不适合 | 退回标准模式 |

## 4. 与本质工坊的关系

```
场景C标准模式 → 一次交付 → 如需迭代 → 切换到迭代模式
场景C迭代模式 → 收敛 → 场景B2（蒸馏）→ 沉淀为Skill
场景F（Skill优化）→ 如需重写 → 场景C迭代模式
```

完整迭代流程见 [iterative-loop.md](../iterative-loop.md)。
