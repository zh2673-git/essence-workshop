# 口播稿管线规范（Broadcast Script Pipeline）

> 为「真人出镜 + 白板讲解 + 几分钟短视频」设计的轻量口播稿格式。
> 输出产物是一份 Markdown 稿子，用户可直接对着镜头念，同时参考白板提示书写/指画。

---

## 适用场景

- 真人出镜的口播视频
- 配合白板书写/图示的讲解
- 时长 2-5 分钟，旁白总字数 800-1500 字
- 触发词：`写口播稿`、`写录制稿`、`口播脚本`、`录制脚本`、`真人出镜稿`

---

## 核心公式

```
口播稿 = 三阶骨架（是什么 → 为什么 → 怎么做）
      × 坡度理解（点出概念 → 建立联系 → 详细解释）
      × 白板提示（每段旁白对应白板上写什么/画什么）
      × 口语短句
      × 可选风格引用（distillation/output/ 下已蒸馏的创作者 Skill）
      × 可选寓言化（敏感内容替换「为什么」段落）
```

---

## 输出格式

文件：`output/{topic}/broadcast-script.md`

```markdown
---
title: "标题"
target_duration: "3-5min"
audience: "外行/初学者/同行"
sensitive_mode: false
style_reference: "pusuzhidao"  # 可选，引用 distillation/output/{name}.skill
total_chars: 0
---

# 口播稿：{标题}

## 镜头 01｜钩子（坡度阶段1：点出概念）
- 时长：15-20s
- 白板：写下一句话 / 画一个核心词
- 旁白：
  - …（每句≤20字，口语化）
  - …

## 镜头 02｜是什么（坡度阶段1：一句话定义）
- 时长：30-45s
- 白板：写出定义 / 画核心结构
- 旁白：
  - …
  - …

## 镜头 03｜为什么（坡度阶段2：建立联系）
- 时长：60-90s
- 白板：画对比 / 写痛点 / 画流程
- 旁白：
  - …
  - …

> 若 sensitive_mode=true，此处替换为「寓言片段」：
> ## 镜头 03｜寓言引入（坡度阶段2：建立联系）
> - 时长：60-90s
> - 白板：画出寓言场景（动物/器物/自然现象）
> - 旁白：
>   - 300-600 字寓言故事，前 80% 不出现真实概念名
>   - 结尾一句点破：「这个故事讲的，其实是 {概念}。」

## 镜头 04｜怎么做（坡度阶段3：详细解释）
- 时长：60-120s
- 白板：分步骤写/画，每步配关键词
- 旁白：
  - 第一步：…
  - 第二步：…
  - 第三步：…

## 镜头 05｜收尾（行动号召 / 记忆锚点）
- 时长：15-30s
- 白板：写下金句 / 画记忆图
- 旁白：
  - …
```

---

## 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `title` | 是 | 视频标题，用于封面和发布 |
| `target_duration` | 是 | 目标时长，如 `2-3min`、`3-5min` |
| `audience` | 是 | 外行 / 初学者 / 同行 |
| `sensitive_mode` | 否 | 是否启用寓言化，默认 `false` |
| `style_reference` | 否 | 参考创作者风格，如 `朴素之道`，需在正文中落实其表达 DNA |
| `total_chars` | 是 | 成稿后填写纯文本字数 |
| 镜头编号 | 是 | 按拍摄顺序编号 |
| 时长 | 是 | 单镜头建议时长 |
| 白板 | 是 | 该镜头中白板上写什么、画什么、指哪里 |
| 旁白 | 是 | 口语短句，每句不超过 20 字 |

---

## 写作规则

1. **口语优先**：去掉书面语、长句、从句，改成短句。
2. **每句≤20字**：方便念、方便剪、方便加字幕。
3. **一段一个白板动作**：每段旁白必须对应一个清晰的白板操作。
4. **坡度三段必须齐全**：
   - 钩子 + 是什么 = 阶段1：点出概念
   - 为什么（或寓言）= 阶段2：建立联系
   - 怎么做 + 收尾 = 阶段3：详细解释
5. **总字数 800-1500 字**：对应 2-5 分钟口播。
6. **不写镜头运动、不写转场**：真人出镜 + 固定机位为主，减少拍摄负担。

## 创作者风格引用

当用户指定某位创作者风格（如「用朴素老师的风格」「像朴素之道那样讲」）时，不直接硬编码其风格规则，而是引用 `distillation/output/` 下已蒸馏的对应 Skill。默认创作者风格为 `pusuzhidao.skill`。

### 执行流程

```
步骤 C1：确认风格引用
  ├── 解析 style_reference 字段
  ├── 映射到 distillation/output/{name}.skill/
  └── 若用户说「朴素老师」「朴素之道」→ 默认映射到 pusuzhidao.skill

步骤 C2：读取蒸馏 Skill
  ├── 读取 SKILL.md 中的「表达DNA」
  ├── 读取「决策启发式」
  ├── 读取「场景路由表」
  └── 读取「反例与黑名单」

步骤 C3：生成风格化口播稿
  ├── 保留三阶骨架（是什么→为什么→怎么做）
  ├── 保留坡度理解（点出概念→建立联系→详细解释）
  ├── 用 Skill 中的表达 DNA 替换通用表达
  └── 用 Skill 中的决策启发式指导每段写作

步骤 C4：自检
  ├── 是否保留了该创作者的核心语感？
  ├── 是否避免了反例与黑名单中的行为？
  ├── 是否仍然符合每句≤20 字？
  └── 是否仍然每个镜头都有白板提示？
```

### 默认风格：朴素之道

默认情况下，口播稿管线使用 `distillation/output/pusuzhidao.skill/` 作为创作者风格示例。

如需使用其他创作者风格：
1. 先通过 distillation Skill 将该创作者蒸馏为独立 Skill（产出到 `distillation/output/{name}.skill/`）
2. 在 `style_reference` 中填写 Skill 目录名（如 `pusuzhidao`）
3. 口播稿管线自动读取并应用其表达 DNA

### 引用规则

- **禁止凭空想象**：任何新创作者风格必须先蒸馏，再引用
- **聚焦表达 DNA**：口播稿主要引用 Skill 中的「表达DNA」「决策启发式」「反例与黑名单」
- **保留管线约束**：无论引用何种风格，仍需遵守每句≤20字、每段白板提示、坡度三阶段等硬性规则
- **风格冲突处理**：若创作者风格与管线约束冲突，优先满足管线约束，同时最大限度保留风格语感

## 敏感内容处理

当用户声明敏感，或内容涉及以下情况时，标记 `sensitive_mode: true`：

- 商业机密 / 内部数据
- 人物隐私
- 平台限流 / 监管风险
- 不便公开披露的细节

处理方式：

1. 调用 `fable-concept` Skill 生成一则 300-600 字寓言。
2. 用寓言替换「为什么」段落。
3. 寓言前 80% 不出现真实概念名或该领域术语。
4. 结尾一句点破概念。
5. 点破后接 1-2 句正式解释，把观众拉回真实场景。

## 执行流程

```
步骤 B1：触发判断
  └── 命中「写口播稿」「口播脚本」「录制稿」等 → 进入口播稿管线

步骤 B2：输入来源
  ├── 用户给话题/概念 → 先走 A（知识探索）提炼三阶内容
  ├── 用户给文章/笔记 → 直接精简改写
  └── 用户提供零散想法 → 按三阶补全

步骤 B3：敏感判断
  ├── 敏感 → 调用 G（概念寓言化），标记 sensitive_mode=true
  └── 公开 → sensitive_mode=false

步骤 B4：生成口播稿
  ├── 写钩子（1 段，15-20s）
  ├── 写是什么（1 段，30-45s）
  ├── 写为什么 / 寓言（1 段，60-90s）
  ├── 写怎么做（1-2 段，60-120s）
  └── 写收尾（1 段，15-30s）

步骤 B5：口播化改写
  ├── 书面语 → 口语
  ├── 长句 → 短句
  └── 补全每段的白板提示

步骤 B6：自检
  ├── 总字数 800-1500 字？
  ├── 每句≤20 字？
  ├── 每个镜头都有白板提示？
  ├── 坡度三阶段完整？
  ├── 敏感内容已寓言化？
  └── 若指定创作者风格，其表达 DNA 已落实？
```

## 一键生成白板录制内容

口播稿生成后，可以调用 `whiteboard-recorder` 的桥接脚本，自动把每个镜头的「旁白」和「白板提示」转成 Excalidraw 白板画面 + 提词器脚本。

```bash
python -m whiteboard-recorder.scripts.broadcast_to_whiteboard output/{topic}/broadcast-script.md
```

### 桥接逻辑

```
口播稿 broadcast-script.md
  ├── 解析 frontmatter（标题/时长/受众/敏感标记）
  ├── 按「镜头」拆分
  │   ├── 镜头编号
  │   ├── 场景标题
  │   ├── 时长
  │   ├── 旁白 → 作为提词器脚本
  │   └── 白板提示 → 作为 AI 生成白板画面的依据
  └── 调用 LLM 为每个镜头生成 Excalidraw 元素
       └── 输出 .whiteboard.json + .excalidraw + 提词器.md
```

### 输出文件

| 文件 | 用途 |
|------|------|
| `{作品名}.whiteboard.json` | WhiteboardCaster 项目文件 |
| `{作品名}.excalidraw` | 可在 Excalidraw 中二次编辑 |
| `{作品名}-提词器.md` | 与镜头同步的提词器文本 |

### 与视频号管线的关系

口播稿管线本身**不生成 MP4**，只产出稿子。

但通过与 `whiteboard-recorder` 桥接，可以实现：

1. 口播稿 → 白板内容 + 提词器
2. 真人出镜 + 白板录制 → MP4
3. 也可以继续用现有视频号管线做 TTS + 自动录制（无人出镜场景）

默认场景下，口播稿 + 白板桥接就是为真人出镜录制的完整前置流程。

---

## 失败模式

| 步骤 | 失败场景 | 处理 |
|------|---------|------|
| B2 | 输入太散，无法提炼三阶 | 先走 A（知识探索） |
| B3 | 敏感判断不明确 | 询问用户是否启用寓言化 |
| B4 | 成稿字数 <800 字 | 补充案例或步骤细节 |
| B4 | 成稿字数 >1500 字 | 删减次要细节，或拆分为上下集 |
| B5 | 句子过长 | 强制拆分为≤20 字短句 |
| B5 | 某段缺少白板提示 | 补全，必须每段对应一个白板动作 |
| B5 | 创作者风格生硬套搬 | 回归该创作者真实表达 DNA，用其惯用场景和比喻重写 |
| B6 | 坡度阶段缺失 | 补齐缺失阶段 |
