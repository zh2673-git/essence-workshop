# 场景 E2：连载小说 · 多章节叙事系统

> 将知识、案例、话题或原创构想转化为多章节连载小说，通过世界观、人物、情节的渐进展开，持续吸引读者。

---

## 触发条件

- 用户说「写连载小说」「小说连载」「连载XX」
- 用户希望把某个知识领域/历史事件/行业话题写成故事
- 用户要求管理一个已有的连载系列（续写、查看状态、调整大纲）

---

## 核心流程

```
步骤0：确定小说类型与定位
步骤1：世界观构建
步骤2：人物设定
步骤3：故事主线设计
步骤4：分季/分章大纲
步骤5：单章写作
步骤6：章节配图生成
步骤7：连载前检查
步骤8：推送到公众号草稿箱
步骤9：更新连载状态
```

---

## 步骤0：确定小说类型与定位

### 小说类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 知识小说 | 用故事讲清楚一个知识体系 | 用侦探小说讲逻辑学 |
| 行业小说 | 展现某个行业的规则与人性 | 投行、医院、律所故事 |
| 历史小说 | 在历史背景下虚构人物命运 | 宋代小吏的职场沉浮 |
| 科幻/奇幻 | 构建架空世界，探讨核心议题 | AI 觉醒后的社会 |
| 现实题材 | 贴近当下生活，反映普遍困境 | 中年程序员的转型 |

### 定位三问

1. **读者是谁？** 他们关心什么？
2. **核心冲突是什么？** 人物与世界、人物与人物、人物与自我的冲突。
3. **每期带走什么？** 每章结束时，读者除了悬念，还能获得什么认知或情绪？

---

## 步骤1：世界观构建

使用三阶方法论：

| 阶段 | 问题 | 输出 |
|------|------|------|
| 是什么 | 这是一个什么样的世界？ | 时代、地点、社会规则、核心技术/制度 |
| 为什么 | 这个世界为什么值得写？ | 核心议题、普遍冲突、与读者的连接点 |
| 怎么做 | 怎么让读者进入这个世界？ | 开场场景、代入感设计、信息释放节奏 |

输出文档：`output/{series-name}/world-building.md`

详见 [references/world-building-guide.md](references/world-building-guide.md)

---

## 步骤2：人物设定

为每个主要人物创建人物卡：

| 人物 | 必填项 |
|------|--------|
| 主角 | 姓名、身份、欲望、恐惧、核心冲突、成长弧线 |
| 反派 | 姓名、动机、与主角的对立点、复杂性 |
| 配角 | 姓名、功能、与主角关系、关键场景 |

输出路径：`output/{series-name}/characters/`

详见 [references/character-card-template.md](references/character-card-template.md)

---

## 步骤3：故事主线设计

### 三幕式骨架

| 幕 | 占比 | 内容 |
|----|------|------|
| 第一幕：建置 | 25% | 介绍世界、人物、日常，触发核心冲突 |
| 第二幕：对抗 | 50% | 主角面对障碍，展开主要情节 |
| 第三幕：结局 | 25% | 高潮、解决、人物变化 |

### 核心冲突类型

- **人物 vs 人物**：利益、理念、情感对立
- **人物 vs 世界**：制度、时代、规则压迫
- **人物 vs 自我**：恐惧、欲望、价值观挣扎

---

## 步骤4：分季/分章大纲

### 分季原则

- 一季对应一个完整的故事弧线
- 每季 8-20 章
- 季末必须有一个阶段性结局，同时留下下一季钩子

### 分章原则

每章大纲必须包含：

| 要素 | 说明 |
|------|------|
| 章节编号 | 如 第1章、第2章 |
| 标题 | 吸引点击的短标题 |
| 场景 | 时间、地点 |
| 出场人物 | 本章出现的人物 |
| 本章冲突 | 人物遇到什么麻烦 |
| 情节推进 | 本章让整体故事前进多少 |
| 结尾钩子 | 悬念、反转、情绪爆点 |
| 字数目标 | 2000-4000字 |
| 配图需求 | 人物卡/场景图/关系图 |

详见 [references/chapter-outline-template.md](references/chapter-outline-template.md)

---

## 步骤5：单章写作

### 单章规范

- **字数**：2000-4000字
- **开头**：直接进入场景或冲突，不要先铺垫世界观
- **中间**：对话和动作推动情节，控制叙述比例
- **结尾**：必须留悬念或情绪钩子
- **段落**：一段1-3句话，适配手机屏幕

### 每章必备结构

```
场景切入 → 冲突升级 → 人物选择 → 意外/转折 → 钩子收尾
```

### 下章预告

每章末尾加「下一章：XXX」预告，增强追读动力。

详见 [references/fiction-style-guide.md](references/fiction-style-guide.md)

---

## 步骤6：章节配图生成

每章配图 3-5 张：

| 类型 | 数量 | 说明 |
|------|------|------|
| 章节封面 | 1张 | 标题+核心场景/人物，wide画幅 |
| 人物卡 | 0-2张 | 新出场人物或重点人物 |
| 场景图 | 1-2张 | 关键场景可视化 |
| 关系图/动态图 | 0-1张 | 人物关系或情节转折动画 |

小说配图风格：
- 强调情绪和氛围
- 可使用插画风格、剪影、场景氛围图
- 文字量较少，视觉焦点明确

---

## 步骤7：连载前检查

| 检查项 | 标准 | 处理方式 |
|--------|------|---------|
| 单章字数 | 2000-4000字 | 不达标则扩展或精简 |
| 开头是否直接 | 3段内进入场景/冲突 | 否则删除铺垫 |
| 结尾是否有钩子 | 必须有 | 补充悬念或情绪爆点 |
| 人物是否一致 | 符合人物卡设定 | 不一致则调整 |
| 配图数量 | 3-5张 | 不足则补充 |
| 下章预告 | 必须有 | 补充 |
| 封面图 | 1张 | 生成章节封面 |

---

## 步骤8：推送到公众号草稿箱

走公众号管线，按"连载小说"标准执行：

```bash
python -m scripts.pipelines.wechat.publish chapter-XX.md --cover output/wechat/images/cover.png --author "公众号名"
```

连载小说的 frontmatter 示例：

```yaml
---
title: "第3章：入局"
series: "示例连载"
series_order: 3
style: serial-fiction
author: "公众号名"
---
```
