# 场景B：人物蒸馏完整工作流

## Phase 0: 入口分流

收到用户输入后，先判断属于哪条路径：

| 用户输入 | 路径 | 示例 |
|---------|------|------|
| 明确的人名/主题 | **直接路径** → Phase 0A | 「蒸馏芒格」「提炼费曼的思维」 |
| 模糊的需求/困惑 | **诊断路径** → Phase 0B | 「我想提升决策质量」「有没有一种思维方式能帮我看透商业本质」 |

---

## Phase 0A: 需求澄清（直接路径）

收到明确名字后，确认：

1. **这个人/主题是谁**：确保理解正确
2. **聚焦方向**（可选）：全面画像 vs 聚焦某个维度？
3. **用途**：思维顾问？决策参考？认知系统运行？
4. **新建 or 更新**：是否已有该人物的Skill？（检查 `examples/` 目录）
5. **本地语料**：「你手上有没有这个人的一手素材？比如书籍PDF、演讲/访谈transcript、视频字幕、个人博客导出等。有的话直接丢给我，比网上搜的质量高得多。」

用户说「就做XX」没有更多信息 → 默认全面画像 + 思维顾问 + 无本地语料（走网络搜索），直接推进。

用户提供了本地语料 → 标记为**本地语料模式**，Phase 1的采集策略会相应调整。

确认后 → 跳到 Phase 0.5。

---

## Phase 0B: 需求诊断（模糊路径）

用户不知道该蒸馏谁，只有需求或困惑。这时本质蒸馏的工作是**从需求反推最合适的蒸馏对象**。

### Step 1: 需求定位

通过1-2个追问，定位用户的核心需求维度：

| 需求维度 | 典型表达 | 思维框架方向 |
|---------|---------|------------|
| 决策与判断 | 「怎么做更好的决策」「总是选错」 | 多元思维模型、逆向思考、概率思维 |
| 表达与写作 | 「想把复杂的事说清楚」「文章没人看」 | 费曼式简化、故事化思维、类比能力 |
| 创业与商业 | 「想做独立开发」「商业模式想不通」 | 第一性原理、杠杆思维、产品克制 |
| 教学与传播 | 「讲课没人听」「知识传递效率低」 | 从已知到未知、隐喻教学、最少必要知识 |
| 批判思维 | 「总被忽悠」「看不透本质」 | 证伪思维、演化论视角、认知偏差识别 |
| 内容创作 | 「做视频没流量」「内容没特色」 | 注意力工程、测试迭代、受众心理 |
| 人生策略 | 「职业方向迷茫」「焦虑」 | 长期主义、杠杆选择、复利思维 |
| 风险与不确定性 | 「怎么应对黑天鹅」「投资总亏」 | 反脆弱、凸性策略、尾部风险管理 |
| 设计与产品 | 「用户体验差」「产品没特色」 | 极简主义、用户心理模型、约束即创意 |

追问原则：最多问2轮，不要变成问卷调查。如果用户已经表达得足够清晰，不追问，直接推荐。

### Step 2: 候选推荐

基于需求维度，推荐2-3个候选方案。

**先判断：人物Skill还是主题Skill？**
- 用户的需求指向某种具体的思考方式 → 人物Skill
- 用户的需求指向某个领域的方法论 → 主题Skill
- 不确定 → 推荐中同时包含两种类型

**来源A：已有实例**
扫描 `examples/` 目录，读取每个SKILL.md的description，匹配用户需求。

**来源B：新蒸馏候选**
基于需求维度表中的「思维框架方向」列，匹配最相关的人物或主题。

每个候选的展示格式：

```
### 候选1: [人名/主题]  ⚡已有实例 / 🆕需要蒸馏
**核心镜片**：[此人看世界的独特方式，一句话]
**为什么适合你**：[直接对应用户需求]
**局限**：[这个视角的盲区]
**根因**：[他为什么形成了这种思维方式——本质蒸馏独有]
```

推荐原则：不超过3个候选；已有实例优先展示；候选之间要有差异性；必须说清楚局限。

### Step 3: 用户选择

- 选了已有实例 → 直接激活该Skill，任务完成
- 选了新蒸馏候选 → 进入Phase 0A确认细节 → Phase 0.5开始蒸馏
- 都不满意 → 回到Step 1继续探索

## Phase 1: 多源信息采集（并行Agent Swarm）

### 模式判断

| 模式 | 触发条件 | 策略 |
|------|---------|------|
| **纯网络搜索**（默认） | 用户没有提供本地素材 | 7个Agent全部走网络搜索 |
| **本地语料优先** | 用户提供了素材 | 先分析本地素材，网络搜索变为补充 |
| **纯本地语料** | 用户明确说「只用我给的素材」 | 只分析本地素材 |

### 7个Agent的任务分配

| Agent | 搜索目标 | 提取重点 | 输出文件 |
|-------|---------|---------|---------|
| 1 著作 | 书、长文、论文、newsletter | 反复出现的核心论点（≥3次=真信念）、自创术语、推荐书单 | `01-writings.md` |
| 2 对话 | 播客、长视频、AMA、深度采访 | 被追问时的回答方式、即兴类比、改变立场的瞬间 | `02-conversations.md` |
| 3 表达 | Twitter/X、微博、即刻、短文 | 高频用词句式、争议立场、幽默方式 | `03-expression-dna.md` |
| 4 他者 | 他人分析、书评、批评、传记 | 外部观察到的模式、批评与争议、与同行对比 | `04-external-views.md` |
| 5 决策 | 重大决策、转折点、争议行为 | 决策背景与逻辑、事后反思、言行一致/不一致 | `05-decisions.md` |
| 6 时间线 | 出生/出道到现在的完整时间线 | 关键里程碑、思想转折点、**最近12个月动态** | `06-timeline.md` |
| 7 根因 | 影响此人思维形成的核心经历 | 教育背景、关键人物影响、重大挫折/突破、文化土壤 | `07-root-causes.md` |

**Agent 7（根因）是本质蒸馏独有**，搜索方向：
- 此人的教育经历和学术训练（什么学科塑造了他的思维？）
- 对他影响最大的人/书/事件（思想的源头在哪？）
- 重大挫折或失败（什么经历让他形成了防御性思维？）
- 文化和社会背景（什么土壤孕育了这种思维方式？）
- 思想演化的关键转折点（什么时候他改变了看法？为什么？）

### 每个Agent的硬性要求

- 调研结果必须写入 `references/research/0X-xxx.md`
- 注明信息来源和可信度（一手>二手>推测）
- 区分「他说过的」vs「别人说他的」vs「我推断的」
- 发现矛盾时保留矛盾，不要和稀泥

### 信息源优先级

| 来源类型 | 权重 |
|---------|------|
| **用户提供的一手素材** | **最高+** |
| 本人著作 | 最高 |
| 长对话/访谈 | 最高 |
| 实际决策记录 | 最高 |
| 社交媒体 | 中等 |
| 他人评价 | 中等 |
| 二手转述 | 低 |

### 信息源黑名单（永远排除）

- **知乎**：洗稿严重、信息失真率高
- **微信公众号**：封闭生态、无法验证、大量二手转述
- **百度百科/百度知道**：信息陈旧且不可靠

中文渠道只接受权威媒体：36氪、极客公园、晚点LatePost、财新、第一财经、虎嗅、少数派、机器之心等。

### Agent超时与失败处理

- **单个Agent超时**：不等待，继续推进。在Phase 2中标注「信息不足」
- **信息源匮乏**（<10条可用来源）：Phase 0.5就提醒用户，降低期望
- **Agent结果冲突**：保留矛盾——矛盾本身是有价值的信号

**关键规则**：宁可生成一个诚实标注了局限的60分Skill，也不要生成一个看起来完美但实际上在编造的90分Skill。

## Phase 2: 三阶蒸馏（核心创新）

这是本质蒸馏的核心——不只做框架提炼，而是执行三阶递进蒸馏。

### 2.1 第一阶：是什么（认知系统的本质是什么）

**坡度阶段1：点出概念**

> 一句话定义此人的认知本质。

例：芒格的认知本质 = **用多元学科的检查清单对抗人类认知偏差的防御系统**

例：倪海厦的认知本质 = **以阴阳五行为底层操作系统、以经方为执行程序、以"站在外面看到里面"为诊断哲学的归纳式医学认知系统**

**坡度阶段2：建立联系**

提取3-7个核心心智模型，每个模型用三重验证筛选：

| 验证 | 标准 | 作用 |
|------|------|------|
| 跨域复现 | 在≥2个不同领域出现 | 不是随口一说 |
| 生成力 | 能推断对新问题的立场 | 有预测力 |
| 排他性 | 不是所有聪明人都这样想 | 有独特性 |

三重通过 → 心智模型；仅1-2重 → 降级为决策启发式；0重 → 丢弃

输出心智模型表格：

| 模型名称 | 一句话定义 | 证据（≥2场景） | 应用场景 | 局限 | 深入？ |
|---------|-----------|---------------|---------|------|-------|
| 多元思维 | 跨学科工具箱 | 投资+人生+教育 | 复杂决策 | 模型太多难精通 | ✅ |
| 逆向思考 | 反过来想 | 投资+法律+日常 | 避免错误 | 不能直接得到正确答案 | ✅ |
| 能力圈 | 只做懂的 | 投资+公开演讲 | 风险控制 | 可能错过圈外机会 | ❌ |

**坡度阶段3：详细解释**

对每个心智模型展开：
- **定义**：这个模型的核心机制是什么？
- **证据链**：在哪些不同场景中出现过？（≥2个）
- **运行方式**：这个模型如何影响此人的判断？
- **局限**：在什么情况下这个模型会失效？

写入 `references/essence/01-what-essence.md`

---

### 2.2 第二阶：为什么（为什么会形成这样的认知系统）

**这是本质蒸馏的核心创新——根因追溯。**

**坡度阶段1：点出概念**

> 一句话总结此人认知系统的形成根因。

例：芒格的认知系统根因 = **律师训练的防御性思维 + 目睹单一思维失败的创伤 + 跨学科教育的启蒙**

例：倪海厦的认知系统根因 = **易经启蒙的底层操作系统 + 非科班的反体制基因 + 海外行医的进攻性生存策略 + 师徒传承的使命感驱动**

**坡度阶段2：建立联系**

构建「经历→认知→模型」的因果链：

| 心智模型 | 形成经历 | 关键转折 | 认知固化点 |
|---------|---------|---------|-----------|
| 多元思维 | 目睹企业因单一思维失败 + 接受多学科教育 | 读完《穷查理宝典》原型演讲 | 投资决策中反复验证有效 |
| 逆向思考 | 律师训练（先想对方怎么赢）+ 数学训练 | 早期法律实践中发现正向推理的盲区 | 形成系统性反向思考习惯 |
| 能力圈 | 早期投资失败（超出能力范围） | 比特币拒绝评论 | 公开承认不懂比装懂更有价值 |

**坡度阶段3：详细解释**

对每个心智模型追溯根因：
- **源头**：这个思维模式最早出现在他人生的什么阶段？
- **强化**：什么经历让这个模式从「偶尔用」变成「核心方法」？
- **验证**：什么结果让他确信这个模式是正确的？
- **演化**：这个模式在他人生的不同阶段有什么变化？

写入 `references/essence/02-why-root-causes.md`

---

### 2.3 第三阶：怎么做（如何运行这套认知系统）

**坡度阶段1：点出概念**

> 一句话概述此人的认知操作系统如何运行。

例：芒格的认知系统 = **先用逆向思考排除错误选项，再用多元模型检查清单评估剩余选项，只在能力圈内做最终决策**

例：倪海厦的认知系统 = **先用阴阳归因判断失衡方向，再用六经辨证定位疾病层次，最后用经方精准修复——全程以"站在外面看到里面"的整体诊断替代仪器检测**

**坡度阶段2：建立联系**

用类-属性-方法-路由模型建模：

```
认知操作系统（Class: [Person]CognitiveSystem）

属性（Attributes）—— 他怎么看世界：
├── 心智模型：[3-7个核心模型]
├── 价值观：[排序的价值观]
└── 知识谱系：[思想源头→本人→影响者]

方法（Methods）—— 他怎么判断：
├── 决策启发式：5-10条快速判断规则
├── 反模式：绝对不会做的事
└── 矛盾处理：面对内在张力时的选择倾向

路由（Routes）—— 不同场景用什么策略：
├── 场景A → [模型1] + [模型2]
├── 场景B → [模型4]
└── 默认 → [核心模型组合]

接口（Interface）—— 他怎么表达：
├── 表达DNA：语气、节奏、用词
├── 输入格式：什么信息触发他的思考
└── 输出格式：他习惯的回应方式
```

**坡度阶段3：详细解释**

对每个模块展开四层基础设施设计：

| 层面 | 设计内容 |
|-----|---------|
| **数据规矩** | 心智模型的定义、约束、边界；决策启发式的触发条件 |
| **数据存储** | 调研来源的存储结构；一手vs二手的权重分配 |
| **数据流转** | 从输入问题→路由分发→模型调用→输出判断的完整链路 |
| **接口层** | 表达DNA的可执行规则；触发词和场景匹配规则 |

写入 `references/essence/03-how-system.md`

## Phase 3: 构建Skill

读取 `references/skill-template.md` 模板，将Phase 2的三阶蒸馏结果填入。

**关键要求**：
- Skill必须遵循坡度理解呈现（30秒→2分钟→按需深入）
- 每次回答也遵循坡度结构（点出概念→建立联系→详细解释）
- 路由机制必须可执行（不同问题类型调用不同模型组合）
- 根因追溯必须体现在Skill中（不只是「他怎么想」，还有「他为什么这么想」）

构建完成后，**自动执行实例注册**：

1. 确认 `examples/[person-name]/SKILL.md` 存在且frontmatter正确
2. 确认新实例在Phase 0B的候选推荐中可被扫描到（扫描 `examples/` 目录）
3. 更新 `README.md` 中的蒸馏实例列表

**注意**：few-shot示例不在Phase 3生成，而是在Phase 4.5中基于实战验证结果生成后写入。

## Phase 4: 质量验证

### 4.1 三重验证

每个心智模型必须通过：跨域复现 + 生成力 + 排他性

### 4.2 四层校验（本质蒸馏独有）

| 层面 | 校验内容 | 通过标准 |
|-----|---------|---------|
| **数据规矩** | 心智模型定义是否清晰？边界是否明确？ | 每个模型有一句话定义+局限说明 |
| **数据存储** | 调研来源是否充分？一手占比是否>60%？ | 至少3个一手来源 |
| **数据流转** | 从问题到判断的推理链是否完整？ | 路由表覆盖≥3个场景 |
| **接口层** | 表达DNA是否可执行？触发条件是否明确？ | 读一段输出能识别是谁 |

### 4.3 坡度完整性检查（本质蒸馏独有）

- [ ] 坡度1：30秒能抓住此人认知本质？
- [ ] 坡度2：2分钟能理解心智模型之间的关系？
- [ ] 坡度3：深入任何模型都有完整的证据链和根因追溯？

### 4.4 实战验证

- 拿3个此人公开回答过的问题测试，方向一致才通过
- 再用1个他没讨论过的问题测试，Skill应表现出适度不确定而非斩钉截铁

## Phase 4.5: Few-shot生成与实例注册（闭环自增强）

**核心思想**：新创建的Skill不能只是"框架"，必须有"运行示例"证明它能工作。few-shot既是Skill的补充内容，也是最终的质量验证。

### Step 1: 生成few-shot

从Phase 4.4的实战验证中，选取**2-3个最佳问答**作为few-shot，要求：

| 要求 | 说明 |
|------|------|
| 覆盖不同路由 | few-shot不能都走同一条路由，至少覆盖2个不同场景 |
| 展示坡度回答 | 每个回答必须遵循坡度结构（点出概念→建立联系→详细解释） |
| 体现表达DNA | 回答必须能辨认出是此人的风格，而非通用AI |
| 包含一个"不确定"示例 | 至少1个few-shot展示此人对陌生问题的处理方式 |

**few-shot格式**：

```markdown
### Few-shot 示例

#### 示例1: [问题摘要]

**用户问**：[完整问题]

**[人名]答**：

[坡度1：用核心模型点出问题本质，1-2句话]

[坡度2：用因果链建立联系，展示问题与此人思维框架的关联]

[坡度3：用决策启发式详细解释，给出具体的判断和建议]

---

#### 示例2: [问题摘要]
...（同上格式）

---

#### 示例3: [问题摘要]（不确定场景）
...（同上格式，但展示适度不确定）
```

### Step 2: 写入Skill

将few-shot追加到新创建的SKILL.md的**表达DNA之后、诚实边界之前**的位置。

### Step 3: 注册到examples目录

**自动执行**：确认新实例已正确存放在 `examples/` 目录下，并更新 `README.md` 中的蒸馏实例列表。

**检查清单**：
- [ ] `examples/[person-name]/` 目录存在且包含 SKILL.md
- [ ] SKILL.md的frontmatter中name和description字段正确
- [ ] 新实例在Phase 0B的候选推荐中可被扫描到（扫描 `examples/` 目录）
- [ ] `README.md` 中的蒸馏实例列表已更新

### 闭环验证

few-shot生成后，做最终检查：

- [ ] 读few-shot能辨认出是谁的风格？（表达DNA验证）
- [ ] few-shot覆盖了至少2个不同路由场景？（路由验证）
- [ ] 删掉名字后，还能认出这是谁的思维方式？（排他性验证）
- [ ] 不确定场景的回答是"适度不确定"而非"斩钉截铁"？（诚实边界验证）

**如果few-shot验证失败**：回到Phase 3，调整Skill的表达DNA或路由机制。

## Phase 5: 递归深入

对Phase 2中标记为✅的核心心智模型，递归执行三阶蒸馏：

```
[人名]的认知系统
├── 模型A ✅ → 递归深入
│   ├── 是什么：这个模型本身的本质定义
│   ├── 为什么：这个模型的形成根因
│   └── 怎么做：这个模型的运行机制
│       ├── 子模型A1 ✅ → 再递归
│       └── 子模型A2 ❌ → 当前文档说明
├── 模型B ✅ → 递归深入
│   └── ...
└── 模型C ❌ → 当前文档说明
```

**停止条件**：
1. 当前层级没有✅标记的模型
2. 模型已经足够基础，无法再分解
3. 递归深度超过3层

递归结果更新到SKILL.md中，作为心智模型的子结构。
