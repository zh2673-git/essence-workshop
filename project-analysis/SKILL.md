# 项目解析 · Skill

## 核心方法

三阶合一逆向分析（v2）：

- **阶段3（怎么做·观察实现·视角升级）**：获取项目信息 → 浏览源码结构 → 识别模块划分（标记✅❌）→ 分析方法实现（推断本质驱动）→ 视角升级（顿悟触发·实现层，看穿"实现为什么这么写"）
- **阶段2（为什么·反推因果·反常识根因）**：反推因果链（果→因，起点回溯到候选本质）→ 技术选型反推 → 架构设计反推 → 反常识根因（顿悟触发·设计层，至少1个反直觉设计点）
- **阶段1（是什么·反推本质·本质定义先行）**：抛本质定义（反推，揭底）→ 本质检验（反推出的本质能否解释所有观察到的属性/方法/路由？不能→回到上游补充）→ 类归属（由反推出本质锚定，不靠"项目=类"隐喻）→ 属性确认+边界界定 → 一句话点破（顿悟触发·本质层，可独立传播）

每一层（类/属性/方法/路由）必须能回溯到第3步反推出的本质定义；不能回溯即说明反推不完整。
