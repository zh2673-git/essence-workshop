# 蒸馏方法论详解

## 信息采集的Agent Prompt模板

spawn subagent时，用以下结构给任务（以Agent 7根因为例）：

```
你的任务：调研[人名]的思维形成根因。
搜索方向：
- 此人的教育经历和学术训练（什么学科塑造了他的思维？）
- 对他影响最大的人/书/事件（思想的源头在哪？）
- 重大挫折或失败（什么经历让他形成了防御性思维？）
- 文化和社会背景（什么土壤孕育了这种思维方式？）
- 思想演化的关键转折点（什么时候他改变了看法？为什么？）

输出要求：
1. 写入文件：[skill目录]/references/research/07-root-causes.md
2. 注明信息来源和可信度（一手>二手>推测）
3. 区分「他说过的」vs「别人说他的」vs「我推断的」
4. 发现矛盾时保留矛盾，不要和稀泥
```

其他Agent类似，替换搜索方向和输出文件即可。

---

## 根因可信度分级

| 级别 | 标准 | 标注方式 |
|------|------|---------|
| **确定根因** | 此人亲口说过「因为X经历，我形成了Y想法」 | 🟢 确定 |
| **强推测** | 经历和模型高度吻合，有多个间接证据 | 🟡 强推测 |
| **弱推测** | 逻辑上说得通但缺乏直接证据 | 🟠 弱推测 |
| **无法追溯** | 找不到合理的形成原因 | ⚪ 未知 |

---

## 矛盾处理原则

矛盾是人格的核心特征，不是需要修复的Bug。

### 三种矛盾类型

1. **时间性矛盾**（观点演化）→ 记录演化轨迹，标注「早期」「近期」
2. **领域性矛盾**（不同场景不同规则）→ 分领域记录，不强求统一
3. **本质性张力**（价值观内在冲突）→ 明确记录为「核心张力」

### 错误的处理方式

- ❌ 选一边忽略另一边
- ❌ 编一个调和的解释
- ❌ 假装矛盾不存在

---

## 信息不足时的处理

| 情况 | 处理 |
|------|------|
| 某个维度公开信息很少 | 在Skill中标注「信息不足，此维度为推测」 |
| 只有二手信息 | 降低置信度，标注为「据[来源]报道」 |
| 信息相互矛盾且无法判断 | 并列呈现，让用户自行判断 |
| 此人刻意不公开某方面 | 尊重边界，注明「此人对这个话题保持沉默」 |
| 根因无法追溯 | 标注为⚪未知，不编造因果链 |

## 话题/现象蒸馏（场景B2）

与人物蒸馏（场景B）并行，共享三阶蒸馏方法论，但调研维度适配话题特征。

详细工作流见 [B2-topic-distillation.md](../workflows/B2-topic-distillation.md)。

### 7个调研Agent

| Agent | 搜索目标 | 与人物蒸馏的差异 |
|-------|---------|----------------|
| 1 定义 | 核心定义、概念边界、分类体系 | 替代"著作与系统思考" |
| 2 起源 | 历史起源、演化脉络、关键节点 | 替代"对话与即兴思考" |
| 3 视角 | 不同学科/流派/立场的观点 | 替代"表达风格DNA" |
| 4 案例 | 典型案例、实证研究、应用场景 | 替代"他者视角与批评" |
| 5 趋势 | 最新进展、前沿动态、未来预测 | 替代"决策记录与行动" |
| 6 争议 | 批评、质疑、反面论证 | 替代"人物时间线" |
| 7 本质 | 话题的底层逻辑、第一性原理 | 共享"根因分析"但追问不同 |

### 核心差异

| 维度 | 人物蒸馏 | 话题蒸馏 |
|------|---------|---------|
| 核心单元 | 心智模型 | 认知框架 |
| 根因追溯 | 经历→认知→模型 | 条件→机制→表现 |
| 独有追问 | 矛盾处理、防御性思维 | 反事实推理（去掉某条件，现象还存在吗？） |
| 输出格式 | Skill（认知操作系统） | Skill（认知操作系统） |
