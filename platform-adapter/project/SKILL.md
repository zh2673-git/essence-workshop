# 项目交付平台适配器

> 将项目开发产出适配为标准项目仓库。这不是传统平台，而是项目交付物规范。

## 平台约束

| 约束项 | 说明 |
|--------|------|
| 仓库结构 | 必须含 README.md、src/、tests/、docs/、.gitignore |
| 文档 | README含项目简介+安装+使用；docs/含设计文档 |
| 依赖 | requirements.txt（Python）或 package.json（Node.js） |
| 构建 | 必须有可运行的构建/启动命令 |
| 测试 | tests/含至少基础测试用例 |
| .gitignore | 排除 node_modules/、__pycache__/、.env、dist/、output/ 等 |

## 标准项目结构

```
{project-name}/
├── README.md              # 项目简介+安装+使用
├── .gitignore             # 排除构建产物和依赖
├── requirements.txt       # Python依赖（或package.json）
├── src/                   # 源代码
│   ├── __init__.py
│   └── ...
├── tests/                 # 测试用例
│   └── ...
├── docs/                  # 设计文档
│   ├── design.md          # 架构设计
│   └── api.md             # API文档（如有）
└── examples/              # 使用示例（可选）
```

## 适配流程

```
1. 项目开发产出 → 检查结构完整性
2. 生成/完善 README.md（如缺失）
3. 生成/完善 .gitignore（如缺失）
4. 检查依赖文件（requirements.txt / package.json）
5. 检查测试用例（tests/至少有基础测试）
6. 验证构建命令可运行
7. 导出项目仓库
```

## 与项目开发场景的关系

| 层级 | 职责 |
|------|------|
| **场景（scenarios/project-dev/）** | 定义项目开发工作流（四层设计+Ratchet Loop） |
| **认知引擎** | 提供项目本质定义和设计思路 |
| **格式管线（markdown/）** | 生成设计文档 |
| **平台适配（本适配器）** | 将代码+文档组装为标准项目仓库 |

## 特殊说明

项目交付不是传统意义的"平台"（如公众号、视频号），而是：
- **纯工具产品**：代码+文档的可运行包
- **协同产品**：代码+文档+LLM运行时+符号系统

项目开发的两种路径由 `scenarios/project-dev/` 定义，本适配器负责最终交付物规范。

## 使用场景

- 项目开发完成后的交付打包
- 迭代开发收敛后的版本发布
- 项目解析后的重构交付
