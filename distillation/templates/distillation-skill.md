# 蒸馏Skill模板（v2方法论内置）

> 本模板用于场景B（人物蒸馏）和场景B2（话题蒸馏）的Phase 3构建阶段。
> 完整模板详见 [skill-optimization/references/skill-template.md](../../skill-optimization/references/skill-template.md)
>
> **v2 方法论内置**：
> - **三阶合一**（取代9宫格正交矩阵）：阶段1（是什么·点出本质）→ 阶段2（为什么·建立因果）→ 阶段3（怎么做·展开方法·视角升级），三阶 = 坡度三阶段，不再正交
> - **本质定义先行**：阶段1必须用本质定义（揭底），不是描述性定义（外延）；必须通过本质检验（能解释所有其他属性吗？）
> - **顿悟触发**（v2新增）：阶段1结尾一句话点破 + 阶段2中段反常识根因 + 阶段3结尾视角升级
> - **类-属性-方法-路由由本质定义锚定**：本质定义 → 类归属 → 属性展开 → 方法体现 → 路由适配，每一层必须能回溯到本质定义（不再靠"项目=类"隐喻硬接）
> - **递归只重做阶段1**：递归深入时只重新找子概念的本质定义，不再重复阶段2/3

## 使用方式

1. 读取 `references/skill-template.md` 获取完整模板
2. 将Phase 2三阶蒸馏结果填入模板对应位置
3. 确保Skill内置v2三阶合一主线（阶段1点出本质→阶段2建立因果→阶段3展开方法·视角升级），不再走"三阶×坡度三阶段"的9宫格
4. 确保Skill内置本质定义先行（阶段1抛本质定义+本质检验，禁止描述性定义）
5. 确保Skill内置顿悟触发（一句话点破+反常识根因+视角升级）
6. 确保类-属性-方法-路由每一层能回溯到本质定义，杜绝隐喻硬接
7. 确保路由机制可执行（不同问题类型调用不同模型组合）
8. 确保根因追溯体现在Skill中（不只是「他怎么想」，还有「他为什么这么想」）

## 三阶蒸馏子文件生成要求（v2模板内置）

> **关键**：`references/essence/` 下的三个子文件必须按v2方法论内置格式生成，禁止使用v1的9宫格结构。

### `01-what-essence.md`（阶段1：是什么·点出本质）

按以下顺序生成，禁止跳步或重排：

```markdown
## 阶段1：是什么（本质定义先行）

### 本质定义
> [一句话揭底——揭示本质属性，不是描述外延]

| 类型 | 形式 | 候选 | 是否本质 |
|-----|------|------|---------|
| 描述性定义（旧版默认） | 属加种差，描述外延 | ... | ❌ 只说"长什么样" |
| **本质定义（v2强制）** | 揭示本质属性 | ... | ✅ 决定所有其他属性 |

### 本质检验（必须通过）
| 心智模型/核心框架（待解释属性） | 候选定义能否解释 | 说明 |
|--------------------|---------------|------|
| 属性A | ✅/❌ | ... |
| 属性B | ✅/❌ | ... |
| 属性C | ✅/❌ | ... |

- 全部 ✅ → 是本质，保留
- 任意 ❌ → 是表面描述，降级重写

### 类归属（由本质定义确定）
[由本质定义锚定类归属——不再靠"项目=类"隐喻硬接]

### 一句话点破（顿悟触发·阶段1结尾）
> [最简洁的本质表达，可独立传播——30秒能抓住本质，旧框架被打破（"原来不是X，是Y"）]
```

## 反例与黑名单（必填章节）

> **设计依据**：借鉴 Darwin Skill dim9 + SkillLens 实证——有"不要做什么"清单的Skill质量显著更高。
> 详见 [skill-optimization/references/skill-antipatterns.md](../../skill-optimization/references/skill-antipatterns.md)

在Skill模板的"价值观与反模式"章节之后，必须增加"反例与黑名单"章节，格式如下：

```markdown
### 反例与黑名单

| # | 反模式 | 为什么不要做 | 替代做法 |
|---|--------|-------------|---------|
| 1 | [此人物/话题绝对不会做的事] | [为什么这违背其本质] | [此人/此框架会怎么做] |
| 2 | [此人物/话题绝对不会做的事] | [为什么这违背其本质] | [此人/此框架会怎么做] |
| ... | | | |
```

**填充规则**：
- 至少列出5条反模式
- 反模式必须具体可执行（不是"不要做坏事"这种空话）
- 每条反模式必须对应此人/此话题的核心心智模型/认知框架或价值观（即必须能回溯到阶段1本质定义）
- 反模式是"此人/此框架绝对不会做的事"，不是"偶尔会做的事"

## 递归深入子结构（v2·只重做阶段1）

> **v2 关键变更**：递归时只重做阶段1（重新找子概念的本质定义），不再重复阶段2/3，避免无限展开。子概念的本质定义找到后，因果链和怎么做沿用父级。

对Phase 2标记为✅的核心心智模型/认知框架，递归生成子结构时只生成：

```markdown
### 子模型：[模型名]

#### 阶段1：本质定义（重新找）
> [这个模型本身的本质定义——揭底，不是描述]

#### 本质检验
| 待解释属性 | 候选定义能否解释 |
|----------|---------------|
| ... | ✅/❌ |

#### 一句话点破
> [最简洁的子概念本质表达]

[不再重复阶段2因果链和阶段3怎么做——沿用父级]
```
