# 场景B2：话题/现象蒸馏完整工作流

> 与场景B（人物蒸馏）并行，专门用于蒸馏一个话题、现象、趋势或领域。
> 共享三阶蒸馏方法论，但调研维度和产出结构适配话题特征。

## Phase 0: 入口分流

收到用户输入后，先判断属于哪条路径：

| 用户输入 | 路径 | 示例 |
|---------|------|------|
| 明确的人名 | **场景B** → 人物蒸馏 | 「蒸馏芒格」「提炼费曼的思维」 |
| 明确的话题/现象/领域 | **场景B2** → 话题蒸馏 | 「蒸馏内卷」「提炼第一性原理」「分析AI Agent趋势」 |
| 模糊的需求/困惑 | **诊断路径** → Phase 0B | 「我想提升决策质量」「怎么应对不确定性」 |

---

## Phase 0A: 需求澄清

收到明确话题后，确认：

1. **话题边界**：这个话题的核心范围是什么？有没有需要排除的子话题？
2. **聚焦方向**（可选）：全面画像 vs 聚焦某个维度（如只看中国语境、只看技术层面）？
3. **用途**：认知框架？决策参考？内容创作素材？
4. **新建 or 更新**：是否已有该话题的Skill？（检查 `output/` 目录）
5. **本地语料**：「你手上有没有关于这个话题的一手素材？比如论文PDF、行业报告、调研数据等。有的话直接丢给我，比网上搜的质量高得多。」

用户说「就做XX」没有更多信息 → 默认全面画像 + 认知框架 + 无本地语料（走网络搜索），直接推进。

确认后 → 跳到 Phase 0.5。

---

## Phase 0B: 需求诊断（模糊路径）

用户不知道该蒸馏什么，只有需求或困惑。这时需要**从需求反推最合适的蒸馏对象**。

### Step 1: 需求定位

通过1-2个追问，定位用户的核心需求维度：

| 需求维度 | 典型表达 | 蒸馏方向 |
|---------|---------|---------|
| 理解现象 | 「内卷到底是什么」「为什么大家都在焦虑」 | 话题蒸馏：现象本质 |
| 掌握方法 | 「怎么用第一性原理思考」「如何做产品决策」 | 话题蒸馏：方法论体系 |
| 把握趋势 | 「AI Agent会怎么发展」「未来教育会怎样」 | 话题蒸馏：趋势与推演 |
| 解决问题 | 「怎么应对不确定性」「如何做艰难决策」 | 话题蒸馏：问题→框架 |
| 领域入门 | 「想了解行为经济学」「区块链到底在说什么」 | 话题蒸馏：领域地图 |

### Step 2: 候选推荐

基于需求维度，推荐2-3个候选方案。

**判断：人物Skill还是话题Skill？**
- 用户的需求指向某种具体的思考方式 → 人物Skill（场景B）
- 用户的需求指向某个领域/现象/方法论 → 话题Skill（场景B2）
- 不确定 → 推荐中同时包含两种类型

每个候选的展示格式：

```
### 候选1: [话题名]  🆕需要蒸馏
**核心问题**：[这个话题要回答的根本问题，一句话]
**为什么适合你**：[直接对应用户需求]
**已知局限**：[这个话题的争议或未解之处]
```

推荐原则：不超过3个候选；候选之间要有差异性；必须说清楚局限。

## Phase 1: 多源信息采集（并行Agent Swarm）

### 7个Agent的任务分配

| Agent | 搜索目标 | 提取重点 | 输出文件 |
|-------|---------|---------|---------|
| 1 定义 | 核心定义、概念边界、分类体系 | 不同学科/流派的定义差异、概念的演化、边界模糊处 | `01-definitions.md` |
| 2 起源 | 历史起源、演化脉络、关键节点 | 何时/何地/为何出现、关键推动者/事件、演化转折点 | `02-origins.md` |
| 3 视角 | 不同学科/流派/立场的观点 | 共识与分歧、各流派的独特贡献、跨学科交叉点 | `03-perspectives.md` |
| 4 案例 | 典型案例、实证研究、应用场景 | 正面/反面案例、案例的可复现性、边界条件 | `04-cases.md` |
| 5 趋势 | 最新进展、前沿动态、未来预测 | 最近12个月动态、技术/政策/市场变化、可信预测 | `05-trends.md` |
| 6 争议 | 批评、质疑、反面论证 | 主要批评者及其论点、争议焦点、未解问题 | `06-controversies.md` |
| 7 本质 | 话题的底层逻辑、第一性原理 | 为什么存在这个现象？底层驱动力是什么？什么条件下会消失？ | `07-essence.md` |

**Agent 7（本质）是话题蒸馏独有**，搜索方向：
- 这个话题/现象为什么存在？它的存在条件是什么？
- 如果去掉某个条件，这个现象还会存在吗？（反事实推理）
- 有没有更底层的原理可以解释这个现象？（还原论视角）
- 这个现象在不同文化/时代/领域中有什么变体？不变的是什么？
- 这个话题的"第一性原理"是什么？最不可删除的核心是什么？

### 每个Agent的硬性要求

- 调研结果必须写入 `references/research/0X-xxx.md`
- 注明信息来源和可信度（一手>二手>推测）
- 区分「事实」vs「观点」vs「推测」
- 发现矛盾时保留矛盾，不要和稀泥

### 信息源优先级

| 来源类型 | 权重 |
|---------|------|
| **用户提供的一手素材** | **最高+** |
| 学术论文/专著 | 最高 |
| 权威机构报告 | 最高 |
| 一手数据/统计 | 最高 |
| 深度调查报道 | 中等 |
| 行业分析 | 中等 |
| 社交媒体讨论 | 低 |
| 二手转述/科普文 | 低 |

### 信息源黑名单（永远排除）

- **知乎**：洗稿严重、信息失真率高
- **微信公众号**：封闭生态、无法验证、大量二手转述
- **百度百科/百度知道**：信息陈旧且不可靠

中文渠道只接受权威来源：知网论文、36氪深度、极客公园、晚点LatePost、财新、第一财经、虎嗅、少数派、机器之心、学术期刊等。

## Phase 2: 三阶蒸馏（核心创新）

### 2.1 第一阶：是什么（这个话题的本质是什么）

**坡度阶段1：点出概念**

> 一句话定义这个话题/现象的本质。

例：内卷的本质 = **在总量受限的系统中，个体通过增加投入争夺相对位置，导致整体效率下降而无人能退出的囚徒困境**

例：第一性原理的本质 = **剥离所有类比和经验假设，从最基本的事实出发重新构建认知的思维方式**

**坡度阶段2：建立联系**

提取3-7个核心认知框架（对应人物蒸馏的心智模型），每个框架用三重验证筛选：

| 验证 | 标准 | 作用 |
|------|------|------|
| 跨域解释力 | 能解释≥2个不同领域的案例 | 不是特例 |
| 生成力 | 能预测新场景下的表现 | 有预测力 |
| 排他性 | 不是所有类似话题都适用 | 有独特性 |

三重通过 → 核心框架；仅1-2重 → 降级为观察视角；0重 → 丢弃

输出核心框架表格：

| 框架名称 | 一句话定义 | 证据（≥2场景） | 应用场景 | 局限 | 深入？ |
|---------|-----------|---------------|---------|------|-------|
| 位置竞争 | 总量不变时争夺相对位 | 教育+职场+婚恋 | 资源稀缺分析 | 不适用增量市场 | ✅ |
| 囚徒困境 | 个体理性导致集体非理性 | 补课+加班+军备竞赛 | 破解内卷 | 需要信息不对称条件 | ✅ |
| 制度锁定 | 退出成本高于忍受成本 | 户口+学历+编制 | 制度改革 | 需要强制度约束 | ❌ |

**坡度阶段3：详细解释**

对每个核心框架展开：
- **定义**：这个框架的核心机制是什么？
- **证据链**：在哪些不同场景中出现过？（≥2个）
- **运行方式**：这个框架如何解释话题中的现象？
- **局限**：在什么情况下这个框架会失效？

写入 `references/essence/01-what-essence.md`

### 2.2 第二阶：为什么（为什么会出现这个话题/现象）

**这是话题蒸馏的核心创新——本质追问。**

**坡度阶段1：点出概念**

> 一句话总结这个话题/现象的深层成因。

例：内卷的深层成因 = **资源分配的零和博弈结构 + 社会流动通道收窄 + 退出机制缺失的三重锁定**

例：第一性原理流行的深层成因 = **信息爆炸导致类比思维失效 + 复杂系统需要底层锚点 + 硅谷成功叙事的传播效应**

**坡度阶段2：建立联系**

构建「条件→机制→表现」的因果链：

| 核心框架 | 存在条件 | 驱动机制 | 表现形式 | 消失条件 |
|---------|---------|---------|---------|---------|
| 位置竞争 | 资源总量受限 + 评价标准单一 | 相对收益>绝对收益 | 排名焦虑、过度投入 | 增量市场、多元评价 |
| 囚徒困境 | 信息不对称 + 缺乏信任机制 | 个体最优≠集体最优 | 军备竞赛、底线竞争 | 重复博弈、强制协调 |
| 制度锁定 | 沉没成本高 + 替代路径少 | 退出成本>忍受成本 | 路径依赖、改革困难 | 降维打击、外部冲击 |

**坡度阶段3：详细解释**

对每个核心框架追问本质：
- **底层驱动力**：什么力量在维持这个现象的存在？
- **反事实推理**：如果去掉某个条件，现象会怎样变化？
- **跨域变体**：这个现象在不同领域/文化中有什么变体？不变的是什么？
- **消失条件**：在什么条件下这个现象会自然消退？

写入 `references/essence/02-why-root-causes.md`

### 2.3 第三阶：怎么做（如何理解和应对这个话题）

**坡度阶段1：点出概念**

> 一句话概述如何运用这个话题的认知框架。

例：内卷的认知框架 = **先判断是否在零和博弈中，再评估退出成本，最后选择竞争/合作/退出策略**

**坡度阶段2：建立联系**

用类-属性-方法-路由模型建模：

```
话题认知框架（Class: [Topic]CognitiveFramework）

属性（Attributes）—— 这个话题的核心维度：
├── 核心框架：[3-7个认知框架]
├── 争议焦点：[未解决的分歧]
└── 知识谱系：[起源→发展→当前状态]

方法（Methods）—— 如何运用这些框架：
├── 分析启发式：5-10条快速判断规则
├── 常见误区：最容易犯的认知错误
└── 应对策略：不同立场下的行动方案

路由（Routes）—— 不同场景用什么框架：
├── 场景A（理解现象）→ [框架1] + [框架2]
├── 场景B（做出决策）→ [框架4] + [框架5]
├── 场景C（预测趋势）→ [框架3] + [框架6]
└── 默认 → [核心框架组合]

接口（Interface）—— 如何表达和传播：
├── 表达DNA：讲解这个话题的最佳方式
├── 输入格式：什么问题触发这个框架
└── 输出格式：这个框架产出的判断类型
```

**坡度阶段3：详细解释**

| 层面 | 设计内容 |
|-----|---------|
| **数据规矩** | 核心框架的定义、约束、边界；分析启发式的触发条件 |
| **数据存储** | 调研来源的存储结构；事实vs观点vs推测的权重分配 |
| **数据流转** | 从输入问题→路由分发→框架调用→输出判断的完整链路 |
| **接口层** | 表达DNA的可执行规则；触发词和场景匹配规则 |

写入 `references/essence/03-how-system.md`

## Phase 3: 构建Skill

读取 `templates/distillation-skill.md` 模板，将Phase 2的三阶蒸馏结果填入。

**关键差异（vs人物蒸馏Skill）**：
- 人物Skill的核心是「心智模型」→ 话题Skill的核心是「认知框架」
- 人物Skill的根因追溯是「经历→认知→模型」→ 话题Skill是「条件→机制→表现」
- 人物Skill的路由是「不同问题调用不同模型组合」→ 话题Skill是「不同场景调用不同框架组合」
- 人物Skill有「表达DNA」→ 话题Skill有「讲解DNA」（如何向他人讲清这个话题）

### 产出清单

Phase 3 完成后，`output/[topic-name]/` 目录必须包含以下文件：

| 文件 | 必须 | 说明 |

## Phase 4: 验证与迭代

### 4.1 一致性检查

- [ ] 核心框架之间没有逻辑矛盾
- [ ] 每个框架都有≥2个跨域证据
- [ ] 本质追问的因果链逻辑自洽
- [ ] 路由机制覆盖主要使用场景

### 4.2 诚实边界标注

| 标注类型 | 说明 |
|---------|------|
| 🟢 确定结论 | 有充分实证支持 |
| 🟡 主流观点 | 多数专家认同但仍有争议 |
| 🟠 推测性结论 | 逻辑上说得通但缺乏直接证据 |
| ⚪ 未知领域 | 当前研究无法回答 |

### 4.3 Few-shot示例生成

为Skill生成3-5个使用示例，覆盖不同路由场景：

```
用户问：[典型问题]
→ 路由到：[框架X + 框架Y]
→ 回答思路：[用框架X分析...，用框架Y补充...]
→ 预期输出：[判断类型]
```
