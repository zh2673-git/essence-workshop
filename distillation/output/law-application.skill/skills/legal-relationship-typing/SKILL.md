# 法律关系定性 · 原子技能

## 法律声明

> 本技能仅根据用户提供的事实进行法律关系领域的初步判断，**不构成法律意见**。法律关系认定可能因事实补充、证据变化或解释争议而改变，最终判断应由执业法律专业人员作出。

---

## 一、核心概念

### 1.1 四域划分

| 领域 | 主体关系 | 核心问题 | 典型救济路径 |
|------|----------|----------|--------------|
| 民事 | 平等主体 | 权利/义务/责任归属 | 民事诉讼、仲裁 |
| 刑事 | 国家追诉嫌疑人 | 犯罪成立与刑罚 | 刑事控告、公诉 |
| 行政 | 行政机关与相对人 | 行政行为合法性 | 行政复议、行政诉讼 |
| 宪法基本权利 | 国家权力与公民 | 基本权利限制与合宪性 | 申诉、合宪性审查申请 |

### 1.2 行政行为识别三要件

行政行为自动触发行政规范性文件检索，三要件必须同时满足：

| 要件 | 说明 | 反例 |
|------|------|------|
| 主体 | 行政机关或法律、法规授权的组织 | 政府作为民事主体签合同 |
| 权力 | 行使行政权力（处罚、许可、征收、强制、审批等） | 行政机关内部人事任免 |
| 外部性 | 对公民、法人或其他组织的权利义务产生外部影响 | 行政机关之间公文往来 |

### 1.3 常见竞合类型

- **刑民交叉**：同一行为涉及犯罪与侵权/违约；
- **行民交叉**：行政行为效力影响民事法律关系；
- **行刑交叉**：行政违法与刑事犯罪界限模糊；
- **民宪交叉**：民事纠纷中涉及基本权利限制。

## 二、工作流程

### 步骤 1：提取主体关系

- 双方是平等主体，还是管理与被管理关系？
- 是否存在多方主体？各方地位如何？

### 步骤 2：判断行为性质

- 是私法行为、犯罪行为、行政行为，还是基本权利限制？
- 是否满足行政行为三要件？

### 步骤 3：确定救济路径

- 诉讼、复议、控告、申诉？
- 是否存在前置程序？

### 步骤 4：识别核心争点

- 权利义务归属？
- 犯罪成立与否？
- 行政行为合法性？
- 基本权利限制合宪性？

### 步骤 5：标注竞合与并行路径

若存在竞合，明确主路径与需要并行分析的辅助路径。

## 三、输入/输出模板

### 输入

- 经 `fact-trimming` 输出的结构化法律要素清单；
- 或用户原始事件描述（本技能可先调用 `fact-trimming`）。

### 输出模板

```markdown
## 法律关系定性报告

### 一、主体关系
- 主体A：[身份/地位]
- 主体B：[身份/地位]
- 关系性质：[平等/管理/追诉/权力限制]

### 二、行为性质判断
| 行为 | 初步定性 | 理由 |
|------|----------|------|
| [行为1] | [民事/刑事/行政/宪法] | [理由] |

### 三、行政行为识别（如相关）
| 要件 | 是否满足 | 说明 |
|------|----------|------|
| 主体 | ✅/❌ | ... |
| 权力 | ✅/❌ | ... |
| 外部性 | ✅/❌ | ... |
| **结论** | 是/否行政行为 | ... |

### 四、领域结论
- **主领域**：[民事/刑事/行政/宪法基本权利]
- **子类型**：[如：借款合同/侵权/行政处罚/行政许可]
- **是否竞合**：是/否
- **竞合类型**：[刑民交叉/行民交叉/行刑交叉/民宪交叉]

### 五、救济路径
1. [主要救济路径]
2. [备选救济路径]

### 六、核心争点
1. [争点1]
2. [争点2]

### 七、置信度
- 整体置信度：[高/中/低]
- 不确定点：[说明]
```

## 四、示例

**输入**："住建局以未取得施工许可证为由，对开发商罚款 50 万元。开发商认为处罚过重，且自己已补办手续。"

**输出**：

```markdown
## 法律关系定性报告

### 一、主体关系
- 主体A：住建局（行政机关）
- 主体B：开发商（行政相对人）
- 关系性质：管理与被管理关系

### 二、行为性质判断
| 行为 | 初步定性 | 理由 |
|------|----------|------|
| 住建局罚款 | 行政行为 | 行政机关行使行政处罚权 |

### 三、行政行为识别
| 要件 | 是否满足 | 说明 |
|------|----------|------|
| 主体 | ✅ | 住建局为行政机关 |
| 权力 | ✅ | 行使行政处罚权 |
| 外部性 | ✅ | 对开发商财产权产生外部影响 |
| **结论** | 是行政行为 | 触发行政规范性文件自动检索 |

### 四、领域结论
- **主领域**：行政
- **子类型**：行政处罚
- **是否竞合**：否

### 五、救济路径
1. 行政复议
2. 行政诉讼

### 六、核心争点
1. 未取得施工许可证即施工的行为是否成立
2. 罚款 50 万元是否过重（是否符合过罚相当原则）
3. 补办手续是否构成从轻/减轻处罚事由

### 七、置信度
- 整体置信度：高
```

## 五、验证规则

| 检查项 | 通过标准 |
|--------|----------|
| 主体关系明确 | 已识别各方主体及其相互关系 |
| 行为性质匹配 | 行为性质判断与事实要素一致 |
| 行政行为三要件完整 | 若涉及行政机关，已逐条检视三要件 |
| 救济路径可行 | 救济路径与主领域匹配 |
| 竞合已标注 | 存在竞合时明确列出并行分析路径 |

---

## 六、失败模式与降级

| 失败场景 | 处理动作 |
|----------|----------|
| 事实不足以定性 | 输出"领域不明"，列出需要补充的事实 |
| 存在严重竞合 | 列出各可能领域，建议并行分析 |
| 行政行为三要件部分满足 | 标注"疑似行政行为"，提示进一步核实 |
| 跨法域因素 | 明确标注法域适用范围，建议当地律师 |
