---
name: fact-trimming
description: |
  当用户描述具体事件、案件经过或生活事实时，从原始叙述中提取具有规范意义的法律事实。
  典型触发场景：
  - 用户提供口语化、情绪化的案件描述，需要整理为法律分析可用的事实结构；
  - 需要区分客观事实、主观判断、法律评价与背景信息；
  - 需要将生活语言转化为法律语言；
  - 需要为后续的请求权基础分析、涵摄推理准备要件事实。
  本技能不直接给出法律结论，只输出结构化事实清单。
  边界：不能替代证据调查，只能整理已确认或待确认的事实；不涉及法条检索与法律适用。

# 事实裁剪 · 原子技能

## 法律声明

> 本技能仅对用户提供的事实进行结构化提取与表述转化，**不构成法律意见，不替代证据审查**。提取结果应被视为后续法律分析的草稿输入，最终事实认定需由执业法律专业人员结合证据材料完成。

---

## 一、核心概念

### 1.1 三层事实结构

```
原始叙述（生活叙事）
    ↓ 提取与筛选
法律相关事实（具有规范意义的事实单元）
    ↓ 类型化与对应
要件事实（可直接对应法律构成要件的事实）
```

### 1.2 九类事实单元

| 事实类型 | 核心问题 | 示例 | 常见证据来源 |
|----------|----------|------|--------------|
| 主体事实 | 何人？ | 行为人、受害人、合同当事人、行政机关 | 身份证、营业执照、组织机构代码证 |
| 行为事实 | 做了什么？ | 签约、支付、处罚、拒绝履行 | 合同、转账记录、处罚决定书、往来函件 |
| 时间事实 | 何时？ | 合同成立时间、履行期限、处罚时间 | 合同日期、邮戳、系统日志、行政文书日期 |
| 地点事实 | 何地？ | 履行地、损害发生地、管辖连接点 | 不动产登记地址、现场照片、GPS 记录 |
| 对象事实 | 指向什么？ | 款项、货物、个人信息、行政行为 | 发票、收据、交付凭证、行政许可证 |
| 结果事实 | 产生什么后果？ | 损害发生、合同未履行、权益受限 | 鉴定意见、医院病历、催告回执 |
| 因果关系事实 | 行为与结果之间有无联系？ | 某一行为是否导致某一结果 | 鉴定意见、专家意见、连续事件记录 |
| 主观事实 | 主观状态如何？ | 故意、过失、明知、应知、善意 | 聊天记录、邮件、知情同意书、公示材料 |
| 程序事实 | 采取了什么程序行动？ | 催告、申诉、起诉、提出异议 | 送达回证、起诉状、受理通知书、政府信息公开申请回执 |

### 1.3 事实属性四分法

| 属性 | 特征 | 识别标志 | 处理方法 |
|------|------|----------|----------|
| 客观事实 | 可通过外部证据证明 | 具体行为、时间、地点、对象 | 保留并标注可能证据来源 |
| 主观判断 | 当事人基于感受的判断 | "我觉得""显然""肯定" | 剥离情绪，追问客观事实 |
| 法律评价 | 已带有规范归类 | "构成欺诈""属于违约" | 还原为支持评价的客观事实 |
| 背景信息 | 与法律判断无直接关联 | 性格描述、无关前因后果 | 剔除或仅作辅助理解 |

## 二、工作流程

### 阶段一：解构与提取

**步骤 1：切分最小事实单元**

将原始叙述拆解为"谁 + 做了什么 + 何时 + 何地 + 指向什么 + 产生什么"的最小单元。每个单元应可独立判断。

**步骤 2：事实类型化归类**

将切分后的事实单元归入九类事实单元。

**步骤 3：属性辨析与净化**

对每个事实单元，判断其属于客观事实、主观判断、法律评价还是背景信息，并进行净化。

### 阶段二：对应与转化

**步骤 4：生活语言 → 法律语言**

将净化后的客观事实转写为中性、规范的表述。

**步骤 5：标注待确认与争议事实**

- 缺失但对分析重要的事实 → 标注"待补充"，并提示可通过何种材料核实；
- 叙述中存在矛盾或需证据证实的事实 → 标注"争议/待核实"，并提示建议获取的证据类型。

### 阶段三：输出

按"结构化法律要素清单"模板输出。

### 输入

- 用户原始事件描述（可包含聊天记录、媒体报道、当事人陈述等）。

### 输出模板

```markdown

### 一、原始叙述摘要
[1-2 句话概括核心内容]

### 二、结构化法律要素

#### 主体事实
- [事实1]
- [事实2]

#### 行为事实
- [事实1]
- [事实2]

#### 时间事实
- [事实1]

#### 地点事实
- [事实1]

#### 对象事实
- [事实1]

#### 结果事实
- [事实1]

#### 因果关系事实
- [事实1]

#### 主观事实
- [事实1]

#### 程序事实
- [事实1]

### 三、生活语言 → 法律语言

| 原始表述 | 法律语言转化 | 说明 |
|----------|-------------|------|
| [原句] | [规范表述] | [为什么这样转化] |

### 四、待补充事实
1. [缺失事实1] → 建议核实材料：[材料/渠道]

### 五、争议/待核实事实
1. [争议事实1] → 建议获取证据：[证据类型/来源]

- [原句] → [剔除原因]
```

## 四、示例

**输入**："我去年借给朋友五万元，他当时说一个月后一定还我。后来过了三个月他都没还，每次问他都说再等等，最后把我微信删了。"

**输出**：

```markdown
## 事实裁剪报告

### 一、原始叙述摘要
出借人向借款人出借5万元，约定1个月归还，逾期3个月未还，借款人删除出借人微信。

### 二、结构化法律要素

#### 主体事实
- 出借人（叙述者）与借款人（朋友）之间成立借款关系
- 双方为自然人

#### 行为事实
- 双方就5万元借款达成合意
- 出借人向借款人交付借款
- 借款人承诺1个月后归还
- 借款人在履行期限届满后未归还
- 出借人多次催告
- 借款人删除出借人微信

#### 时间事实
- 借款时间：去年（需确认具体日期）
- 约定还款期限：1个月
- 逾期时间：3个月

#### 对象事实
- 借款标的：人民币5万元

#### 结果事实
- 借款人未按约定期限返还借款
- 双方失去联系

#### 因果关系事实
- 借款人未履行还款义务 → 出借人债权未实现

#### 主观事实
- 借款人明知还款义务（曾承诺还款）
- 借款人有履行能力但拒不履行（多次推诿后断绝联系）

#### 程序事实
- 出借人多次催告（可构成时效中断）

### 三、生活语言 → 法律语言

| 原始表述 | 法律语言转化 |
|----------|-------------|
| "他一直拖着不还" | 债务履行期限届满后，债务人未履行还款义务 |
| "每次问他都说再等等" | 债权人多次催告，债务人多次承诺履行但未实际履行 |
| "最后把我微信删了" | 债务人通过删除联系方式的方式，拒绝与债权人沟通 |

### 四、待补充事实
1. 借款的具体日期（影响诉讼时效计算）
2. 借款交付方式（现金/转账）
3. 是否有借条、欠条或借款合同
4. 借款人身份信息

### 五、已剔除信息
- "我真的太生气了" → 主观情绪
- "这种人简直不讲信用" → 主观评价
```

## 五、验证规则

| 检查项 | 通过标准 |
|--------|----------|
| 最小单元化 | 每个事实单元包含主体、行为、对象、结果等核心要素 |
| 类型完整 | 至少覆盖主体、行为、时间、结果四类事实 |
| 属性净化 | 主观判断和法律评价已还原为客观事实或明确标注 |
| 待确认标注 | 缺失或矛盾事实已明确列出 |
| 法律语言中性 | 转化后的表述不含情绪词和评价词 |

---

## 六、失败模式与降级

| 失败场景 | 处理动作 |
|----------|----------|
| 用户描述严重信息缺失 | 输出已提取事实 + 列出关键待补充信息 |
| 用户描述存在明显矛盾 | 标注争议事实，不自行调和 |
| 用户描述包含大量情绪化表达 | 剥离情绪后提取客观事实 |
| 无法判断某一事实的法律意义 | 保留原始表述，标注"法律意义待判断" |
