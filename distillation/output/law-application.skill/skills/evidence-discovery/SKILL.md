# 证据发现 · 辅助原子技能

## 法律声明

> 本技能仅根据法律要件反向推导可能的取证方向与材料来源，**不构成法律意见，不替代实际证据调查**。所列材料是否存在、是否可取得、最终是否被采信，均需由执业法律专业人员或有权机关依法核实。

---

## 一、核心概念

### 1.1 C-E 映射（构成要件 → 证据方案）

```
构成要件 C
    ↓ 反向推导
待证事实 F
    ↓ 匹配
证据材料 E（书证/物证/电子数据/视听资料/证人证言/鉴定意见/勘验笔录/当事人陈述）
    ↓ 标注
获取主体/渠道 + 证明力等级 + 获取难度 + 优先级
```

### 1.2 证明力等级

| 等级 | 含义 | 典型材料 |
|------|------|----------|
| 强 | 公权力背书或原始载体，反驳成本高 | 行政机关盖章文件、不动产登记簿、公证文书、法院裁判、银行原始流水 |
| 中 | 可相互印证的原始材料或第三方记录 | 合同原件、银行流水、聊天记录原始载体、发票、快递单据 |
| 弱 | 单方制作、复制件、利害关系人提供 | 手写便条、复印件、单方陈述、利害关系人证言 |

### 1.3 常见获取渠道分类

| 渠道类型 | 示例 | 适用材料 |
|----------|------|----------|
| 行政机关公开 | 政府信息公开申请、政务服务网、现场查询 | 审批、备案、许可、处罚、登记信息 |
| 司法/仲裁程序 | 申请法院调取、调查令、证据保全 | 对方持有材料、第三人持有材料 |
| 金融机构 | 银行柜台、网上银行、支付平台 | 转账流水、账户信息、交易凭证 |
| 市场主体 | 物业公司、开发商、平台运营方 | 合同、台账、监控、会议纪要 |
| 公证/鉴定机构 | 公证处、司法鉴定中心 | 证据保全公证、笔迹/质量/伤情鉴定 |
| 个人持有 | 当事人、证人 | 借条、聊天记录、照片、视频 |

## 二、工作流程

### 步骤 1：接收输入

读取 `subsumption` 或 `fact-trimming` 输出的「待证要件清单」与「事实裁剪报告」。

### 步骤 2：要件-事实映射

将每个构成要件还原为需要证明的具体事实单元。

### 步骤 3：证据材料推荐

为每个待证事实推荐 1-3 类主要证据材料，按证明力从高到低排序。

### 步骤 4：获取主体/渠道标注

为每类材料标注：
- 材料保管或制作主体；
- 获取方式（申请公开、自行调取、法院调取、公证保全等）；
- 可能的法律依据或程序要求；
- 时效或保管期限提示。

### 步骤 5：证明力与优先级评估

结合证据种类、来源、形成时间、是否原件、与当事人利害关系，给出证明力等级与取证优先级。

### 步骤 6：替代与补强方案

当主证据难以取得时，提供替代证据或补强路径。

### 步骤 7：输出取证指引表

按模板输出结构化报告。

## 三、输入/输出模板

### 输入

- `subsumption` 输出的「要件对照表」与「待证要件清单」；
- 或 `fact-trimming` 输出的「结构化法律要素」与「待补充事实」；
- 或用户直接描述的待证事项。

### 输出深度

- **默认**：输出完整报告（含输入摘要、要件-证据映射表、行政材料专项、替代与补强方案、证据链组合建议、风险提示、下一步行动建议）。
- **用户要求简明/快速指引时**：至少保留「一、输入摘要」「三、要件-证据映射表」「八、下一步行动建议」，其余可折叠或省略。

### 输出模板

```markdown
## 证据发现报告

- 法律关系领域：[民事/刑事/行政/宪法]

| C1 | [材料1] | [种类] | [主体/渠道] | 强/中/弱 | 高/中/低 | P0/P1/P2 |
| C1 | [材料2] | [种类] | [主体/渠道] | 强/中/弱 | 高/中/低 | P0/P1/P2 |

|----------|----------|----------|----------|------|

### 五、替代与补强方案
```

## 四、示例

**输入**：subsumption 输出显示，某小区楼顶平台改造案需证明：C1 改造行为存在；C2 改造未经业主大会同意；C3 改造未取得规划许可。

**输出**（节选）：

```markdown

| 要件编号 | 构成要件 | 需证明的核心事实 | 当前证据状态 |
|----------|----------|------------------|--------------|
| C1 | 改造行为存在 | 楼顶平台被搭建阳光房并出租 | 部分 |
| C2 | 未经业主大会同意 | 未形成合法有效的业主大会决议 | 无 |
| C3 | 未取得规划许可 | 阳光房未取得建设工程规划许可证 | 无 |

### 三、要件-证据映射表

| 要件 | 推荐证据材料 | 证据种类 | 获取主体/渠道 | 证明力 | 获取难度 | 优先级 |
|------|-------------|---------|--------------|--------|---------|--------|
| C1 | 现场照片/视频 | 视听资料 | 业主自行拍摄或公证保全 | 中 | 低 | P0 |
| C1 | 租赁合同/租金流水 | 书证/电子数据 | 物业公司、承租人、银行 | 强 | 中 | P0 |
| C1 | 阳光房现状勘验 | 勘验笔录 | 法院/公证处 | 强 | 高 | P1 |
| C2 | 业主大会会议记录及表决票 | 书证 | 业主委员会、街道办事处 | 强 | 中 | P0 |
| C2 | 业主清册及表决权面积统计 | 书证 | 业主委员会、住建局 | 强 | 中 | P0 |
| C3 | 建设工程规划许可证 | 行政许可 | 自然资源局 / 政府信息公开 | 强 | 低 | P0 |

| 材料名称 | 保管机关 | 获取方式 | 法律依据 | 备注 |
|----------|----------|----------|----------|------|

### 五、替代与补强方案

| 主证据 | 替代证据 | 补强方式 | 适用场景 |
|--------|----------|----------|----------|
```

## 五、验证规则

| 检查项 | 通过标准 |
|--------|----------|
| 要件覆盖完整 | 每个待证要件都有对应证据材料 |
| 获取主体明确 | 每条材料都标注保管/制作主体 |
| 渠道可执行 | 获取方式具体，非空泛建议 |
| 证明力分级 | 已按强/中/弱分级并说明理由 |
| 替代方案 | 对难以取得的主证据提供替代路径 |
| 风险提示 | 已标注时效、取得、证明力、程序四类风险 |

## 六、失败模式与降级

| 失败场景 | 处理动作 |
|----------|----------|
| 某一要件无明确对应证据 | 标注「证据缺位」，建议通过举证责任倒置、法律推定或申请法院调取 |
| 材料由对方或第三人持有 | 提示可申请法院依职权调取或申请调查令 |
| 行政机关不予公开 | 输出救济路径：行政复议 / 行政诉讼 / 投诉 |
| 涉及国家秘密、商业秘密、个人隐私 | 标注限制，建议申请不公开审理或保密质证 |
| 无法确认材料保管机关 | 标注「待核实」，给出可能的机关清单 |
| 多个机关相互推诿 | 给出牵头机关与协同机关，并提示保留沟通记录 |
| 历史档案可能灭失 | 提示替代证据（证人证言、历史航拍图、缴费凭证等） |

## 七、反例与黑名单

| # | 反模式 | 为什么不要做 | 替代做法 |
|---|--------|-------------|---------|
| 1 | 只推荐材料，不告诉去哪里取 | 用户无法落地执行 | 每条材料必须标注获取主体/渠道 |
| 2 | 把复印件与原件证明力等同 | 误导用户证据策略 | 明确区分原件/复制件，复制件需补强 |
| 3 | 忽略行政机关公章的公信力 | 浪费最强证据来源 | 优先推荐行政机关盖章的公文书证 |
| 4 | 不区分证明对象，堆砌材料 | 重点不清，效率低 | 按构成要件逐项映射 |
| 5 | 不考虑替代方案 | 主证据取不到时流程卡死 | 每个主证据至少给一个替代或补强路径 |
| 6 | 把非法取证材料列为推荐 | 可能导致证据排除或法律责任 | 只推荐合法取证渠道，非法渠道明确标红警告 |
