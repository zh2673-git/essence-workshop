# 第一阶：是什么 —— 法条适用的本质

## 一句话定义

> 法条适用是**在具体事实与抽象规范之间建立理性连接的推理技术**。

## 坡度1：点出概念

法条适用不是背诵法条，而是在面对一个具体事件时，知道：
- 从哪里开始找法条
- 找到多个法条时该用哪个
- 法条含义不明时如何解释
- 没有直接法条时如何谨慎填补

## 坡度2：建立联系 —— 7个核心认知框架

| 框架名称 | 一句话定义 | 跨域证据 | 应用场景 | 局限 |
|---------|-----------|---------|---------|------|
| **事实裁剪框架** | 从生活事实中提炼具有规范意义的法律事实 | 民事侵权、刑事定罪、行政违法 | 用户描述具体事件时 | 无法替代证据认定 |
| **请求权基础框架** | 按特定顺序检索支持权利主张的规范 | 合同纠纷、侵权诉讼、物权保护 | 判断"能不能告""凭什么告" | 主要用于民法 |
| **涵摄框架** | 将事实逐一对照法条构成要件 | 民事、刑事、行政各领域 | 判断某行为是否满足法条条件 | 需要充分证据 |
| **冲突裁决框架** | 用位阶与优先规则解决规范冲突 | 民法与公司法、新法与旧法 | 多个法条都可适用时 | 不能解决事实争议 |
| **兜底适用框架** | 穷尽具体规则后再启用兜底条款 | 反不正当竞争、一般侵权责任 | 没有具体规则可用时 | 容易被滥用 |
| **法律解释框架** | 从文义出发，经体系、历史到目的解释 | 知假买假、直系亲属认定 | 法条含义模糊时 | 不能超越文义边界 |
| **抗辩检视框架** | 任何请求权成立判断都要检查抗辩事由 | 诉讼时效、同时履行抗辩 | 评估权利最终能否实现 | 容易被初学者忽视 |

## 坡度3：详细解释

### 框架1：事实裁剪框架

**核心机制**：生活事实 → 法律事实 → 规范对象

**运行方式**：
1. 听取/阅读当事人描述
2. 剥离情绪、猜测、无关细节
3. 保留"主体、行为、客体、结果、时间、地点"等规范要素
4. 用法律语言重新表述

**局限**：事实裁剪建立在证据基础上，不能替代证据调查。

### 框架2：请求权基础框架

**核心机制**：合同 → 类合同 → 无因管理 → 物权 → 不当得利 → 侵权

**运行方式**：
- 先找当事人之间最紧密的法律关系
- 再考虑更一般的补充规则
- 最后考虑侵权等兜底保护

**局限**：主要用于民法；刑法和行政法有各自的审查结构。

### 框架3：涵摄框架

**核心机制**：法条 = 构成要件（T）+ 法律效果（R）；事实满足T → R发生

**运行方式**：
1. 分解法条构成要件
2. 为每个要件寻找事实对应
3. 判断每个要件是否成立
4. 综合判断法律效果

**局限**：需要事实清楚、证据充分；不能处理法律漏洞。

### 框架4：冲突裁决框架

**核心机制**：上位法优于下位法；特别法优于一般法；新法优于旧法

**运行方式**：
1. 判断规范层级是否相同
2. 相同层级时判断是否为一般/特别关系或新旧关系
3. 不同层级时直接适用上位法
4. 新旧一般/特别冲突时依法定程序裁决

**局限**：只解决规范冲突，不解决事实认定和价值权衡。

### 框架5：兜底适用框架

**核心机制**：具体类型化条款 → 例示性兜底 → 概括性兜底 → 一般条款

**运行方式**：
1. 穷尽具体规则
2. 例示性兜底用同类解释规则
3. 概括性兜底最后使用
4. 一般条款作为最后手段

**局限**：容易被滥用，导致裁判恣意。

### 框架6：法律解释框架

**核心机制**：文义是起点，体系是边界，目的是最高准则

**运行方式**：
1. 先确定词句通常含义
2. 再看条文在法律体系中的位置
3. 参考立法历史和目的
4. 做合宪性审查

**局限**：解释不能超越文义可能范围。

### 框架7：抗辩检视框架

**核心机制**：请求权成立 ≠ 权利最终实现，还需排除抗辩

**运行方式**：
1. 检视请求权是否成立
2. 检视是否存在抗辩事由
3. 判断抗辩是否成立
4. 得出最终法律效果

**局限**：不同法律领域的抗辩事由不同。
