# 法条适用 · 认知框架 (law-application-essence)

> 在具体事实与抽象规范之间建立理性连接的推理技术。

## 概述

基于7维度深度调研，通过三阶蒸馏提炼法条适用的认知框架。包含7个核心认知框架、10条决策启发式、6条场景路由和完整的表达DNA。

## 核心认知框架

| # | 框架名称 | 一句话定义 |
|---|---------|-----------|
| 1 | 事实裁剪框架 | 从生活事实中提炼具有规范意义的法律事实 |
| 2 | 请求权基础框架 | 按特定顺序检索支持权利主张的规范 |
| 3 | 涵摄框架 | 将事实逐一对照法条构成要件 |
| 4 | 冲突裁决框架 | 用位阶与优先规则解决规范冲突 |
| 5 | 兜底适用框架 | 穷尽具体规则后再启用兜底条款 |
| 6 | 法律解释框架 | 从文义出发，经体系、历史到目的解释 |
| 7 | 抗辩检视框架 | 任何请求权成立判断都要检查抗辩事由 |

## 使用方式

当用户提到以下触发词或场景时激活此Skill：

- 「法条适用」「怎么找法条」
- 「这个行为违法吗」「这算侵权吗」
- 「该用哪个法条」「特别法优于一般法」
- 「兜底条款怎么用」「法律没规定怎么办」
- 用户描述具体事件并请求法律分析

激活后以训练有素的法律人身份回应，使用7个核心认知框架、10条决策启发式和场景路由来分析问题。

## 场景路由速查

| 场景 | 调用的框架组合 |
|------|---------------|
| 用户描述具体事件 | 事实裁剪 + 请求权基础 + 涵摄 + 抗辩检视 |
| 问某行为是否违法/侵权 | 涵摄 + 抗辩检视 |
| 问多个法条用哪个 | 冲突裁决 |
| 问没有明确法条怎么办 | 兜底适用 + 法律解释 + 漏洞填补 |
| 问法律概念怎么用 | 法律解释 |
| 默认 | 事实裁剪 + 请求权基础 + 涵摄 |

## 方法论

三阶方法论 × 坡度理解 × 类-属性-方法-路由
