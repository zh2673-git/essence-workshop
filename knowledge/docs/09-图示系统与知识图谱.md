# 知本（KnowledgeRoot）- 图示系统与知识图谱设计

## 一、图示系统

### 设计原则

1. **图示是认知的脚手架，不是装饰**——每个图示必须帮助理解一个维度
2. **SVG内联，零依赖**——所有图示用SVG直接嵌入HTML，不依赖外部库
3. **按范式族匹配图示类型**——不同认知范式需要不同的视觉语言

### 图示类型与范式族映射

| 范式族 | 图示类型 | 适用维度 | 说明 |
|--------|---------|---------|------|
| 形式科学族 | **数轴/坐标系** | 定义锚点 | 展示概念在数轴上的位置和边界 |
| 形式科学族 | **逻辑困境图** | 现实矛盾 | 展示"没有这个概念会怎样"的逻辑断裂 |
| 形式科学族 | **实验示意图** | 应用场景 | 展示可动手验证的实验步骤 |
| 形式科学族 | **单位关系图** | 思维延伸 | 展示上下级单位的换算关系 |
| 人文学科族 | **时间线** | 历史溯源 | 4W时间锚的可视化 |
| 人文学科族 | **情感光谱** | 定义锚点 | 展示情感/手法的位置和边界 |
| 人文学科族 | **表达困境图** | 现实矛盾 | 展示"没有这个表达会怎样" |
| 人文学科族 | **手法谱系图** | 知识网络 | 展示手法之间的继承和对比关系 |
| 生命科学族 | **系统循环图** | 应用场景 | 展示生命/自然系统的循环关系 |
| 生命科学族 | **层级结构图** | 思维延伸 | 从个体到系统的层级关系 |

### 图示生成规则

图示由大模型在生成内容时按需创建，规则如下：

1. **定义锚点维度**：形式科学族必须生成数轴/坐标系图；人文学科族必须生成情感光谱/手法定位图
2. **历史溯源维度**：人文学科族必须生成时间线图（4W可视化）
3. **现实矛盾维度**：形式科学族必须生成逻辑困境图；人文学科族必须生成表达困境图
4. **其他维度**：按需生成，不强制

### SVG图示规范

```css
/* 图示通用样式 */
.diagram {
  background: var(--bg-secondary);
  border: 1px solid var(--border-color);
  border-radius: var(--radius);
  padding: 20px;
  margin: 16px 0;
  text-align: center;
}
.diagram svg {
  max-width: 100%;
  height: auto;
}
.diagram .caption {
  font-size: 0.8rem;
  color: var(--text-muted);
  margin-top: 8px;
}
```

SVG配色规则：
- 使用CSS变量 `var(--accent-color)` 作为强调色
- 背景透明，与卡片背景融合
- 文字用 `var(--text-primary)` 和 `var(--text-secondary)`
- 连线/箭头用 `var(--text-muted)`

## 二、知识图谱与递归生成

### 设计原则

1. **知识图谱是网状结构，不是树状结构**——知识点之间有前导/后续/平行/跨学科多种关系
2. **递归生成是用户驱动的**——默认不自动递归，用户点击"生成"按钮后递归
3. **图谱数据持久化**——所有已生成的知识点信息存入JSON文件，避免重复生成

### 知识图谱数据结构

```json
{
  "meta": {
    "version": "2.0",
    "updated_at": "2026-06-14"
  },
  "nodes": {
    "负数": {
      "id": "negative-number",
      "name": "负数",
      "subject": "math",
      "phase": "初中",
      "grade": 7,
      "paradigm_family": "formal_science",
      "html_file": "output/负数.html",
      "generated_at": "2026-06-11"
    }
  },
  "edges": [
    {
      "from": "正数和0",
      "to": "负数",
      "type": "prerequisite",
      "label": "前导"
    },
    {
      "from": "负数",
      "to": "有理数",
      "type": "subsequent",
      "label": "后续"
    },
    {
      "from": "负数",
      "to": "绝对值",
      "type": "parallel",
      "label": "平行"
    }
  ]
}
```

**phase 规则**：grade 1-6 → "小学"，grade 7-9 → "初中"，grade 10-12 → "高中"。

**知识域划分**：以 phase + subject 为第一层划分（如"初中/数学"），grade 用于学段内排序。项目节点（type=project）作为跨学科聚合层，通过 contains 边引用不同知识域的知识点。

**全局视图支持两种视角**：
1. **知识域视角**（默认）：按 phase + subject 分区，区内知识点按依赖关系排列
2. **项目视角**：项目节点为中心，辐射到各知识域中被包含的知识点

### 递归生成流程

```
用户输入知识点A
    ↓
生成知识点A的七维拆解 + HTML
    ↓
提取知识网络维度中的关联知识点（前导/后续/平行）
    ↓
将知识点A和关联信息写入知识图谱JSON
    ↓
在HTML中渲染知识地图：
  - 已生成的节点 → 可点击跳转（链接到对应HTML）
  - 未生成的节点 → 显示"生成"按钮
    ↓
用户点击"生成"按钮 → 递归生成该知识点
    ↓
更新知识图谱JSON，更新所有相关HTML中的知识地图
```

### 递归深度控制

| 模式 | 行为 | 适用场景 |
|------|------|---------|
| **手动递归** | 用户点击"生成"按钮才递归 | 默认模式，避免失控 |
| **自动递归1层** | 自动生成直接前导和后续（1层） | 用户明确要求"生成完整知识链" |
| **自动递归N层** | 自动生成N层关联 | 需要用户指定深度，防止无限递归 |

### HTML中知识地图的交互设计

**已生成节点**：
```html
<a href="有理数.html" class="map-node">有理数</a>
```
点击跳转到对应HTML。

**未生成节点**：
```html
<span class="map-node pending" data-point="绝对值" data-subject="math" data-grade="5">
  绝对值 <button class="generate-btn" onclick="requestGenerate('绝对值','math',5)">生成</button>
</span>
```
点击"生成"按钮后，Skill接收请求并生成该知识点的HTML。

### 知识图谱全局视图

除了每个知识点HTML中的局部知识地图，还需要一个全局知识图谱页面：

- 文件：`output/knowledge-graph.html`
- 功能：展示所有已生成知识点的完整图谱
- 使用SVG绘制节点和连线
- 节点可点击跳转到对应HTML
- 按学科分色显示
