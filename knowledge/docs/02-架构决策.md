# 知本（KnowledgeRoot）- 架构决策

## 二、为什么（坡度三阶段）

### 阶段1：点出概念

> **存在理由一句话总结**：因为知识的认知过程被跳过了，学习者只得到结论却不知道结论从何而来，知本系统要还原这个被跳过的过程。

### 阶段2：建立联系

| 决策项 | 选择 | 推导逻辑 |
|-------|------|---------|
| **框架结构** | 主框架 + 范式族插件 + 学科微调 | 学科差异的本质是认知范式差异，3-4个范式族可覆盖所有学科，避免框架爆炸 |
| **内容生成** | 大模型 + 框架约束 + 质量校验 | 大模型有深度解释能力但输出不稳定，框架约束保证结构化，质量校验兜底 |
| **知识点粒度** | 课标最小单元 | 与教学实践对齐，教师可直接用于备课；太粗则无法深入，太细则碎片化 |
| **学段适配** | 范式族内嵌学段规则 | 学段差异是认知能力差异，与学科范式正交，可独立调整 |
| **呈现方式** | HTML交互优先 | 最灵活、最快验证、支持渐进式呈现和知识地图导航 |
| **技术栈** | Python + FastAPI + 大模型API | 快速迭代、类型安全、生态丰富 |
| **数据存储** | JSON文件（MVP）→ 数据库（扩展） | MVP阶段用文件存储降低复杂度，验证核心逻辑后再引入数据库 |
| **知识图谱** | 手动构建核心节点 → 半自动扩展 | 先手动构建验证价值，再引入算法辅助扩展 |

### 阶段3：详细解释

#### 架构设计理由

**1. 为什么用"主框架+范式族插件"而不是"每学科独立框架"？**

| 方案 | 规则数量 | 维护成本 | 跨学科支持 | 扩展性 |
|------|---------|---------|-----------|-------|
| 每学科独立框架 | 10+套 | 极高 | 差（无法处理跨学科知识点） | 差（新学科=新框架） |
| 主框架+范式族插件 | 1+3~4套 | 中 | 好（同族学科自然互通） | 好（新学科→归入已有族或新建族） |
| 纯主框架无插件 | 1套 | 低 | 无法处理学科差异 | 差 |

**2. 为什么七维框架归为三组？**

七维不是七个平行的维度，而是三个递进的认知任务：

```
认知组（理解）→ 应用组（迁移）→ 验证组（确认）
```

这个递进关系是认知心理学的基本规律：不理解就无法迁移，不迁移就无法确认。把它显性化，可以：
- 指导内容生成的顺序（先认知组，再应用组，最后验证组）
- 指导呈现的渐进展开（默认只展开认知组）
- 指导质量校验的优先级（认知组不通过则整体不通过）

**3. 为什么大模型+框架约束而不是纯大模型？**

| 方案 | 输出稳定性 | 内容深度 | 可控性 |
|------|-----------|---------|-------|
| 纯大模型 | 差（风格飘移、维度遗漏） | 取决于prompt | 差 |
| 大模型+框架约束 | 好（结构固定、规则明确） | 有保障（框架保证维度完整） | 好 |
| 纯模板 | 极好 | 差（模板无法处理知识点差异） | 极好 |

框架约束的本质是：**告诉大模型"必须回答哪些问题"和"必须用什么方式回答"**，而不是"必须说什么"。

**4. 为什么MVP阶段选数学学科？**

- 数学是形式科学族的代表，范式规则最清晰
- 数学的"现实矛盾"维度最容易设计（逻辑困境）
- 数学的"验证理解"维度最容易区分本质理解和计算能力
- 数学的知识图谱结构最清晰（前导/后续关系明确）

#### 设计模式选择

**插件模式（Plugin Pattern）**：主框架定义接口，范式族插件实现接口。

```
主框架（接口定义）
  ├── 范式族插件A（形式科学族）── 实现接口
  ├── 范式族插件B（人文学科族）── 实现接口
  └── 范式族插件C（生命科学族）── 实现接口
```

选择理由：
1. **开闭原则**：新增学科不需要修改主框架，只需归入已有范式族或新建范式族
2. **单一职责**：主框架只管"拆解什么"，范式族插件只管"怎么拆解"
3. **可替换性**：同一知识点可以用不同范式族插件拆解（如跨学科知识点），对比效果

**策略模式（Strategy Pattern）**：学段适配作为策略，与范式族插件正交组合。

```
范式族插件 × 学段策略 = 最终生成规则

形式科学族 × 低年级 = 数学低年级规则
形式科学族 × 高年级 = 数学高年级规则
人文学科族 × 低年级 = 语文低年级规则
人文学科族 × 高年级 = 语文高年级规则
```

选择理由：学段和学科是两个正交维度，用策略模式可以独立变化，避免组合爆炸。
