# 项目分析框架

逆向三阶合一——从成品反推本质，方法固定，执行灵活：

```
实现观察（怎么做·逆向入口）
  │ 方法：浏览源码结构、识别模块划分、分析方法实现
  │       标记 ✅需深入 的模块供递归
  │ 执行：读取范围、模块识别粒度 → 按项目规模决定
  ↓
设计推导（为什么·反推因果）
  │ 方法：从源码反推技术选型原因；从结构反推架构设计思想
  │       提取反常识根因（至少1个反直觉设计点）
  │ 执行：推导深度、反常识根因数量 → 按代码可读性决定
  ↓
本质反推（是什么·反推本质）
  │ 方法：反推本质定义（揭底，不是描述外延）
  │       通过本质检验（反推出的本质能否解释所有观察到的属性/方法/路由）
  │       结尾一句话点破（最简洁的本质表达，可独立传播）
  │ 执行：本质表达方式、类归属命名 → 按项目特性决定
  ↓
递归深入（标记模块）
  │ 方法：对 ✅需深入 的模块重做逆向三阶（只重做阶段1重新找子概念本质，
  │       不重复阶段2/3 完整流程，避免无限展开）
  │ 执行：递归层数上限、模块分析顺序 → 按项目复杂度决定
```

**思考层 vs 输出层分离**：逆向三阶是思考层脚手架（决定反推深度），产物按反推报告固有结构组织（实现观察→设计推导→本质反推→改进建议），不按三阶分节。

**硬性约束**：反推出的本质必须通过本质检验（能解释所有观察到的属性/方法/路由）；任意属性/方法/路由无法回溯到本质，说明反推不完整，需回到上游补充观察。
