# K12知识拆解框架

**K12七维框架**（本质定义先行 → 七维展开，每个维度必须能回溯到本质）：

```
定义锚点（升级为本质锚点）
  │ 方法：揭示本质属性，不是描述外延；通过本质检验
  ↓
溯源
  │ 方法：概念的历史起源 / 发明者 / 解决的原始问题
  ↓
矛盾
  │ 方法：概念诞生时面对的核心矛盾 / 认知冲突
  ↓
应用
  │ 方法：典型应用场景 + 常见误用
  ↓
延伸
  │ 方法：概念的边界、局限、与其他概念的关系
  ↓
网络
  │ 方法：概念在知识图谱中的位置（前置、平行、后续概念）
  ↓
验证
  │ 方法：通过例题/反例验证理解，迁移到新场景
```

**三种模式**（框架结构，具体编排由工作流决定）：

| 模式 | 产出 |
|------|------|
| 知识点模式 | 单个知识点的七维拆解 |
| 项目式模式 | 项目路径 + 多个知识点的七维拆解 |
| 讲课模式 | 单页跨知识点导航，支持自动演示 |
