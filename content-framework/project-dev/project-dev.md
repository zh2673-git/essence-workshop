# 项目开发框架

**正向设计要点**（三阶合一框架的项目开发应用）：

```
阶段1（是什么·点出本质）
  │ 方法：本质定义锚定类归属（不靠"项目=类"隐喻硬接）
  │       关键属性展开标记✅❌
  ↓
阶段2（为什么·建立因果）
  │ 方法：技术选型与架构决策由本质推导
  │       结构性比喻内部结构与本质同构
  ↓
阶段3（怎么做·展开方法·视角升级）
  │ 方法：模块划分 + 接口契约 + 四层设计 + 方法设计 + 路由设计
  │       可运行代码 + 验证契约
```

**两种执行模式**：

| 模式 | 流程 | 适用场景 |
|------|------|---------|
| 标准模式 | 设计→实现→交付 | 需求明确，一次交付 |
| 迭代模式 | 设计→实现→测试→评估→保留/回退→下一轮 | 需求渐进，测试驱动 |

**两种产品开发路径**：

| 路径 | 产物形态 | 核心设计增量 | 适用场景 |
|------|---------|-------------|---------|
| 纯工具产品 | 无 LLM 运行时的规则/代码/数据系统 | 数据结构、状态机、接口契约、P/Q/I 验证 | 需求明确、结构化、重复性高 |
| 协同产品 | 含 LLM 运行时 + 符号系统约束的系统 | 额外设计 LLM 接入层、符号校验层、Prompt 管理、失败降级 | 需求模糊、需自然语言理解、动态生成、创新性任务 |

两种路径与标准/迭代模式自由组合，形成 4 种子模式。迭代模式借鉴 Karpathy autoresearch 的 Ratchet Loop（棘轮循环）。
