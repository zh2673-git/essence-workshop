# 蒸馏框架

蒸馏是把人物/话题的认知系统提炼成可复用 Skill 的方法。从工作流提炼出的精髓框架，方法固定，执行灵活：

```
入口分流（定路径）
  │ 方法：判断明确对象（直接路径）还是模糊需求（诊断路径反推候选）
  │ 执行：追问轮数、候选数量 → 模型自行决定
  ↓
多维度采集（七Agent并行）
  │ 方法：7个维度并行采集——著作/对话/表达/他者/决策/时间线/根因
  │       根因维度是本质蒸馏独有（追溯思维形成的源头经历）
  │ 执行：信息源选择、本地语料 vs 网络搜索 → 按素材情况决定
  ↓
三阶合一提炼（调用认知层）
  │ 方法：思考层走认知层三阶合一（点出本质→建立因果→展开方法）
  │       输出层按蒸馏产物结构（心智模型+启发式+路由+反模式+few-shot）
  │ 执行：心智模型数量、路由场景 → 按人物/话题特性决定
  ↓
材料验证（保矛盾）
  │ 方法：保留矛盾不和稀泥；区分「他说过」vs「别人说他」vs「我推断」
  │       注明信息来源可信度（一手>二手>推测）
  │ 执行：矛盾处理方式、局限标注 → 按实际情况决定
  ↓
Skill 产出（自包含）
  │ 方法：产出自包含目录（SKILL.md + references/research + references/essence + sources）
  │       复制整个目录即可独立使用
  │ 执行：目录细节、附加文件 → 按蒸馏对象决定
```

### 入口分流（定路径）：判断走直接路径还是诊断路径

**方法固定**

- 明确人名/主题 → 直接路径：确认对象、聚焦方向、用途、新建 or 更新
- 模糊需求/困惑 → 诊断路径：从需求维度反推 2-3 个候选蒸馏对象

**执行灵活**

- 追问轮数、候选推荐数量按用户表达清晰度决定

### 多维度采集（七Agent并行）：7个维度并行采集

**方法固定**

- 7 个维度并行：著作/对话/表达/他者/决策/时间线/根因
- 根因维度是本质蒸馏独有：追溯教育背景、关键影响、重大挫折、文化土壤、思想转折点
- 每个 Agent 必须把调研结果写入对应文件，不存文件的调研等于没做

**执行灵活**

- 信息源选择（本地语料优先 vs 纯网络搜索）、Agent 超时处理 → 按素材情况决定

### 三阶合一提炼（调用认知层）：思考层走三阶，输出层按产物结构

**方法固定**

- 思考层：调用认知层三阶合一（阶段1点出本质→阶段2建立因果→阶段3展开方法·视角升级）
- 本质定义先行：阶段1抛本质定义（揭底，不是描述外延）并通过本质检验
- 输出层：SKILL.md 按「心智模型+决策启发式+路由+反模式+few-shot」组织，不按三阶分节

**执行灵活**

- 心智模型数量（3-7个）、路由场景划分 → 按人物/话题特性决定

### 材料验证（保矛盾）：保留矛盾，诚实标注局限

**方法固定**

- 保留矛盾，不和稀泥——矛盾本身是有价值的信号
- 区分「他说过的」vs「别人说他的」vs「我推断的」
- 注明信息来源可信度（一手>二手>推测）

**执行灵活**

- 矛盾处理方式、局限标注的详细程度 → 按实际情况决定

### Skill 产出（自包含）：产出可独立使用的自包含目录

**方法固定**

- 产出自包含目录：SKILL.md + references/research/（7个调研文件）+ references/essence/（三阶思考归档）+ sources/（一手素材）
- 复制整个目录即可独立使用

**执行灵活**

- 目录细节、附加文件 → 按蒸馏对象决定
