# 周易占卜框架 · 内容组织方法论

> **本框架源自「有疑再问」微信小程序的认知精华提炼**。
> 将小程序中耦合在 WXML/JS 里的占卜方法论与数据结构，抽象为不绑定形式的可复用框架。
> 框架可被任意工作流调用：浏览器交互占卜、公众号周易文章、K12国学课件、白板讲解视频等。

---

## 一、框架本质

周易占卜是一种**结构化的认知活动**，其内容组织遵循固定模式：

```
心法约束 → 问题输入 → 随机起卦 → 卦象计算 → 卦辞解释 → 历史沉淀
```

本框架定义上述每个环节的内容结构规范，不绑定具体执行形式（小程序/网页/文章）。

---

## 二、心法原则（认知约束）

### 不疑不问
- **原则**：只在真正有疑时卜问，不可游戏试探
- **节制**：每日建议不超过3次，过度预测影响准确性
- **诚心**：预测前静心，想清楚要咨询的问题
- **参考**：周易预测仅供参考，一切以自身理性判断为主

> 心法属于认知层「易经视角」的范畴，本框架仅记录其作为内容约束的接口规范。

## 三、内容结构规范

### 3.1 起卦流程结构

```
{
  question: string,              // 用户问题
  lines: number[6],              // 六爻值，从下往上
                                 // 6=老阴(变) 7=少阳 8=少阴 9=老阳(变)
  changingLines: number[],       // 变爻索引（0-5）
  original: { name, value },     // 本卦
  changed: { name, value }       // 变卦（无变爻时同本卦）
}
```

### 3.2 起卦方法：三硬币法

```
每次摇三枚硬币：
  阳面(1)计3，阴面(0)计2
  三硬币之和 ∈ {6, 7, 8, 9}
    6 = 老阴（变爻，阴→阳）
    7 = 少阳（不变）
    8 = 少阴（不变）
    9 = 老阳（变爻，阳→阴）

摇六次，从下往上累加，得六爻卦象。
```

### 3.3 卦象计算规则

```
本卦数值 = 二进制(爻值转换: 7/9→1, 6/8→0) → 十进制(0-63)
变卦数值 = 将变爻反转后再计算
卦名 = HEXAGRAM_NAMES[数值]
```

### 3.4 上下卦映射

```
下卦 = 第1-3爻（索引0-2）
上卦 = 第4-6爻（索引3-5）

八卦二进制映射（从下往上读）：
  000=坤  001=艮  010=坎  011=巽
  100=震  101=离  110=兑  111=乾

64卦 = 上卦 × 下卦 的组合查表
```

## 四、数据结构规范

### 4.1 八卦（TRIGRAMS）

| 二进制 | 索引 | 卦名 | 自然象 |
|--------|------|------|--------|
| 111 | 0 | 乾 | 天 |
| 110 | 1 | 兑 | 泽 |
| 101 | 2 | 离 | 火 |
| 100 | 3 | 震 | 雷 |
| 011 | 4 | 巽 | 风 |
| 010 | 5 | 坎 | 水 |
| 001 | 6 | 艮 | 山 |
| 000 | 7 | 坤 | 地 |

### 4.2 64卦结构（HEXAGRAM_NAMES）

64卦按索引0-63排列，索引由六爻二进制值决定（阳=1，阴=0，从下往上）。

完整卦名序列（索引0-63）：
```
乾 坤 屯 蒙 需 讼 师 比
小畜 履 泰 否 同人 大有 谦 豫
随 蛊 临 观 噬嗑 贲 剥 复
无妄 大畜 颐 大过 坎 离 咸 恒
遯 大壮 晋 明夷 家人 睽 蹇 解
损 益 夬 姤 萃 升 困 井
革 鼎 震 艮 渐 归妹 丰 旅
巽 兑 涣 节 中孚 小过 既济 未济
```

### 4.3 卦象查询表（HEXAGRAM_LOOKUP）

每卦的结构：
```
{
  '卦名': {
    upper: '上卦名',      // 八卦之一
    lower: '下卦名',      // 八卦之一
    index: number        // 0-63
  }
}
```

### 4.4 卦辞与解释（HEXAGRAM_TEXTS / HEXAGRAM_DESCRIPTIONS）

```
HEXAGRAM_TEXTS[卦名] = '卦辞原文'
HEXAGRAM_DESCRIPTIONS[卦名] = '卦象解释（含图象解）'
```

> 实质卦辞与解释内容由形式层在生成时注入（或由数据文件加载），框架仅定义结构。

## 五、历史记录结构

```
{
  id: number,            // 时间戳ID
  date: 'YYYY-MM-DD',
  question: string,
  result: {
    original: { name, value },
    changed: { name, value },
    changingLines: number[],
    hexagramText: string,
    lineTexts: string[]
  },
  timestamp: number
}
```

存储：本地持久化（小程序用 wx.setStorageSync，网页用 localStorage）

## 七、与其他层的接口

- **上游（认知层）**：调用 `cognitive/yijing-perspective/` 提供"不疑不问"心法与取象视角
- **下游（形式层）**：
  - HTML形式 → 交互式占卜网页
  - Markdown形式 → 周易解读文章
  - 白板形式 → 64卦讲解白板
  - Notebook形式 → 周易学习笔记本
- **调用方（工作流层）**：
  - `workflow/iching-divination/` → 浏览器交互占卜
  - `workflow/wechat-publish/` → 公众号周易文章
  - `workflow/k12-decomposition/` → 国学课件
  - `workflow/whiteboard-recorder/` → 卦象讲解视频

---

## 八、设计原则

1. **框架不含实质卦辞内容**：64卦的具体卦辞、解释由数据文件或形式层注入
2. **框架不绑定形式**：同一框架可输出为小程序、网页、文章、视频
3. **框架不绑定平台**：不感知微信/浏览器/公众号等平台差异
4. **心法优先**：占卜内容必须先遵守"不疑不问"心法约束
