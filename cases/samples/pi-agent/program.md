# PI-Agent 迭代策略

> 本文件是迭代模式的核心入口，由人类编写，Agent读取执行。

## 目标

基于PI极简架构理念，使用Ollama本地小模型构建一个"增强型极简"编码Agent，通过Ratchet Loop迭代验证小模型能力边界并确定最优架构。

## 验证契约

> 三维全过才保留，任一失败则回退。

### 前置条件 P（构建前必须为真）

| 条件 | 含义 | 验证方式 |
|------|------|---------|
| Ollama运行 | Ollama服务可用 | `ollama list` 返回模型列表 |
| 模型可用 | 目标模型已拉取 | `ollama list` 包含目标模型 |
| 依赖安装 | Python依赖已安装 | `pip list` 包含ollama |
| 测试可运行 | 测试脚本本身无bug | `pytest tests/ --collect-only` |

### 后置条件 Q（构建后必须为真）

| 条件 | 含义 | 验证方式 | 目标值 |
|------|------|---------|-------|
| 功能正确 | 新功能按预期工作 | 测试用例通过 | 全部通过 |
| 工具调用正确率 | 模型正确调用工具的比例 | 100次调用统计 | >90% |
| 任务完成率 | 简单编码任务完成比例 | 10个标准任务 | >70% |
| Agent Loop效率 | 平均每任务工具调用次数 | 10个标准任务 | <8步 |

### 不变量 I（全程必须保持为真）

| 条件 | 含义 | 验证方式 |
|------|------|---------|
| 旧功能不坏 | 已有测试仍然通过 | 回归测试 |
| 接口不破 | Agent/ToolExecutor/Config接口不变 | 接口兼容性检查 |
| 三层架构不变 | llm/core/tools分层不合并 | 目录结构检查 |

### 不可验证项

| 功能 | 为什么不可验证 | 替代验证方式 |
|------|--------------|------------|
| 代码质量 | 无自动化质量标准 | 人工review |
| 提示词效果 | 依赖模型随机性 | 多次运行取平均 |

### 判定规则

| P | Q | I | 决策 |
|---|---|---|------|
| ❌ | - | - | 阻断，修复前置条件 |
| ✅ | ✅ | ✅ | 保留（commit） |
| ✅ | ✅ | ❌ | 回退（最危险：退步被误判为改进） |
| ✅ | ❌ | ✅ | 回退（改动无效） |
| ✅ | ❌ | ❌ | 回退（双重退步） |

## 修改范围

### 可以修改

- `src/pi_agent/` 目录下的所有文件
- `tests/` 目录下的测试文件

### 不能修改

- `core/` 目录（基础设施）
- `program.md`（本文件，只有人类可以修改）
- `pyproject.toml`（项目依赖）

## 约束条件

1. 单次测试执行时间不超过120秒
2. 不变量I必须每轮检查，不可跳过
3. 每次只改一个维度，便于归因
4. 代码总行数不超过2000行
5. 使用Ollama原生API（/api/chat），不用OpenAI兼容API（避免thinking模式问题）

## 研究方向

### 优先级1（当前聚焦）

- 最小Agent验证：LLM + 2工具(read+bash) + while循环能否跑通
- 渐进式工具暴露：2工具→3工具→4工具

### 优先级2（备选）

- 提示词增强：few-shot示例、错误恢复指令
- 模型对比：qwen3:4b vs qwen3:8b vs gemma3:12b

### 优先级3（探索性）

- 总结轮兜底：工具调用后追加总结轮
- 安全层：命令白名单/黑名单
