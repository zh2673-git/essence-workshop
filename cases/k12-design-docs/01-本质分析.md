# 知本（KnowledgeRoot）- 本质分析

## 一、是什么（坡度三阶段）

### 阶段1：点出概念

> **一句话定义**：知本是一个认知脚手架系统——把知识点从"结论"还原为"认知过程"，让学习者重走发现之路。

**本质特征**：

| 推导要素 | 逻辑分析 | 结论 |
|---------|---------|------|
| **核心问题** | 学习者"知道"但"不理解"，会做题但不会迁移；老师想讲透但备课成本高 | 知识的"认知过程"被跳过了，只留下了"结论" |
| **目标用户** | 学生（K12）、教师（备课） | 学生是终端受益者，教师是核心生产者 |
| **核心价值** | 让理解可抵达、可迁移、可验证 | 不是替代教学，而是还原被跳过的认知过程 |
| **AI的角色** | 大模型有能力做深度解释，但输出不稳定 | AI是"认知还原引擎"，框架是"稳定器" |

### 阶段2：建立联系

| 属性名 | 推导来源 | 类型 | 说明 | 深入？ |
|-------|---------|-----|------|-------|
| **拆解框架** | "认知过程被跳过"→需要结构化还原 | 核心属性 | 七维拆解框架，把知识点的认知过程结构化 | ✅ |
| **学科插件** | 不同学科的认知方式不同 | 核心属性 | 每个学科有自己的认知范式，影响拆解规则 | ✅ |
| **学段适配** | 不同年龄的认知能力不同 | 核心属性 | 同一知识点，低年级和高年级的抵达方式不同 | ✅ |
| **内容生成** | 需要借助大模型能力 | 方法属性 | 大模型+框架约束=稳定高质量输出 | ✅ |
| **知识图谱** | 知识点不是孤立的 | 结构属性 | 前导/后续/跨学科关联 | ✅ |
| **呈现管线** | 内容需要多形态输出 | 方法属性 | HTML交互/视频/PPT等 | ❌ |
| **用户系统** | 需要区分教师和学生 | 基础属性 | 身份、权限、进度 | ❌ |

### 阶段3：详细解释

#### 区分属性（边界）

- ✅ 做：知识点的认知过程还原、结构化拆解、学科差异化规则、学段适配、本质理解型验证
- ❌ 不做：题库和刷题系统、在线课堂、作业批改、成绩管理、学习社区

#### 设计哲学

**还原认知，而非传递结论**——系统的价值不在于"告诉学生答案"，而在于"让学生看到答案是怎么被发现的"。

## 关键问题：是否需要对所有学科建立框架？

### 深入分析

**答案：不需要对所有学科建立独立框架，但需要识别学科的"认知范式族"。**

#### 为什么不需要每个学科都建独立框架？

1. **认知范式有共性**：数学和物理都是"逻辑推导+实验验证"的混合范式，语文和历史的"情感体验+时空定位"有交叉
2. **框架数量爆炸**：K12涉及10+学科，每个学科独立框架=10+套规则，维护成本极高
3. **跨学科知识点**：同一个知识点可能横跨两个学科（如"惯性"既是物理概念也是生活常识），独立框架无法处理

#### 为什么需要"认知范式族"？

不同学科的认知方式确实不同，但差异不在"学科"层面，而在"认知范式"层面：

```
学科 → 认知范式 → 范式族

数学 → 逻辑推导范式 ──┐
物理 → 实验验证范式 ──┤→ 形式科学族（逻辑+验证）
化学 → 物质转化范式 ──┘

语文 → 情感体验范式 ──┐
历史 → 时空定位范式 ──┤→ 人文学科族（体验+定位）
英语 → 交际功能范式 ──┘

生物 → 生命系统范式 ──→ 生命科学族（系统+进化）
```

#### 具体方案：3层架构

| 层级 | 内容 | 数量 | 说明 |
|------|------|------|------|
| **主框架** | 七维拆解框架（通用） | 1套 | 所有学科共用，定义"拆解什么" |
| **范式族插件** | 认知范式族的差异化规则 | 3-4套 | 定义"怎么拆解"，覆盖所有学科 |
| **学科微调** | 学科特有的微约束 | 按需 | 只在范式族规则不够时补充 |

#### 范式族划分

**族1：形式科学族（数学、物理、化学）**

| 维度 | 族规则 | 学科微调 |
|------|--------|---------|
| 定义锚点 | 用可操作的对象锚定概念 | 数学：生活物品；物理：可观察现象；化学：可感知变化 |
| 历史溯源 | 讲关键发现者的逻辑链条 | 数学：公理→定理；物理：假设→实验→定律；化学：观察→假设→验证 |
| 现实矛盾 | 展示"没有这个概念会怎样"的逻辑困境 | 数学：逻辑不可能；物理：直觉与实验冲突；化学：宏观与微观矛盾 |
| 应用场景 | 可动手验证的场景 | 数学：可操作任务；物理：可观察实验；化学：可感知变化 |
| 验证理解 | 本质理解型题目 | 数学：辨析+选择；物理：实验设计；化学：现象解释 |

**族2：人文学科族（语文、历史、英语）**

| 维度 | 族规则 | 学科微调 |
|------|--------|---------|
| 定义锚点 | 用可体验的感受锚定概念 | 语文：情感共鸣；历史：时空代入；英语：交际场景 |
| 历史溯源 | 讲创作者的生命处境+时代背景 | 语文：作者处境+历史背景（含4W时间锚）；历史：时代背景+因果链条；英语：语言使用的真实场景 |
| 现实矛盾 | 展示"没有这个表达会怎样"的表达困境 | 语文：情感的不可言说；历史：历史的必然与偶然；英语：母语思维的干扰 |
| 应用场景 | 可体验的表达场景 | 语文：仿写/改写；历史：类比当下；英语：角色扮演 |
| 验证理解 | 体验理解型题目 | 语文：感受辨析；历史：因果推理；英语：交际选择 |

**族3：生命科学族（生物、地理）**

| 维度 | 族规则 | 学科微调 |
|------|--------|---------|
| 定义锚点 | 用可观察的生命/自然现象锚定 | 生物：结构→功能；地理：位置→特征 |
| 历史溯源 | 讲发现者的观察过程 | 生物：观察→假设→验证；地理：探索→发现→解释 |
| 现实矛盾 | 展示"表象与本质的差异" | 生物：看似简单实则复杂；地理：看似偶然实则必然 |
| 应用场景 | 可观察的自然现象 | 生物：身边的生命；地理：脚下的土地 |
| 验证理解 | 观察理解型题目 | 生物：现象解释；地理：特征推理 |

#### 学科微调的判断规则

```
知识点输入 → 判断学科 → 加载范式族插件
    ↓
检查：范式族规则是否足够覆盖该知识点的拆解？
    ↓
足够 → 直接生成
不足够 → 加载学科微调规则 → 合并后生成
```

**什么情况下需要学科微调？**

1. 该学科有范式族规则未覆盖的认知特征（如英语的"母语干扰"是独有的）
2. 该学科的验证方式有特殊要求（如化学需要"实验安全"约束）
3. 该学科有独有的微框架（如历史的4W时间锚框架）

#### 结论

| 问题 | 答案 |
|------|------|
| 需要对所有学科建独立框架吗？ | **不需要** |
| 需要多少套规则？ | **1套主框架 + 3-4套范式族插件 + 按需学科微调** |
| 哪些学科需要微调？ | 英语（交际功能独特）、化学（实验安全约束）、地理（空间认知独特） |
| 哪些学科直接用范式族？ | 数学、物理（形式科学族）；语文、历史（人文学科族）；生物（生命科学族） |

## 本质特征总结

知本系统的本质是**认知还原**——不是教知识，而是还原知识被发现的认知过程。这个本质决定了：

1. **框架是核心**：七维框架定义了"还原什么"，范式族插件定义了"怎么还原"
2. **AI是引擎**：大模型提供还原的能力，框架约束提供还原的稳定性
3. **学科差异是认知范式的差异**：不是规则的差异，而是"认识世界的方式"的差异
4. **学段差异是认知能力的差异**：不是语言的差异，而是"能抵达多深"的差异
