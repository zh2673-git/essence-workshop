# 实战案例：PI-Agent 从0到1迭代开发

> 本案例展示了如何用本质工坊的场景C2（迭代开发/Ratchet Loop），从零构建一个基于PI极简架构的本地小模型编码Agent。
> 完整6轮迭代，0次回退，100%保留率。

## 项目背景

### 是什么

PI (Prompt Interface) 是一个极简编码Agent框架，核心理念：Agent = LLM + 4个工具 + while循环。系统提示词不到1000 token，却能在Terminal-Bench 2.0基准测试中达到与Claude Code相当的性能。

### 核心矛盾

PI的极简设计**依赖强模型能力**，而本地小模型恰恰**缺乏这种能力**。需要验证：小模型能否驱动一个实用的编码Agent？

### 策略

**增强型极简**：保持架构简洁，但增强引导和辅助。

## 验证合同

| 维度 | 内容 |
|------|------|
| **P（前置）** | Ollama运行 + 模型可用 + 依赖安装 + 测试可运行 |
| **Q（后置）** | 功能正确 + 工具调用正确率>90% + 任务完成率>70% + 平均<8步 |
| **I（不变量）** | 旧功能不坏 + 接口不破 + 三层架构不变 |

## 6轮迭代实录

### v1: 最小Agent验证 — 3/3 通过

**改动**：minimal profile (read+bash) + Ollama原生API + 总结轮兜底

**关键发现**：
- 模型先尝试`ls`（Linux命令），失败后自动切换到`python -c` — 错误恢复能力好
- read工具调用完全正确
- 直接回答问题不调用工具 — 正确

**Ratchet决策**：P✅ Q✅(3/3) I✅ → **保留**

---

### v2: standard profile — 4/4 通过

**改动**：增加write工具

**关键发现**：write工具调用精确，参数正确；回归测试通过

**Ratchet决策**：P✅ Q✅(4/4) I✅ → **保留**

---

### v3: full profile + edit模糊匹配 — 4/4 通过

**改动**：增加edit工具 + 模糊匹配提示

**关键发现**：
- edit工具对小模型很难 — 模型生成的old_string缩进/内容与文件不一致
- **Windows换行符`\r\n`**导致匹配失败
- 模型展现了极强的错误恢复能力：edit失败后降级到write重写文件

**修复**：edit工具增加三级匹配策略（精确→去空白→单行模糊）

**Ratchet决策**：P✅ Q✅(4/4) I✅ → **保留**

---

### v4: 模型对比 — 8/8 通过

**改动**：qwen3.5:4b vs qwen3.5:9b 对比测试

| 模型 | 通过率 | 总耗时 |
|------|--------|--------|
| qwen3.5:4b | 4/4 (100%) | 205s |
| qwen3.5:9b | 4/4 (100%) | 360s |

**意外发现**：4b模型比9b更快！9b生成了更长的thinking内容，导致超时风险更高。

**Ratchet决策**：P✅ Q✅(8/8) I✅ → **保留**

---

### v5: 复杂多步任务 — 4/4 通过, 913s

**改动**：测试复杂编码任务（创建模块+使用+修复bug+理解代码）

**关键发现**：913s太慢；Windows兼容性差；edit仍失败

**Ratchet决策**：P✅ Q✅(4/4) I✅ → **保留**（需优化）

---

### v6: Windows兼容性+提示词优化 — 4/4 通过, 258s

**改动**：系统提示词增加Windows环境说明 + edit使用指导

| 指标 | v5 | v6 | 改进 |
|------|----|----|------|
| 总耗时 | 913s | 258s | **3.5x加速** |
| edit修复 | edit失败→write降级 | edit成功 | 提示词优化✅ |

**核心洞察**：对小模型，提示词中的环境说明比few-shot示例更有效！

**Ratchet决策**：P✅ Q✅(4/4) I✅ → **保留**

## 迭代总结

6轮迭代，0次回退，100%保留率

## 核心洞察

1. Agent的本质很简单：LLM + 工具 + 循环
2. 小模型需要"增强型极简"：保持架构简洁，增强引导
3. 环境说明比few-shot更有效：明确告诉模型运行环境比让它试错快3.5倍
4. Ratchet Loop保证前进：三维验证确保每一步都是改进
