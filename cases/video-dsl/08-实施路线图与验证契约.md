## 2. 验证契约（Hoare 逻辑）

### 前置条件 P

| 条件 | 含义 | 验证方式 |
|------|------|---------|
| Node.js 18+ | 运行 Vite 和 renderer | `node --version` |
| Playwright + Chromium | 无头浏览器截图 | `npx playwright install chromium` |
| FFmpeg | 视频编码 | `ffmpeg -version` |
| TypeScript 编译器 | 校验 Agent 代码 | `tsc --noEmit` |
| AgentClient 可用 | 任意 LLM Agent 可调用 | mock 测试 |

### 后置条件 Q

| 条件 | 目标值 | 验证方式 |
|------|-------|---------|
| DSL 管线输出 MP4 | 文件存在且大小 > 0 | `os.path.getsize()` |
| 视频时长正确 | 误差 < 100ms | `ffprobe` |
| 视频分辨率正确 | 与 spec 一致 | `ffprobe` |
| 关键帧与代码一致 | 人工抽检 5 帧 | 像素对比 |
| 音画同步 | 误差 < 200ms | 人工/自动对齐检测 |
| 不破坏现有管线 | 现有测试用例全部通过 | 回归测试 |

### 不变量 I

| 条件 | 含义 | 验证方式 |
|------|------|---------|
| 现有公众号管线功能不变 | 微信文章发布仍可用 | 回归测试 |
| 现有 HTML 录制管线功能不变 | `html_to_video.py` 仍可用 | 回归测试 |
| 现有 article_to_video 功能不变 | 公众号文章转视频仍可用 | 回归测试 |
| Agent Prompt 契约向后兼容 | 旧 Spec 仍能生成合法代码 | Schema 测试 |
| 用户源码不被修改 | 渲染只操作输出目录 | 目录隔离检查 |

## 3. 失败模式与处理

| 步骤 | 失败场景 | 处理动作 |
|------|---------|---------|
| 环境检查 | Node.js / FFmpeg / Chrome 缺失 | 前置检查失败，输出安装指南，终止流程 |
| Spec 生成 | 内容无法转换为 VideoGenerationSpec | 降级到 HTML 录制或 article_to_video |
| Agent 生成代码 | 输出不符合 DSL 契约 | 返回错误给 Agent，最多重试 3 次 |
| Agent 生成代码 | 使用 CSS animation / requestAnimationFrame | 静态扫描拦截，提示改用 frame 驱动 |
| 代码编译 | TypeScript 编译失败 | 返回编译错误给 Agent，要求修复 |
| 资源收集 | `staticFile` 引用的文件不存在 | renderer 前置校验，缺失则报错 |
| 渲染阶段 | Playwright 截图超时 | 单帧设置超时，失败跳过并记录 |
| 渲染阶段 | 帧序列不连续 | 校验 PNG 数量，缺失则重试 |
| FFmpeg 合成 | 音频编码不兼容 | 先重编码音频再合并 |
| 长视频 | 渲染时间超长 | MVP 限制 ≤ 90 秒，超出提示分段 |
| 路由决策 | 决策 confidence 低 | 必须提供 fallback，并记录日志 |

## 4. 验收标准

### MVP 验收

- [ ] `examples/negative-numbers/` 能一键生成 30 秒竖屏 MP4。
- [ ] 生成的 `VideoProgram.tsx` 不依赖具体 Agent，可被任意 LLM 复现。
- [ ] 同一 Spec 重复生成 3 次，输出视频像素级一致。
- [ ] 不破坏现有 `content-output` 测试。

### Phase 2 验收

- [ ] `python -m scripts.cli video --pipeline dsl` 能正常调用。
- [ ] 路由能根据内容特征自动选择 DSL / HTML / article_to_video。
- [ ] TTS 旁白与画面时间对齐误差 < 200ms。
- [ ] 支持 BGM 自动 ducking。

### Phase 3 验收

- [ ] 组件库包含 ≥ 5 个常用动画组件。
- [ ] Agent 生成代码时引用组件库的比例 ≥ 60%。
- [ ] 支持 GIF 输出。
