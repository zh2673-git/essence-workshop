# 三阶合一在寓言中的体现与跨场景调用规范

> **核心定位（v2）**：概念寓言化是三阶合一"阶段2·建立因果"的一种高具象载体，可作为元素层原子资产被多个场景复用。寓言横跨三阶——开头抛本质（阶段1）→中段建立反差（阶段2反常识根因）→结尾视角升级（阶段3）。

---

## 一、三阶合一在寓言中的体现

```
阶段1：点出本质      ← 本质定义先行 + 本质检验 + 一句话点破（寓言开头的隐性本质呈现）
阶段2：建立因果      ← 【寓言在此】用叙事把概念嫁接到熟悉情境 + 反常识根因（寓言中段的反差/反转）
阶段3：展开方法·视角升级 ← 映射表（回溯到本质）+ 迁移问题 + 视角升级（寓言寓意收尾）
```

寓言不是只在阶段2工作，而是横跨三阶：
- **寓言开头**：用故事场景隐性呈现阶段1的本质定义（不是直接说出本质定义，而是让本质在情境中显现）
- **寓言中段**：通过反差/反转触发阶段2的反常识根因，制造认知冲突
- **寓言结尾**：通过寓意收尾实现阶段3的视角升级，重新定义问题

### 寓言 vs 其他"建立因果"载体

| 载体 | 具象度 | 适用受众 | 认知负荷 | 适用场景 |
|------|--------|---------|---------|---------|
| 逻辑类比表格 | 中 | 同行/专业人士 | 低 | 知识探索、项目解析 |
| 生活案例 | 高 | 外行/初学者 | 中 | 内容输出、K12拆解 |
| **寓言故事** | **最高** | **外行/儿童/全受众** | **中** | **全场景通用** |
| 图示/动画 | 高 | 视觉学习者 | 低 | K12拆解、HTML交互 |

**判断规则**：
- 受众是同行 → 用逻辑类比表格即可，不必寓言化
- 受众是外行/儿童 → 优先用寓言
- 受众混合 → 寓言引入 + 类比表格补充

## 二、跨场景调用协议

### 调用接口

所有场景通过统一接口调用概念寓言化：

```
调用方传入：
  concept: 概念名
  audience: 受众类型（child/layman/beginner/expert）
  length: 篇幅偏好（short/medium/long，可选）

概念寓言化返回：
  essence: 本质定义（阶段1·含本质检验和一句话点破）
  fable: 寓言正文（含点破句，阶段2·反差/反转对应反常识根因）
  mapping: 寓言→概念映射表（必须能回溯到本质定义）
  questions: 2个迁移问题
  perspective_shift: 视角升级一句话（阶段3·重新定义问题）
  element: 元素层封装格式（纯文本 + JSON）
```

### 各场景调用方式

#### 场景A：知识探索

```
A的工作流阶段2"建立因果"中，原本用类比表格。
当受众判断为外行时，替换为：
  阶段2：建立因果 = 调用 fable-concept(concept, audience=layman)
  → 寓言替代类比表格，寓言的视角升级收尾与A的阶段3视角升级合并
```

**效果**：降低外行认知门槛，寓言的视角升级直接成为A的阶段3收尾。同行受众不调用。

#### 场景A2：K12知识拆解

```
A2生成的交互式HTML中，增加"寓言引入"模块：
  在知识图谱节点页面顶部插入：
    <section class="fable-intro">
      {fable-concept 返回的 essence + fable}
    </section>
  在交互练习区插入迁移问题 + 视角升级句
```

**效果**：儿童通过故事先建立直觉（阶段1本质 + 阶段2反差），再进入七维框架拆解，理解深度提升。

#### 场景B：蒸馏

```
B蒸馏出的认知操作系统Skill中，表达DNA部分增加"寓言接口"：
  当用户说"用寓言讲讲XX的XX思维"时：
    1. 从蒸馏Skill提取核心心智模型（作为阶段1本质定义的输入）
    2. 调用 fable-concept(model_name, audience=layman)
    3. 用此人的表达DNA改写寓言语气
    4. 视角升级句与该人物认知风格对齐
```

**效果**：蒸馏Skill获得"用故事解释自己思维"的能力，增强传播性。

#### 场景E：内容输出

```
E的"故事风格"文章生成时：
  开头钩子 = fable-concept(concept, audience=layman, length=short)
  → 寓言作为文章开头，后续接观点论证
  → 寓言的视角升级句可作为文章结尾的金句
  → 同一寓言可被公众号/视频号/PPT管线复用
```

**效果**：故事风格文章的开头不再需要人工构思，寓言的视角升级可直接作为文章结尾金句，寓言元素可跨管线复用。

#### 场景E2：连载小说

```
E2的世界观构建时，为每个"概念节点"预生成寓言：
  for each 概念节点 in 世界观.概念图谱:
    fable = fable-concept(概念, audience=layman)
    → 存入章节大纲作为本章情节内核（essence + fable + perspective_shift）
  → 写章节时，角色冒险情节围绕寓言冲突（反常识根因）展开
  → 章节结尾用视角升级句作为主题升华
```

**效果**：连载小说的每章都有概念内核，叙事与知识传递合一，章节结尾有视角升级的主题升华。

## 三、组合流价值评估

### 高价值组合流（推荐优先实现）

| 组合流 | 路径 | 价值 | 复用度 |
|--------|------|------|--------|
| **探索→寓言→输出** | A(探索概念) → G(寓言化) → E(公众号/视频) | 一次探索，故事化输出多平台 | ★★★★★ |
| **K12拆解→寓言→HTML** | A2(拆解) → G(寓言引入) → HTML交互 | 儿童先听故事再学知识 | ★★★★☆ |
| **蒸馏→寓言→输出** | B(蒸馏人物) → G(寓言化其思维) → E(输出) | 用故事传播人物思维模型 | ★★★★☆ |
| **连载小说←寓言内核** | G(预生成概念寓言) → E2(章节情节) | 每章一个概念内核 | ★★★☆☆ |

### 中等价值组合流

| 组合流 | 路径 | 价值 | 限制 |
|--------|------|------|------|
| 项目解析→寓言 | D(解析) → G(寓言化架构理念) → E(输出) | 向非技术成员解释架构 | 仅需时调用 |
| 迭代开发→寓言 | C2(迭代) → G(寓言化Ratchet Loop) | 团队对齐方法论 | 一次性 |

### 不推荐组合流

| 组合流 | 原因 |
|--------|------|
| Skill优化→寓言 | F是元操作，寓言化优化原则无实际价值 |
| 项目开发→全程寓言 | C需要精确技术文档，寓言会降低准确性 |

## 四、元素层封装格式

概念寓言化输出必须封装为两种格式，供管线层调用。两种格式都包含本质定义（阶段1）、寓言正文（阶段2）、映射表（回溯到本质）、迁移问题、视角升级（阶段3）。

### 纯文本格式（公众号/视频/PPT用）

```markdown
## 寓言：{标题}

{寓言正文}

> 这个故事讲的，其实是 **{概念}**。

**本质定义**：{一句话点破最底层逻辑，揭示本质属性，能解释所有其他属性}

**思考题**：
1. {理解检验问题}
2. {跨域迁移问题}

**视角升级**：{重新定义问题，不是总结——给出比阶段1更高的视角}
```

### JSON格式（HTML交互/Notebook/程序调用用）

```json
{
  "element_type": "fable",
  "concept": "{概念名}",
  "audience": "{受众类型}",
  "title": "{寓言标题}",
  "essence": "{本质定义（揭示本质属性，能解释所有其他属性）}",
  "body": "{寓言正文}",
  "reveal": "这个故事讲的，其实是{概念}",
  "mapping": [
    {"fable_element": "...", "concept_element": "...", "essence_trace": "回溯到本质定义的路径"}
  ],
  "questions": {
    "comprehension": "{理解检验问题}",
    "transfer": "{跨域迁移问题}"
  },
  "perspective_shift": "{视角升级一句话，重新定义问题，不是总结}"
}
```
