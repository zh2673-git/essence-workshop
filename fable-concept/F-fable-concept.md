# 场景G：概念寓言化

> **v2.1 思考层 vs 输出层分离原则**（详见 methodology.md 第九章）：
> - **思考层（脚手架，不强制露出产物）**：智能体构思寓言时走 v2 三阶合一——开头呼应本质（阶段1）→中段建立反差（阶段2反常识根因）→结尾视角升级（阶段3）。本质定义、本质检验、映射回溯是思考工具，**不强制在寓言产物中露出**。
> - **输出层（按故事结构）**：寓言按故事节奏组织（开头/冲突/反转/寓意），**不按"## 阶段1/2/3"分节**。分页/分场景按剧情需要。
> - **类比**：盖房子时结构工程师用力学公式算承重（思考层），但房子外形由建筑师设计（输出层）。不能因为用了力学公式，就让房子长得像公式。
>
> **顿悟触发点在寓言中的融入位置（位置由故事节奏决定，不是分节）**：
> - 一句话点破（思考层阶段1结尾）→ 寓言开头的钩子或主角的核心台词
> - 反常识根因（思考层阶段2中段）→ 故事的冲突反转点
> - 视角升级（思考层阶段3结尾）→ 寓言寓意
>
> **关于"寓言横跨三阶"的正确理解**：指的是**思考层**——智能体在构思时让故事节奏与三阶对应（开头抛本质→中段建立反差→结尾视角升级），**不是产物分节**。

---

## 执行流程（思考层走三阶 · 输出层按故事结构）

```
第1步：本质定义先行（思考层阶段1·构思时点出本质）
    ├── 提取本质定义：揭示本质属性（不是描述外延），能解释所有其他属性（思考工具，不强制露出寓言）
    ├── 本质检验：候选定义能否解释这个东西的所有其他属性？不能则降级重写（思考校验，不露出）
    ├── 一句话点破：最简洁的本质表达，可独立传播（思考层顿悟触发·阶段1；输出层融入寓言开头的钩子或主角核心台词）
    ├── 列出2-3个核心要素（由本质定义展开的关键齿轮，作为构思寓言角色/冲突的内部依据）
    ├── 类归属：由本质定义确定它属于什么类（思考工具，不强制露出）
    └── 判断受众类型 → 决定寓言复杂度
        受众判断矩阵：
        | 用户输入特征                    | 判定受众  | 寓言篇幅    |
        |-------------------------------|---------|-----------|
        | 明确说"小学生/儿童/孩子"         | child   | 200-400字 |
        | 明确说"外行/小白/不懂的人"       | layman  | 400-600字 |
        | 概念本身是学科基础知识（如杠杆）   | layman  | 400-600字 |
        | 概念是研究生级/专业术语（如博弈论） | beginner | 400-600字 |
        | 用户用专业词汇提问（如"缓存失效策略"）| expert  | 600-800字 |
        | 未指明受众                      | layman  | 400-600字 |
       ↓
第2步：寓言生成（思考层阶段2·构思时建立反差；输出层按故事结构）
    ├── 选择载体：动物/器物/自然现象/抽象拟人（四选一，禁止用回声城/河流/钟表/群山/村庄）
    ├── 构建核心场景：一个冲突 + 一次转折 + 一个后果
    │   └── 冲突必须映射阶段1本质定义展开的反常识根因（思考层顿悟触发·阶段2；输出层融入故事的冲突反转点）
    ├── 写寓言正文（300-800字，按故事节奏：开头/冲突/转折/后果/点破）
    │   ├── 开头：用故事场景隐性呈现本质（思考层呼应阶段1本质定义；输出层是故事开场的钩子，不是抛本质定义段落）
    │   ├── 中段：建立反差（思考层用反常识根因制造认知冲突；输出层是故事的冲突/转折情节，不是分节）
    │   ├── 前80%篇幅：禁止出现概念名和该领域术语
    │   ├── 角色行为必须映射概念的核心要素（思考层映射，不在产物中显式列出）
    │   └── 结尾处用一句话点破："这个故事讲的，其实是{概念}"
    └── 自检：遮住概念名，读者能否猜到在讲什么
       ↓
第3步：迁移与视角升级（思考层阶段3·构思时展开方法；输出层按故事寓意收尾）
    ├── 寓言→概念映射表（思考层校验：哪个角色/情节对应哪个要素，必须能回溯到本质定义——作为内部思考校验，可选是否在产物中露出）
    │   └── 教学释义版可保留映射表露出；单纯寓言故事可隐藏，仅作为思考校验
    ├── 输出2个迁移问题（输出层按教学结构，不在寓言正文内）
    │   ├── 问题1（理解检验）：这个故事的哪个细节最能体现{概念}的核心？
    │   └── 问题2（跨域迁移）：如果把故事换个场景，{概念}还能怎么体现？
    └── 寓意收尾视角升级：用一句话重新定义问题（思考层顿悟触发·阶段3；输出层融入寓言寓意，不一定是独立段落）
       ↓
第4步：元素层封装（自动执行）
    └── 将本质定义+寓言+映射表+迁移问题+视角升级封装为元素层资产，供内容输出管线复用
```

> **关于"第N步"与"阶段N"的关系说明**：上面的"第1/2/3步"是工作流步骤，恰好对应思考层的"阶段1/2/3"是因为寓言构思的天然节奏与三阶合一——但这只是思考层的对应关系，**输出层（寓言正文）按故事结构组织，不分节为"阶段1/2/3"**。

## 输出结构

> **v2.1 输出层结构**：寓言正文按故事结构组织，**不按"## 阶段1/2/3"分节**。下方结构中标注"必选"的是故事结构的必要部分；标注"可选（思考校验辅助）"的是思考工具，可选择是否在产物中露出（教学释义版可保留，单纯寓言故事可隐藏）。

```markdown
## 寓言：{标题}

{300-800字寓言正文，按故事节奏：开头/冲突/转折/后果/点破}

> 这个故事讲的，其实是 **{概念}**。

---

### 迁移问题（可选 · 教学释义版保留；单纯寓言故事可省略）

1. **理解检验**：{问题1}
2. **跨域迁移**：{问题2}

### 本质定义（可选 · 思考校验辅助，不强制露出）

> {一句话点破最底层逻辑，揭示本质属性，能解释所有其他属性——思考层阶段1的本质定义，教学释义版可露出，单纯寓言故事可省略}

### 寓言→概念映射（可选 · 思考校验辅助，不强制露出）

| 寓言元素 | 对应概念要素 | 回溯到本质（内部校验列） |
|---------|-------------|----------------------|
| {角色/情节} | {概念要素} | {如何回溯到本质定义——思考校验，不强制在产物中露出此列} |
| {角色/情节} | {概念要素} | {如何回溯到本质定义——思考校验，不强制在产物中露出此列} |

> 教学释义版可保留映射表（含"回溯到本质"列）作为释义辅助；单纯寓言故事可隐藏整个映射表，仅作为内部思考校验使用。

### 视角升级（可选 · 思考校验辅助，不强制作为独立段落）

> {寓意收尾，重新定义问题，给出比阶段1更高的视角——思考层阶段3视角升级；输出层融入寓言寓意，可作为寓言结尾的内涵点出，不一定要单独段落}
```

**最小化输出（单纯寓言故事）**：仅保留"寓言标题 + 寓言正文 + 点破句"。本质定义/映射表/迁移问题/视角升级作为思考层校验，不露出产物。

**完整输出（教学释义版）**：保留全部段落，作为教学辅助。

## 失败模式编码

| 步骤 | 失败场景 | 处理动作 |
|------|---------|---------|
| 第1步 本质定义先行 | 概念过于宽泛，无法一句话说本质 | 缩小到具体子概念，重新提炼本质定义；仍失败则告知用户"概念太宽，请指定子概念" |
| 第1步 本质定义先行 | 候选定义未通过本质检验（不能解释所有其他属性） | 降级为描述性定义后再重写为本质定义；找不到核心要素（少于2个）则检查是否是基础常识，若是基础常识则跳过寓言直接解释 |
| 第2步 寓言生成 | 寓言用了禁用意象（河流/钟表/群山/村庄/回声城） | 删除重写，从冷门角度重新切入 |
| 第2步 寓言生成 | 寓言前80%出现了概念名或术语 | 删除术语，用中性词替换 |
| 第2步 寓言生成 | 寓言没有冲突，变成说教 | 强制加入"角色A想做X，但遇到Y阻碍"结构，且冲突必须映射反常识根因（思考层校验，不强制露出） |
| 第2步 寓言生成 | 遮住概念名后读者无法猜到 | 检查角色行为是否映射核心要素（思考层校验），补强映射后重写 |
| 第3步 映射与迁移 | 寓言元素能找到概念对应物但无法回溯到本质定义 | 重新映射，确保每个元素都能回溯到阶段1的本质定义（思考层校验，不强制露出产物） |
| 第3步 视角升级 | 寓意收尾变成了"综上所述"式总结 | 删除总结，重写为"重新定义问题"的视角升级句（思考层要求，输出层融入寓言寓意） |
| 第4步 元素封装 | 调用方管线格式不兼容 | 输出纯文本版+结构化JSON版两种格式 |

---

## 检查点

- 🔵 **AUTO-CHECK · 第1步完成后**（思考层校验）：输出本质定义+本质检验结果+一句话点破+核心要素+受众判断（思考层校验，不强制露出寓言产物），自动继续
- 🔴 **CHECKPOINT · 第2步完成后**（输出层校验）：展示寓言全文，确认"遮住概念名能否猜到"——若不能则重写
- 🔵 **AUTO-CHECK · 第3步完成后**（思考层+输出层校验）：输出映射表（含回溯到本质列，作为内部思考校验）、迁移问题和视角升级句，自动封装元素层资产
- 🔴 **CHECKPOINT · 跨场景调用时**：展示寓言将被插入目标场景的位置，确认后再注入

## 检查清单（两层 · 思考层 + 输出层）

### 思考层校验（不强制露出产物，但智能体构思时必走）

- [ ] 本质定义先行：已提取揭示本质属性的定义（不是描述外延），并通过本质检验
- [ ] 一句话点破：已给出最简洁的本质表达（可独立传播，作为思考层顿悟触发点）
- [ ] 核心要素：已由本质定义展开2-3个关键齿轮
- [ ] 类归属：已由本质定义确定类
- [ ] 思考层阶段2：已挖掘至少1个反常识根因，作为构思冲突反转点的内部依据
- [ ] 思考层阶段3：已生成视角升级句，作为寓意收尾的内部依据
- [ ] 映射回溯校验：寓言元素与概念要素已对应，且每一项都能回溯到本质定义（思考校验，不强制露出产物）

### 输出层校验（按故事结构 · 检查产物实际外形）

- [ ] 受众判断：已确定受众类型并匹配寓言复杂度
- [ ] 寓言正文：300-800字，前80%无概念名和术语
- [ ] 寓言结构：按故事节奏（开头/冲突/转折/后果/点破）组织，**非按"阶段1/2/3"分节**
- [ ] 顿悟触发点融入故事位置：一句话点破→开头钩子或主角台词；反常识根因→冲突反转点；视角升级→寓言寓意
- [ ] 禁用意象：未使用河流/钟表/群山/村庄/回声城
- [ ] 点破句：结尾已用一句话点出概念名
- [ ] 教学释义版（可选）：若保留映射表/迁移问题/视角升级段落，作为释义辅助；单纯寓言故事可省略这些段落

### 通用校验

- [ ] 失败模式：每个关键步骤的失败分支已编码
- [ ] 检查点：🔵/🔴 检查点已显性标记
