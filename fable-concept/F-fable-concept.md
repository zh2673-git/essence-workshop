# 场景G：概念寓言化

> **坡度定位**：寓言承担"阶段2：建立联系"，用隐喻迁移把陌生概念嫁接到熟悉情境。

---

## 执行流程（坡度化）

```
第1步：概念脱水（坡度阶段1前置）
    ├── 提取概念本质：一句话说出这个概念到底在解决什么问题
    ├── 列出2-3个核心要素（概念运转的关键齿轮）
    └── 判断受众类型 → 决定寓言复杂度
        受众判断矩阵：
        | 用户输入特征                    | 判定受众  | 寓言篇幅    |
        |-------------------------------|---------|-----------|
        | 明确说"小学生/儿童/孩子"         | child   | 200-400字 |
        | 明确说"外行/小白/不懂的人"       | layman  | 400-600字 |
        | 概念本身是学科基础知识（如杠杆）   | layman  | 400-600字 |
        | 概念是研究生级/专业术语（如博弈论） | beginner | 400-600字 |
        | 用户用专业词汇提问（如"缓存失效策略"）| expert  | 600-800字 |
        | 未指明受众                      | layman  | 400-600字 |
       ↓
第2步：寓言生成（坡度阶段2：建立联系）
    ├── 选择载体：动物/器物/自然现象/抽象拟人（四选一，禁止用回声城/河流/钟表/群山/村庄）
    ├── 构建核心场景：一个冲突 + 一次转折 + 一个后果
    ├── 写寓言正文（300-800字）
    │   ├── 前80%篇幅：禁止出现概念名和该领域术语
    │   ├── 角色行为必须映射概念的核心要素
    │   └── 结尾处用一句话点破："这个故事讲的，其实是{概念}"
    └── 自检：遮住概念名，读者能否猜到在讲什么
       ↓
第3步：正式解释与迁移（坡度阶段3：详细解释）
    ├── 概念正式定义（1-2句学术/规范表述）
    ├── 寓言→概念映射表（哪个角色/情节对应哪个要素）
    └── 输出2个迁移问题：
        ├── 问题1（理解检验）：这个故事的哪个细节最能体现{概念}的核心？
        └── 问题2（跨域迁移）：如果把故事换个场景，{概念}还能怎么体现？
       ↓
第4步：元素层封装（自动执行）
    └── 将寓言+映射表+迁移问题封装为元素层资产，供内容输出管线复用
```

---

## 输出结构

```markdown
## 寓言：{标题}

{300-800字寓言正文}

> 这个故事讲的，其实是 **{概念}**。

---

### 概念正式解释

{1-2句规范定义}

### 寓言→概念映射

| 寓言元素 | 对应概念要素 |
|---------|-------------|
| {角色/情节} | {概念要素} |
| {角色/情节} | {概念要素} |

### 迁移问题

1. **理解检验**：{问题1}
2. **跨域迁移**：{问题2}
```

---

## 失败模式编码

| 步骤 | 失败场景 | 处理动作 |
|------|---------|---------|
| 第1步 概念脱水 | 概念过于宽泛，无法一句话说本质 | 缩小到具体子概念，重新脱水；仍失败则告知用户"概念太宽，请指定子概念" |
| 第1步 概念脱水 | 找不到核心要素（少于2个） | 检查是否是基础常识；若是基础常识则跳过寓言直接解释 |
| 第2步 寓言生成 | 寓言用了禁用意象（河流/钟表/群山/村庄/回声城） | 删除重写，从冷门角度重新切入 |
| 第2步 寓言生成 | 寓言前80%出现了概念名或术语 | 删除术语，用中性词替换 |
| 第2步 寓言生成 | 寓言没有冲突，变成说教 | 强制加入"角色A想做X，但遇到Y阻碍"结构 |
| 第2步 寓言生成 | 遮住概念名后读者无法猜到 | 检查角色行为是否映射核心要素，补强映射后重写 |
| 第3步 正式解释 | 寓言与概念映射不上 | 回到第2步，确认寓言核心场景确实映射概念本质 |
| 第4步 元素封装 | 调用方管线格式不兼容 | 输出纯文本版+结构化JSON版两种格式 |

---

## 检查点

- 🔵 **AUTO-CHECK · 第1步完成后**：输出概念脱水结果（本质+要素+受众判断），自动继续
- 🔴 **CHECKPOINT · 第2步完成后**：展示寓言全文，确认"遮住概念名能否猜到"——若不能则重写
- 🔵 **AUTO-CHECK · 第3步完成后**：输出映射表和迁移问题，自动封装元素层资产
- 🔴 **CHECKPOINT · 跨场景调用时**：展示寓言将被插入目标场景的位置，确认后再注入

---

## 检查清单

- [ ] 概念脱水：已提取一句话本质 + 2-3个核心要素
- [ ] 受众判断：已确定受众类型并匹配寓言复杂度
- [ ] 寓言正文：300-800字，前80%无概念名和术语
- [ ] 寓言结构：有冲突 + 转折 + 后果，非说教
- [ ] 禁用意象：未使用河流/钟表/群山/村庄/回声城
- [ ] 点破句：结尾已用一句话点出概念名
- [ ] 正式解释：已给出1-2句规范定义
- [ ] 映射表：寓言元素与概念要素已对应
- [ ] 迁移问题：已输出理解检验 + 跨域迁移各1题
- [ ] 失败模式：每个关键步骤的失败分支已编码
- [ ] 检查点：🔵/🔴 检查点已显性标记
